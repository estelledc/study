旧方案把候选选择、多阶段写作/审查、worker 合并、构建、远端发布和跨轮接力组合为一个长任务，并用内容数量作为持续运行目标。

它被停用的原因：

- worker commit 没有严格的路径边界；
- 队列与锁缺少完整事务/所有权语义；
- 发布失败可能被后续步骤掩盖；
- 活动文档与当前小轮次、安全发布政策冲突；
- 数量目标不能代表学习价值或内容可信度。

后续只保留有限小轮次、显式授权、fail-closed 合并与发布。当前合同见 `../operations-policy.md`。
