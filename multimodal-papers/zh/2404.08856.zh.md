## 摘要

多模态大语言模型（Multimodal Large Language Model, MLLM）的推理速度较慢，这是因为其大语言模型（Large Language Model, LLM）骨干网络存在内存带宽瓶颈（memory bandwidth bottleneck），并且以自回归（auto-regressive）方式逐词生成。在本文中，我们探索了将投机解码（speculative decoding）应用于提升 MLLM 的推理效率，特别是针对 LLaVA 7B 模型。我们证明，纯语言模型可以作为 LLaVA 7B 投机解码的优秀草稿模型（draft model），无需从草稿模型中引入图像词元（image tokens）及其相关处理组件。我们在三个不同任务上的实验表明，使用我们从零开始训练的 1.15 亿参数的纯语言模型作为草稿模型，投机解码可以实现高达 2.37 倍的内存受限加速（memory-bound speedup）。此外，我们引入了一种紧凑的 LLaVA 草稿模型，该模型集成了一个图像适配器（image adapter），在图像描述任务上表现出边际性能提升，而在其他任务上保持可比较的结果。

多模态大语言模型（Multimodal Large Language Model, MLLM）的推理速度较慢，这是因为其大语言模型（Large Language Model, LLM）骨干网络存在内存带宽瓶颈（memory bandwidth bottleneck），导致推理速度慢 [19]。

投机解码（speculative decoding）[3, 7, 9, 15, 20] 被提出作为加速 LLM 推理且不损失准确率的解决方案：一个较小的草稿模型（draft model）预测多个未来词元，这些词元在一次 LLM 调用中被验证。考虑到 MLLM 具有 LLM 骨干网络，投机解码可以用来使 MLLM 的推理更加高效。已有许多近期工作研究了投机解码及其变体 [2, 5, 7, 8, 18, 20] 在 LLM 上的应用，但据我们所知，在 MLLM 的背景下尚无此类工作。

在本文中，我们将投机解码应用于 LLaVA 7B 模型（以 LLaMA 7B 模型作为语言模型骨干），使推理更加高效，其框图如图 1 所示。由于 LLaVA 和 LLaMA 系列中缺乏公开可用的小于 70 亿参数的模型，我们从零开始训练了一个 1.15 亿参数的语言模型用于投机解码。我们证明，不考虑图像词元（因此不需要图像编码器（vision encoder）和适配器（adapter））的纯语言模型可以作为 LLaVA 7B 的优秀草稿模型。我们在三个不同任务上进行了实验，包括在 LLaVA Instruct 150K 数据集 [13] 上的图像问答（image QA）、在 COCO 数据集 [11] 上的图像描述（image captioning）以及在 ScienceQA 数据集 [14] 上的推理任务，使用的草稿模型候选者经过了不同阶段的训练和微调。我们的结果表明，仅使用纯语言模型作为草稿模型即可实现高达 2.37 倍的内存受限加速。我们还创建了一个小型 LLaVA 草稿模型，该模型包含一个图像适配器以及我们训练的语言模型，并证明其在 COCO 描述任务和 ScienceQA 任务上略有提升，而在其他任务上与纯语言模型草稿模型表现相似。

## 1. 引言

大语言模型（Large Language Model, LLM）因其令人印象深刻的性能已广泛应用于各个领域。然而，LLM 仅接受文本查询作为输入，而现实世界的数据以多种模态的形式存在，包括视觉数据。多模态大语言模型（Multimodal Large Language Model, MLLM）[1, 13, 21, 22] 为 LLM 提供了图像理解能力，视觉和文本词元的融合增强了模型与用户的交互，从而产生更有依据的响应。MLLM 由一个图像编码器（vision encoder）用于处理图像信息，以及一个适配器（adapter）用于将图像编码转换为语言模型嵌入空间。此外，MLLM 以 LLM 的形式具有语言模型骨干网络，因此继承了自回归（auto-regressive）生成和内存带宽受限（memory-bandwidth）的特性。

## 2. 背景

### 2.1 投机解码（Speculative Decoding, SPD）

投机解码（Speculative Decoding, SPD）[3, 9] 涉及一个较小的草稿模型生成多个词元，这些词元由目标 LLM 并行验证。给定输入上下文 $X_{1:n} := [X_1, \ldots, X_n]$，草稿模型以自回归方式生成词元序列 $\hat{X}_{n+1:n+L}$，其中 $\hat{X}_{n+j} \sim p(\cdot|X_{1:n}, \hat{X}_{n+1:n+j-1})$。然后，草稿词元通过一次目标 LLM（q）调用来验证，使用拒绝采样（rejection sampling）准则，该准则保证与目标 LLM 相同的输出词元分布。具体而言，词元 $\hat{X}_{n+j}$ 以概率

$$
\min\left(1, \frac{q(\hat{X}_j|X_{1:n}, \hat{X}_{n+1:n+j-1})}{p(\hat{X}_j|X_{1:n}, \hat{X}_{n+1:n+j-1})}\right)
$$

被接受。如果草稿词元 $\hat{X}_{n+j}$ 被拒绝，则从残差分布中采样新词元，该残差分布定义为 $p_{\text{res}}(x) = \max(0, q(x) - p(x))$。

### 2.2 多模态大语言模型（Multimodal Large Language Model, MLLM）

基于图像的多模态大语言模型（MLLM）由以下部分组成：1）一个视觉编码器（vision encoder）用于编码输入图像，2）一个适配器（adapter）用于将图像编码转换为语言模型嵌入，以及 3）一个语言模型骨干网络。我们更详细地描述 LLaVA 模型的框架如下：给定输入图像 $I$ 和文本查询 $Q$，图像 $I$ 被转换为长度为 $m$ 的图像编码序列 $H_1, H_2, \ldots, H_m$，文本查询被转换为词元嵌入序列 $X_1, X_2, \ldots, X_n$。图像编码进一步通过适配器 $g_\theta$（一个小型多层感知机（multi-layer perceptron））转换为图像嵌入 $V_i = g_\theta(H_i)$。这是为了将编码 $H_i$ 转换到语言模型嵌入空间。然后词元由语言模型以图像嵌入和词元嵌入为条件生成，如下所示：

$$
X_{n+1} \sim q(\cdot|V_{1:m}, X_{1:n}) \tag{1}
$$

## 3. MLLM 的 SPD

要通过投机解码获得更高的加速收益，我们需要一个草稿模型显著小于目标模型（LLaVA-7B）且与之高度对齐。在先前关于 LLM 的工作中，草稿模型最常见的选择是使用来自与目标模型相同模型家族的小型预训练模型，或训练一个具有与目标模型相同架构的较小模型 [15]。由于 LLaVA 家族中不存在公开可用的更小模型，我们需要从零开始训练一个草稿模型。草稿模型架构的一个自然选择是遵循 LLaVA 的架构，其中草稿模型包含一个适配器和一个参数量少于 LLaVA 7B 的语言模型骨干。在我们的方法中，我们同时使用：1）一个较小的 LLaVA 草稿模型，该模型包含一个较小的图像适配器和一个草稿语言模型，以及 2）一个纯语言草稿模型，该模型仅以文本为条件生成草稿词元。

为了实现更高的投机解码加速收益，我们需要一个草稿模型显著小于目标模型（LLaVA-7B）且与之高度对齐。在先前关于 LLM 的工作中，草稿模型最常见的选择是使用来自与目标模型相同模型家族的小型预训练模型，或训练一个具有与目标模型相同架构的较小模型 [15]。由于 LLaVA 家族中不存在公开可用的更小模型，我们需要从零开始训练一个草稿模型。草稿模型架构的一个自然选择是遵循 LLaVA 的架构，其中草稿模型包含一个适配器和一个参数量少于 LLaVA 7B 的语言模型骨干。在我们的方法中，我们同时使用：1）一个较小的 LLaVA 草稿模型，该模型包含一个较小的图像适配器和一个草稿语言模型，以及 2）一个纯语言草稿模型，该模型仅以文本为条件生成草稿词元。

**草稿模型候选者。** 我们训练了一个遵循 LLaMA-2 架构的 1.15 亿参数的草稿模型。我们遵循 [6] 的训练流程，使用下一个词元预测损失（next-token-prediction loss）在 600B 英文词元上从头预训练（pre-train）一个草稿模型，并使用 TVD++ 损失 [6] 在指令微调（instruction finetuning）数据集上微调草稿模型。我们进一步在 LLaVA Instruct 150K 数据集 [13] 的一个子集上微调草稿模型。在我们的实验中，我们在每个训练和微调阶段之后考虑以下四个草稿模型：1）base-LLaMA，一个在使用下一个词元预测损失在 600B 英文词元上预训练后的草稿 LLaMA 模型；2）chat-LLaMA，一个遵循 [6] 的指令微调草稿 LLaMA 模型，使用 base-LLaMA 草稿模型初始化；以及 3）微调后 LLaVA（fine-tuned-LLaVA, ft-llava），一个微调后的 LLaVA 草稿模型，其中图像适配器通过 LLaVA-7B 图像适配器的子克隆（subcloning）[17] 初始化，语言模型从 chat-LLaMA 草稿模型初始化（该模型随后在 LLaVA 数据集上微调）。我们还包含另一个草稿模型 4）微调后 LLaVA-文本（fine-tuned-LLaVA-text, ft-llava-text），该模型仅使用第 3 个模型的语言模型部分。注意，只有微调后 LLaVA 草稿模型使用图像信息，而所有其他草稿模型仅消耗输入提示的文本部分；当草稿模型使用图像信息时，视觉编码器（基于 CLIP [16]）与目标模型共享，以避免重新计算图像嵌入。详细参数见附录 A.1。

**评估任务。** 我们专注于开放式文本生成和带推理的多选题问答，以鼓励生成更多的词元，这对使用投机解码有利。为此，我们在 1）LLaVA Instruct 150K 数据集 [13]、2）COCO 数据集 [11] 上的图像描述任务以及 3）带思维链（chain-of-thought, CoT）推理 [14] 的 ScienceQA 上进行评估。所有任务的系统提示设置见附录 A.2。

**指标。** 投机解码的有效性通过以下指标评估：1）块效率（block efficiency, $\tau$），即每块（或目标模型运行）平均生成的词元数；对于大小为 $\gamma$ 的块和输入 $x$，$\tau(x)$ 的最大值可以为 $\gamma + 1$，块大小（$\gamma$）在一些文献中也被称为草稿长度（draft length, DL）；2）内存受限加速（memory-bound speedup, MBSU），即对于给定块效率 $\tau(x)$ 和定义为草稿模型与目标模型参数数量之比的相关延迟 $c$ 所达到的假设加速，即 $\text{MBSU}(x) = c\tau(x)^{\frac{c\gamma}{c\gamma+1}}$；3）词元速率（token rate），生成的词元总数除以生成总时间，给出每秒生成的词元估计值。我们使用不同块大小 $\gamma \in \{3, 5\}$ 在各种任务上测量这些指标。

**解码。** 我们在所有实验中使用贪婪解码（greedy decoding），以使投机解码生成与目标模型的自回归生成完全相同。我们将探索基于采样的解码（变化温度、变化 top-p、top-k）在 MLLM 的投机解码背景下的工作留作未来工作。

**结果。** 我们的结果表明，使用投机解码与 LLaVA 7B 目标模型可以显著加速输出生成，我们强调当使用不含任何图像信息的草稿模型时，投机解码仍然可以提供与使用图像信息的草稿模型相当且具有竞争力的加速。从图 2（顶部和中间图）可以看出，使用投机解码在块效率和 MBSU 方面实现了超过 2 倍的提升。对于每个任务，当块大小从 3 增加到 5 时，性能趋势相似，唯一的例外是 SQA 任务，其中 base-llama 草稿模型在块大小为 5 时表现优于其他纯文本草稿模型。对于 LLaVA-eval 任务，在两种块大小（3 或 5）下，ft-llava-text 草稿模型表现最佳，其次是 ft-llava。对于 COCO-Caption 任务，ft-llava 表现最佳，其次是 ft-llava-text，两种块大小均如此。最后，对于 SQA 任务，块大小为 3 时，ft-llava 草稿模型表现最佳，其次是 ft-llava-text；而块大小为 5 时，ft-llava-text 草稿模型表现最佳，其次是 ft-llava。

我们进一步在 COCO-Caption 任务上提供定性结果，以展示在使用微调后 LLaVA-文本草稿模型时生成过程中接受的词元，因此草稿模型不使用图像信息，如图 5 所示。根据图中的输出生成结果，其中蓝色带下划线的词元是被接受的词元，我们观察到草稿模型可以预测常见词汇、介词以及单词的一半。例如，给定 "vege"，草稿模型可以预测 "tables"。类似地，在第二个例子中，给定上下文和额外词元 "app"，草稿模型能够预测 "liances"。我们相信，一般而言，开放式文本生成包含多个由常见词汇、介词和单词补全组成的词元，这些不需要图像词元的知识，因此即使是不使用图像信息的草稿模型也能提供具有竞争力的性能。此外，草稿模型还可以预测一旦生成后某些词元的重复。例如，在第二张图中，"counter" 和 "bowls" 这两个词一旦被目标模型生成，草稿模型就可以多次预测它们。最后，在小型多模态语言模型上进行更严格的训练留作我们未来的工作。

A. 附录
A.1. 模型配置

LLaVA-7B 模型包含三个部分：(i) 视觉编码器（vision encoder），(ii) 基于多层感知机（multi-layer perceptron, MLP）的图像适配器/投影器（image adapter/projector），以及 (iii) LLaMA 7B 语言模型。视觉编码器为 CLIP ViT-L/14，详见 [16]；基于 MLP 的图像适配器包含 2 个线性层（linear layer），尺寸分别为 1024 × 4096 和 4096 × 4096。当草稿模型也包含图像适配器时，尺寸为 1024 × 1024 和 1024 × 1024。

目标语言模型和草稿语言模型部分均采用 LLaMA 架构，具体配置如下：

| | 目标模型 (7B) | 草稿模型 (115M) |
|---|---|---|
| 层数 (Layers) | 32 | 4 |
| 注意力头数 (Attention heads) | 32 | 8 |
| 中间维度 (Intermediate dim) | 11,008 | 2,816 |
| 隐藏维度 (Hidden dim) | 2,048 | 1,024 |
| 激活函数 (Activation) | SiLU | SiLU |

A.2. 系统提示词

各任务使用以下系统提示词。特殊图像标记（image token）`<image>` 用于嵌入图像数据。

LLaVA-eval。我们遵循 [13] 中的提示风格。LLaVA 的每个样本包含多个问题和回答，我们将其拆分为不同样本。

```
<s> 一位好奇的用户与一个人工智能助手之间的对话。助手对用户的问题给予有帮助、详细且礼貌的回答。用户：<image>
问题 Q1 助手：回答 R1 。用户：问题 Q2 。. . . .
```

COCO-caption。由于 COCO 数据集不包含任何问题提示词，我们使用与上述类似的提示词提示模型。

```
<s> 一位好奇的用户与一个人工智能助手之间的对话。助手对用户的问题给予有帮助、详细且礼貌的回答。用户：<image>
请对给定图像提供详细的描述 助手：
```

Science QA。我们遵循 [14] 中的提示风格，使用单个上下文内示例（in-context example）（包含问题、选项、回答和推理过程）来启用思维链（Chain-of-Thought, CoT）推理。此外，我们仅考虑具有关联图像（associated image）的测试样本。

```
问题：question :Iiques
选项：(0) option : Ii1 (1) option : Ii2 (2) option : Ii3
上下文：context : Iicont
回答：答案是 Iians 。因为：lectureIilect 解释：Iiexp

<image>
问题：question :Itest
选项：(0) option : Itest,1 (1) option : Itest,2 (2) option : Itest,3
上下文：context : Itest
回答：
```

其中，下标 i 表示上下文内示例。
