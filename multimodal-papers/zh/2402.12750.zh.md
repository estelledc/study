## 摘要

多模态大语言模型（Multimodal Large Language Model, MLLM）的最新发展取得了快速进展，正朝着创建能够理解来自各种模态输入的通用 MLLM 这一目标迈进。然而，现有方法通常依赖于使用配对的多模态指令数据进行联合训练，这既消耗资源，又难以扩展到新的模态。在本文中，我们提出了一种新的范式——通过对现有 MLLM 进行模型组合来创建一个新模型，该模型保留每个原始模型的模态理解能力。我们的基础实现 NaiveMC 通过复用模态编码器（modality encoder）和合并大语言模型（Large Language Model, LLM）参数来证明该范式的有效性。此外，我们引入了 DAMC，以解决合并过程中的参数干扰和错配问题，从而提升模型性能。为了促进该领域的研究，我们提出了 MCUB，一个用于评估 MLLM 理解来自不同模态输入能力的基准（benchmark）。在该基准以及其他四项多模态理解任务上的实验表明，我们的方法相比基线有显著改进，证明模型组合能够创建出一个能够处理来自多个模态输入的通用模型。

## 1 引言

大语言模型（Large Language Model, LLM）的最新进展已使其成为多模态学习范式的先锋（Liu et al., 2023b,a）。主流方法涉及通过大量模态-文本配对数据将模态编码器与大语言模型对齐，然后使用模态特定的指令数据进行微调（fine-tuning）。这一范式已在多种模态上取得成功应用，如图像（Liu et al., 2023b; Dai et al., 2023）、音频（Gong et al., 2023; Deshmukh et al., 2023; Tang et al., 2024）、视频（Zhang et al., 2023; Lin et al., 2023）和点云（Hong et al., 2023; Xu et al., 2023），从而涌现出多种具有独特模态能力的 MLLM。

也有一些工作致力于使单一 MLLM 能够处理多种模态。一种实现方式是将 LLM 与多模态编码器（如 ImageBind（Girdhar et al., 2023））对齐，仅使用图像-文本数据（图 1(a)）（Su et al., 2023; Han et al., 2023）。该方法利用了多模态编码器内不同模态的内在对齐（alignment）特性，使 MLLM 能够在某种程度上理解各种模态。然而，由于缺乏模态特定的指令数据，该方法通常导致次优性能。另一种方法涉及使用多个模态编码器联合训练 MLLM（图 1(b)）。例如，ChatBridge（Zhao et al., 2023）通过多模态指令数据（即视频-音频对话）的联合训练过程，将图像、视频和音频编码器与 LLM 连接起来。这类方法展现出潜力，但面临两大挑战：首先，收集跨多种模态的配对数据消耗资源；其次，将这些模型适应新模态需要额外的训练，增加了开发过程的复杂性和资源需求。

鉴于当前方法的局限性，我们提出并研究一种更实际的设定：MLLM 的模型组合（图 1(c)）。我们的核心研究问题简单而根本：我们能否通过组合现有 MLLM 来创建一个新的 MLLM，使其无需训练即可继承对不同模态的理解能力？MLLM 模型组合具有两大关键优势：（1）消除了资源密集型的训练和多模态数据收集过程；（2）增强了适应性，便于无缝引入新模态。

一些近期研究，如 X-InstructBLIP（Panagopoulou et al., 2023），是 MLLM 模型组合的开创性工作。这些工作主要通过训练投影器（projector）来将不同编码器与单一 LLM 对齐，并展示了同时处理多种模态的能力。然而，一个关键限制是它们仅适用于具有冻结语言模型权重的 MLLM。这一约束限制了可利用的模型范围，并损害了 MLLM 的整体性能（Zeng et al., 2023）。

在本文中，我们首先提出一个面向 MLLM 的模型组合框架。我们的实现命名为 NaiveMC，简洁优雅：对于要组合的 MLLM，我们直接复用其模态特定的编码器并合并其 LLM 参数。我们证明，只要从同一 LLM 初始化，MLLM 就能通过该模型组合框架实现零样本（zero-shot）多模态扩展，无论 LLM 的参数是否经过了微调。

此外，为了缓解组合过程中的参数干扰并优化组合模型的性能，我们提出了 DAMC，一个具有参数解耦（parameter Decoupling）与调整（Adjustment）的高级模型组合框架。通过在初始 MLLM 训练阶段将模态特定参数与语言模型参数分离，DAMC 允许选择性合并，减少跨模态干扰。此外，DAMC 引入了一个自适应参数调整机制，以确保组合模型的最佳兼容性和有效性，实现了平衡且高效的多模态扩展。

为了评估我们提出的框架的有效性，我们在需要综合理解四种主流模态（图像、音频、视频和点云）的不同组合输入的任务上进行了全面实验。为了促进 MLLM 模型组合的研究，我们还构建了 MCUB，这是一个专门用于评估模型同时理解多种模态能力的基准，通过识别来自不同模态输入的共性来实现。实验结果表明，我们的框架能够组合来自不同模态的现有 MLLM，且无需进一步训练，从而产生一个能够处理这些模态任意组合的通用且高性能的多模态模型。

我们的贡献有三方面：

- 我们提出了 MLLM 模型组合的概念，通过 NaiveMC 框架实现，允许无缝集成不同 MLLM 而无需额外训练，实现零样本多模态扩展。
- 我们引入了 DAMC，一种先进的模型组合框架，采用参数解耦和自适应调整来缓解参数干扰并优化多模态组合模型性能。
- 我们创建了 MCUB，一个用于评估多种模态统一理解的基准，并通过在各种多模态理解任务和 MCUB 上的广泛实验证明了我们模型组合框架的有效性。

## 2 相关工作

### 2.1 多模态大语言模型

近期研究已将大语言模型（Large Language Model, LLM）与图像、视频、音频和点云等多种模态相结合。X-LLM（Chen et al., 2023）等方法使用模态特定的适配器（adapter），而 ChatBridge（Zhao et al., 2023）为每种模态采用一个 Perceiver。Macaw-LLM（Lyu et al., 2023）使用统一的对齐模块，而 NeXT-GPT（Wu et al., 2023）则依赖线性投影进行多模态融合。然而，这些方法通常需要联合多模态数据集训练，带来可扩展性和扩展挑战。虽然 PandaGPT（Su et al., 2023）和 ImageBind-LLM（Han et al., 2023）使用 ImageBind（Girdhar et al., 2023）作为统一编码器绕过了联合训练，但由于缺乏模态特定的指令数据，其性能受限。相比之下，我们的方法受益于每个 MLLM 的指令遵循训练，并通过免训练过程集成其能力。

### 2.2 模型组合

模型组合主要通过权重空间合并策略（Ilharco et al., 2022; Matena and Raffel, 2022; Jin et al., 2023; Huang et al., 2023; Yu et al., 2024; Wortsman et al., 2022）来整合不同模型的能力，增强来自同一初始化的微调模型。然而，现有研究尚未充分探索面向多模态 LLM 的模型组合。先前的多模态任务工作使用线性插值处理语音识别（Sundar et al., 2023）等任务。另一项工作通过使用相同参数将分别处理图像和文本的模型转换为同时处理（Sung et al., 2023），但并未扩展模型的模态处理能力。X-InstructBlip（Panagopoulou et al., 2023）通过为模态编码器和 LLM 训练独立的投影（projection）来引入新模态，但其性能受限于冻结的 LLM。我们的工作通过支持 LLM 训练并将模型组合技术应用于 LLM 权重来推进多模态能力，显著改善了多模态融合的效果。

## 3 方法论

### 3.1 任务定义

在本工作中，我们研究一组多模态大语言模型（MLLM）的组合，记为 $\{M_1, M_2, \ldots, M_n\}$，每个模型能够以不同模态 $m = \{m_1, m_2, \ldots, m_n\}$ 响应文本查询。核心目标是开发一种组合方法 $C$，将这些 MLLM 有效地整合为一个更通用的单一模型 $M_{\text{compose}} = C(M_1, M_2, \ldots, M_n)$。这种整合旨在使模型能够处理和理解为来自 $m$ 中任意模态组合的输入。例如，通过整合两个专用 MLLM——一个视觉 LLM 和一个音频 LLM——所得的组合模型不仅应保留这些模型各自处理图像和音频的能力，还应获得同时处理同时涵盖视觉和听觉信息的输入的零样本能力。

**算法 1 MLLM 的模型组合**

**输入**：MLLM 集合 $\{M_1, M_2, \ldots, M_n\}$，每个模型具有一组参数 $\Theta_i$
**输出**：整合后的参数 $\Theta_{\text{compose}}$

1: 将 $\Theta_{\text{common}}$ 和 $\Theta_{\text{compose}}$ 初始化为空集
2: 定义一个映射 $f: \theta \mapsto G$，该映射根据功能将参数 $\theta$ 映射到组 $G$
3: for $i = 1$ to $n$ do
4:     for $\Theta_i$ 中的每个参数 $\theta$ do
5:         if $\forall j \neq i, \theta \notin \Theta_j$ then
6:             将 $\theta$ 添加到 $\Theta_{\text{compose}}$
7:         else
8:             将 $\theta$ 分配到 $\Theta_{\text{common}}$ 中的组 $f(\theta)$
9:         end if
10:    end for
11: end for
12: for $\Theta_{\text{common}}$ 中的每个组 $G$ do
13:     $\theta_{\text{merge}} = \text{average}(G \text{ 中的参数})$
14:     将 $\theta_{\text{merge}}$ 添加到 $\Theta_{\text{compose}}$
15: end for
16: return $\Theta_{\text{compose}}$

### 3.2 一个模型组合框架

在组合模型时，必须考虑两个关键要素：组件及其权重。主流的 MLLM 通常具有两种组件类型：（1）模态特定组件，如模态编码器和投影器，将模态输入适配到语言嵌入空间；（2）模态无关组件，存在于每个 MLLM 中，主要是底层 LLM 本身。在我们的组合框架中，我们保留来自不同 MLLM 的所有模态特定组件（及其权重）来处理各自的模态输入，并将它们连接到同一个 LLM。如果 LLM 在 MLLM 训练期间未经过适配（如图 2(a) 所示），我们直接采用 LLM 的预训练权重。反之，如果 LLM 在 MLLM 训练过程中经历了适配（图 2(b)），我们只需平均其权重。我们将此组合框架命名为 NaiveMC，并在算法 1 中提供了其形式化过程。

图 2：模型组合过程示意图。为简单起见，仅考虑图像和音频两种模态。(a) 和 (b) 展示了第 3.2 节描述的基础模型组合框架，而 (c) 和 (d) 展示了第 3.3 节详述的带参数解耦的模型组合。

### 3.3 参数解耦

在前面描述的框架中，存在一个未解决的问题，即在合并微调后的 LLM 参数时可能存在参数干扰。在 MLLM 训练期间，LLM 参数针对特定模态输入进行了优化。这种专门化优化可能导致不同 MLLM 的 LLM 参数产生差异。当这些参数被合并时，可能产生冲突，影响模型理解模态输入的能力。

为了缓解模型组合期间的参数干扰问题，我们主张首先使用参数解耦策略训练 MLLM。如图 2(c) 和图 2(d) 所示，主要思想是将 MLLM 内的模态处理参数与语言模型参数分离。例如，在处理模态 $m$ 的 MLLM $M$ 中，每个注意力层的输入记为 $X = [X_m, X_t]$，其中 $X_m$ 和 $X_t$ 分别表示模态特定序列和文本序列。注意力组件计算如下：

$$Q = [X_m W_m^Q, X_t W_t^Q] \tag{1}$$

$$K = [X_m W_m^K, X_t W_t^K] \tag{2}$$

$$V = [X_m W_m^V, X_t W_t^V] \tag{3}$$

其中 $Q$、$K$ 和 $V$ 分别表示注意力机制中的查询（query）、键（key）和值（value）。注意，使用两组权重 $W_m^{(\cdot)}$ 和 $W_t^{(\cdot)}$，分别针对模态特定和文本输入定制。然后对注意力输出应用注意力操作：

$$O = [X_m^O, X_t^O] = \text{split}(\text{Attention}(Q, K, V)) \tag{4}$$

这将注意力输出分割为模态特定输出和文本输出。最终的统一输出表示 $X_o$ 通过以下方式获得：

$$X_o = [X_m^O W_m^O, X_t^O W_t^O] \tag{5}$$

确保模型内每种模态有独立的处理流。类似地，对于每个前馈层的输入 $X$，输出为：

$$\text{FFN}(X) = [\text{FFN}_m(X_m), \text{FFN}_t(X_t)] \tag{6}$$

其中 $\text{FFN}_m$ 和 $\text{FFN}_t$ 分别是用于模态特定输入和文本输入的前馈层。

在组合通过参数解耦训练的 MLLM 时，我们仅合并文本相关参数，保持不同的模态特定参数，如图 2(d) 所示。组合模型作为前面描述方法的自然扩展，保证每种模态的输入由其各自参数独立处理。通过这样做，它有效降低了来自其他模态的干扰风险，确保组合模型在处理多模态数据时保持高保真度。请注意，在 MLLM 训练完成后，组合阶段仍然完全免训练。

### 3.4 自适应参数调整

由于不同 MLLM 在数据质量和训练策略上存在差异，其性能可能显著不同。因此，在组合这些模型时，采用简单的平均策略往往无法达到最优结果。为了提升模型效果，在组合过程中对模型参数实施自适应调整至关重要，以增强不同模型之间的兼容性和灵活性。具体而言，对于 $N$ 个不同的 MLLM $\{M_1, M_2, \ldots, M_N\}$ 以及这些模型中共同的任意参数 $\theta_i$（其中 $i = 1 \ldots N$），合并参数定义为：

$$\theta_{\text{merge}} = \sum_{i=1}^{N} \lambda_i \theta_i \tag{7}$$

其中 $\lambda_i$ 表示调整系数。为简化起见，我们对 $M_i$ 中的所有 $\theta_i$ 采用统一的调整系数 $\lambda_i$。对于使用参数解耦训练的模型，如有需要，我们还可以调整其模态特定参数。这些系数的值可以通过来自目标任务（需要各种模态输入）的验证集来确定。如果没有这样的验证集，一个实用的替代方案是基于模型在每种模态任务上的整体性能来选择系数。我们将带有参数解耦和调整的更新模型组合框架称为 DAMC。

### 3.5 多模态共性理解基准

如前所述，DAMC 天然具有扩展模态数量的优势。然而，很少有可用的基准来评估跨多种模态的性能。为了证明我们的方法在涉及大量模态的任务上的有效性，受 Panagopoulou et al.（2023）启发，我们引入了一个新基准，称为多模态共性理解基准（Multimodal Commonality Understanding Benchmark, MCUB）。我们在图 3 中提供了一个 MCUB 示例。MCUB 的任务是衡量模型识别来自不同模态的输入实体之间共性的能力，并从给定的四个候选中选择最合适的答案。

我们利用 GPT-4（Achiam et al., 2023）从现有的描述数据集（captioning dataset）中创建 MCUB。具体流程如下：我们从现有数据集中采样输入，将具有相同语义的输入归为一组，并保留语义相似度最高的组。这种相似度通过对其各自描述的相似度取平均来量化。随后，以上下文示例（in-context example）为提示让 GPT-4 为每组生成问题、选项和正确答案。在本研究中，我们开发了两个 MCUB 变体：MCUB-3 和 MCUB-4。MCUB-4 包含包含四种模态输入的数据条目——图像、视频、音频和点云。相比之下，MCUB-3 由四个子集组成，每个子集代表这四种模态中任意三种模态的输入组合。更多细节请参阅附录 C。

## 4 实验

### 4.1 实现

在我们的实验设置中，我们探索了四种模态：图像、音频、视频和点云。为了确保全面且具有可比性的分析，我们按照先前工作（Liu et al., 2023a; Lin et al., 2023; Panagopoulou et al., 2023; Xu et al., 2023）重新实现了每种模态的 MLLM。对于每种模态，我们训练三个版本的 MLLM：使用相同的数据和超参数但改变 LLM 参数的可训练性——一个具有冻结 LLM 的模型、一个完全可训练的 LLM 模型，以及一个具有参数解耦的可训练 LLM 模型，如图 2 所示。我们采用 LoRA（Hu et al., 2021）技术进行高效的 LLM 训练。所有模型均基于 Vicuna-7B-v1.5（Zheng et al., 2023）。训练 MLLM 的更多细节见附录 A。我们通过在进行 $N$ 种模态组合时，在集合 $[1/N, 2/N, \ldots, N/N]$ 中搜索最优 $\lambda_i$ 值来应用参数调整，基于每种模态任务的性能进行选择。
## 4.2 评测（Evaluation）

数据集与基准测试（Datasets and Benchmarks）。我们的目标是对组合多模态大语言模型（MLLM）处理来自多模态输入的性能进行评测。除第 3.5 节介绍的 MCUB 基准外，我们还纳入了来自两个关键领域的数据集进行评测：(1) 音视频问答（Audio-Visual Question Answering），包括 MUSIC-AVQA（Li et al., 2022）和 AVQA（Yang et al., 2022），这两个数据集提供了图像、视频和音频输入；(2) ModelNet40（Wu et al., 2015）和 Objaverse（Deitke et al., 2023）上的三维物体分类任务，这两个数据集考虑了图像和点云输入。

基线对比（Baselines）。我们的评测策略主要与不利用多模态训练数据的模型进行比较。我们考虑了两个代表性基线：ImageBind-LLM，该模型使用图文配对数据将多模态编码器 ImageBind 与大语言模型（LLM）对齐；以及 X-InstructBLIP，该模型使用各模态专属指令数据将单个模态编码器与冻结的大语言模型对齐。在模型组合方面，我们进一步将所提出的 DAMC 方法与两种替代中间基线进行比较：Proj-only，其中大语言模型保持冻结，仅组合模态编码器和连接器（connector）；以及 NaiveMC，其直接组合多模态大语言模型，但不采用我们提出的参数解耦与调整技术。

两个任务上零样本音视频问答的实验结果，采用视频（V）、图像（I）和音频（A）输入的不同组合。本研究开发的方法以灰色背景标注以示区分。

## 4.3 主要结果（Main Results）

音视频问答、三维物体分类和 MCUB 的主要实验结果分别列于表 1、表 2 和表 3 中。在表中，符号"X+Y"表示评测模型时使用的不同类型模态输入组合。例如，"V+A"表示将视频（V）和音频（A）输入组合在一起作为模型输入的一部分。在模型组合方面，我们根据所需的特定模态输入来组合多模态大语言模型。总体而言，我们提出的 DAMC 在所有输入组合和任务上均取得了最高性能，证明其能够熟练处理来自多模态的输入，即使未对这些模态组合进行任何训练。此外，我们还观察到：

组合更多模态输入可带来性能提升。结果揭示了一致趋势：随着集成更多模态输入，模型组合方法带来性能改善。例如，在表 1 的 MUSIC-AVQA 任务中，从仅使用视频（V）输入过渡到包含图像（I）和音频（A）输入的组合，带来了显著性能提升。具体而言，当组合这三种模态时，我们的 DAMC 达到峰值 57.32，相比 V+I/A 有明显提升。相比之下，ImageBind-LLM 和 X-InstructBLIP 等模型在从 V+I/A 过渡到 V+I+A 组合时未表现出类似提升。这表明通过模型组合开发的模型能够更有效地处理来自不同模态的输入，展现了其改善多模态理解的能力。

DAMC 优于强基线。与先前方法相比，DAMC 实现了卓越性能，甚至超越了利用配对多模态数据的方法，如 ChatBridge 和 3D-LLM（Hong et al., 2023）。在各种模型组合方法之间进行比较时，DAMC 显著优于 Proj-only 和 NaiveMC，在集成超过两种模态的场景中尤为明显。以 MCUB 基准为例，DAMC 在组合三种模态的场景下实现了显著性能提升，相比次优的 NaiveMC 提高了 +5.10 分；在组合四种模态的子集中提高了 +6.05 分。这一差异证明了 DAMC 在合并多个多模态大语言模型时能够有效缓解干扰，展现了其鲁棒性和效率。

## 5 讨论与分析（Discussion and Analysis）

**具有不同训练策略的多模态大语言模型组合（Composition of MLLMs with different training strategies）。** 在第 3.3 节中，我们主张使用参数解耦训练多模态大语言模型。有趣的是，这种解耦也可以在推理时实现，方法是将参数复制以模拟在模态特定输入和文本输入上的相同参数。通过这种方式，我们可以组合具有不同训练策略的模型。我们在 MUSIC-AVQA 数据集上采用 V+I+A 输入进行了实验。我们改变视频和图像多模态大语言模型的训练策略，并与使用参数解耦训练的音频多模态大语言模型进行组合。表 6 的结果表明，推理时的解耦同样有效，这对于旨在集成未使用参数解耦训练的现有模型的情况十分有益。

**组合模型在单模态任务上的性能（Composite model performance on the single-modality task）。** 为探索组合模型在单模态任务上的效果，我们在视觉语言理解评测基准 MME（Fu et al., 2023）上进行了实验。结果如表 5 所示。每个组合模型仅处理图像和文本问题。∆ 值表示 DAMC 相对于 NaiveMC 的相对提升。随着集成模型数量的增加，NaiveMC 的性能趋于下降，而 DAMC 则持续表现出提升。这凸显了我们提出的方法在组合模型时有效缓解参数干扰的能力。此外值得注意的是，模型组合即使在仅涉及单模态的任务中也能提升性能。我们将其归因于我们的方法可能在组合过程中整合了来自不同模型的知识，这一点我们将留待未来研究。

## 6 结论（Conclusion）

本文中，我们提出并探索了多模态大语言模型的模型组合（model composition）概念，展示了其无需额外训练即可无缝集成多种模态能力的能力。首先，我们引入了一个基础模型组合框架，称为 NaiveMC。在此基础上，我们设计了 DAMC，该框架采用参数解耦与调整策略来减少跨模态干扰并优化组合模型的性能。此外，我们构建了 MCUB 基准用于多模态共性的相对评测。大量实验证明了我们方法的有效性。我们希望这项工作能够激发对多模态模型组合的进一步探索。

**局限性（Limitations）。** 我们的探索仅限于四种常用模态，未对整个潜在模态谱进行全面考察。这一局限性可能导致错失发现模型组合更多益处或挑战的机会。此外，我们提出的方法已在特定尺寸的模型上进行了测试，这些方法在更大规模多模态大语言模型上的适用性仍有待未来研究。

## C. 多模态共性理解基准的详细信息

表 11 展示了我们用于生成 MCUB 任务的描述数据集。对于点云模态，我们遵循 Xu 等人（2023）的方法，从 Objaverse（Deitke 等人，2023）数据集中保留 3000 个点云，并从 Cap3D（Luo 等人，2023）数据集中获取其描述。注意，这部分点云未用于训练和三维目标分类任务。

一组实体的语义相似性通过对所有两两实体组合的描述的相似性取平均来获得，由 all-MiniLM-L6-v2 模型（Reimers 和 Gurevych，2019）计算。例如，如果给定一组实体，其描述对为 (A, B, C)，则该组实体的相似度为 (A, B)、(A, C) 和 (B, C) 之间相似度的平均值。
