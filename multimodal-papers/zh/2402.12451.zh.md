## 摘要

连接文本和视觉模态在生成式智能中扮演着至关重要的角色。因此，受大语言模型（Large Language Models, LLMs）成功的启发，大量研究工作正投入到多模态大语言模型（Multimodal Large Language Models, MLLMs）的开发中。这些模型能够无缝整合视觉与文本模态，同时提供基于对话的界面和遵循指令的能力。在本文中，我们对近期基于视觉的 MLLMs 进行了全面综述，分析了其架构选择、多模态对齐策略和训练技术。我们还对这些模型在广泛任务上的表现进行了详细分析，包括视觉定位（visual grounding）、图像生成与编辑、视觉理解以及领域特定应用。此外，我们整理并描述了训练数据集和评估基准，从性能和计算需求角度对现有模型进行了比较。总体而言，本综述全面概述了当前的最新技术水平，为未来的 MLLMs 奠定了基础。

## 1 引言

注意力机制（attention operation）和 Transformer 架构（Vaswani et al., 2017）的提出使得创建能够在大规模上处理多种模态的模型成为可能。这一进步很大程度上归功于该算子的通用性和架构的适应性。最初，这一突破被应用于语言特定模型（Devlin et al., 2018; Brown et al., 2020），但很快扩展到支持多种模态（Li et al., 2019; Lu et al., 2019）并促进它们在统一嵌入空间中的整合（Radford et al., 2021）。

复杂大语言模型（Large Language Models, LLMs）的兴起，特别是其上下文学习（in-context learning）能力，鼓励研究人员将这些模型的范围扩展到涵盖多种模态，包括作为输入和输出。这一扩展催生了 GPT-4V（Achiam et al., 2023）和 Gemini（Anil et al., 2023）等前沿模型，展现了最先进的性能。

多模态大语言模型（Multimodal Large Language Models, MLLMs）的发展涉及将视觉和语言的单模态架构进行融合，通过视觉到语言适配器（vision-to-language adapters）建立有效的连接，并设计创新的训练方法。这些方法对于确保模态对齐和准确遵循指令的能力至关重要。

在模型快速发布的背景下，我们的目标是提供对 MLLM 领域的全面概述，重点关注利用视觉模态的模型。这一概述既是对当前现状的更新，也是对未来发展的灵感来源。我们确定了定义这些模型的三个核心方面：其架构、训练方法论以及它们被设计来执行的任务。我们首先详细介绍了为 LLMs 配备跨模态能力所采用的视觉编码器（vision encoders）和适配器模块的主流选择。随后，我们深入探讨了训练过程和数据。接着，我们探索了 MLLMs 所解决的任务范围。综述最后讨论了该领域持续存在的挑战和未来研究的有前景方向。有关训练数据、评估数据集、性能和计算需求的更多细节见补充材料。

本综述的动机源于 MLLM 领域日益增长的学术兴趣，这一点从不断增加的发表作品数量中可见一斑。与现有综述（Yin et al., 2023a; Wu et al., 2023b; Huang et al., 2023a）相比，本文存在显著差异。 notably，它解决了先前工作中被忽视的几个关键领域，包括视觉定位、图像生成和编辑。此外，我们的综述详细介绍了每个讨论的 MLLM 使用的主要组件，例如视觉编码器（visual encoders）和所使用的特定 LLM。此外，我们的分析从定性和计算需求角度提供了讨论论文的性能比较，结合了定量结果和基准的详细信息。通过这种全面方法，本综述旨在填补现有空白，并提供对该领域当前格局的更细致理解。

## 2 赋予 LLMs 多模态能力

### 2.1 预备知识

**大语言模型**。Brown et al. (2020) 发现，上下文学习（in-context learning），即在提示词前添加几个展示 LLM 期望输出的示例（Chowdhery et al., 2023; Hoffmann et al., 2022; Tay et al., 2022），可以提高其性能，尤其是在未见过的任务上。通过为 LLM 提供每个训练样本期望任务的自然语言描述，可以进一步增强泛化能力。这种技术称为指令微调（instruction-tuning）（Chung et al., 2022; Wang et al., 2022b,a; Jiang et al., 2024），它对于将 LLM 的行为与人类行为对齐至关重要，并赋予了最先进的 LLMs 能力，最终通过人类反馈的强化学习（reinforcement learning from human feedback, RLHF）得到提升（Ouyang et al., 2022; Achiam et al., 2023; Chen et al., 2023l; Bai et al., 2023a）。

**参数高效微调（PEFT）**。当预训练的 LLM 需要适应特定领域或应用时，参数高效微调（parameter-efficient fine-tuning, PEFT）方案是训练整个 LLM 的重要替代方案，因为这些策略仅引入少量新参数。其中，提示调优（prompt-tuning）（Hambardzumyan et al., 2021; Lester et al., 2021; Li and Liang, 2021; Liu et al., 2023j）学习一小组向量，在输入文本之前以软提示（soft prompts）的形式输入到模型中。而 LoRA（Hu et al., 2021）则通过学习低秩矩阵来限制新权重的数量。这种技术与量化方法（如 QLoRA（Dettmers et al., 2024））正交，后者进一步降低了 LLM 的内存占用，相较于通常的半精度权重。

**走向多模态 LLMs**。MLLMs 的发展遵循与 LLMs 相似的路径，其中 Flamingo（Alayrac et al., 2022）是第一个在大规模上探索视觉-语言领域上下文学习的模型。随后，视觉指令微调（visual instruction-tuning）（Liu et al., 2023e）迅速成为多模态领域最突出的训练范式，以及使用 PEFT 技术微调 LLM。任何 MLLM 都包含至少三个组件（图 1）：作为与用户交互界面的 LLM 主干网络（LLM backbone），一个（或多个）视觉编码器（visual encoders），以及一个或多个视觉到语言适配器（vision-to-language adapter）模块。LLM 主干网络的常用选择通常属于 LLaMA 系列（Touvron et al., 2023a,b），因为它们的权重可以免费获取，完全在公开数据上训练，并且具有不同规模以适应各种用例。此外，它们的衍生版本也很流行，如 Alpaca（Taori et al., 2023）和 Vicuna（Chiang et al., 2023）。前者使用 GPT-3 编写的指令对 LLaMA 进行微调，而后者则利用用户与 ChatGPT（OpenAI, 2022）共享的对话。替代选择包括 OPT（Zhang et al., 2022b）、Magneto（Wang et al., 2023b）、MPT（MosaicML, 2023），以及指令调优（Chung et al., 2022）或多语言（Xue et al., 2020）版本的 T5（Raffel et al., 2020），这是一个为多任务预训练的编码器-解码器语言模型。

**模型组件的预训练**。MLLMs 的主要组件是视觉编码器和语言模型。视觉编码器旨在为 LLMs 提供视觉信息，最常用的是基于 CLIP 的架构（Radford et al., 2021; Wortsman et al., 2022），其预训练目标是对齐 CLIP 嵌入（CLIP embeddings），这得益于对比损失（contrastive loss），该损失对齐正确的图像-文本对。例外是 EVA-CLIP 模型系列（Fang et al., 2023），它利用 MAE 预训练策略（He et al., 2022）在可见图像块的条件下重建被掩码的图像-文本对齐视觉特征来重建被掩码的视觉特征。另一方面，LLMs 主要依赖广泛使用的 Transformer 模型，尽管 Mamba 架构（Gu and Dao, 2023）近期也崭露头角。该架构提出使状态空间模型（State-Space Model, SSM）具有时间依赖性，有效地创建了一个具有优良性质的选择性 SSM：（i）推理成本和内存需求随序列长度线性扩展，以及（ii）得益于算法的智能 GPU 实现而实现高效的并行训练。与 Transformer 类似，用于语言建模的 Mamba 模型使用下一个词预测（next token prediction）任务进行预训练。非常近期的研究提出了以 Mamba 作为语言主干的 MLLMs（Qiao et al., 2024; Zhao et al., 2024）。

### 2.2 视觉编码器

在 MLLMs 中，关键组件之一是视觉编码器，它专门设计用于向 LLM 提供提取的视觉特征。通常的做法是在训练期间使用冻结的预训练视觉编码器，仅训练一个可学习的接口来连接视觉特征和底层 LLM。虽然这通常使用固定宽高比的分辨率图像完成，但一些尝试（Xu et al., 2024; Li et al., 2023l）涉及使预训练的视觉主干能够处理不同分辨率和宽高比的图像。有关如何处理高分辨率图像的更多细节见补充材料。

最常使用的视觉编码器基于具有 CLIP 目标的预训练视觉 Transformer（Vision Transformer, ViT）模型，以利用 CLIP 嵌入的内在对齐。流行的选择包括来自 CLIP（Radford et al., 2021）的 ViT-L 模型、来自 OpenCLIP（Wortsman et al., 2022）的 ViT-H 主干，以及来自 EVA-CLIP（Fang et al., 2023）的 ViT-g 版本。

如（Li et al., 2023g）所示，更强的图像编码器带来更好的性能。基于这一见解，Lin et al. (2023b) 和 Gao et al. (2024) 提出了冻结的视觉主干集成，以捕获鲁棒的视觉表示和不同层次的信息粒度。同时，PaLI 模型（Chen et al., 2023j,h）注意到语言和视觉参数之间的不平衡，提议将视觉主干分别扩展到 40 亿和 220 亿参数的 ViT。

使用如此大而强大的模型之所以可行，归因于训练期间保持视觉编码器冻结的常见做法（Li et al., 2023g; Huang et al., 2023b; Gao et al., 2023; Chen et al., 2023f）。然而，使用冻结的视觉编码器存在一些局限性，主要源于参数数量受限，导致视觉和语言模态之间的对齐不足。具体来说，从视觉模型提取的密集特征可能会碎片化细粒度图像信息，并在输入语言模型时因序列过长而带来大量计算。为了缓解这一问题，其他方法（Ye et al., 2023c,d）采用两阶段训练范式。在第一阶段，他们结合可训练的视觉主干，同时保持预训练的 LLM 冻结。根据他们的发现，使视觉编码器可训练提高了视觉问答或视觉描述等任务的性能。然而，它可能导致其他任务的性能下降，表明了一定程度的遗忘和对通用视觉表示的损害。

### 2.3 视觉到语言适配器

来自不同模态的输入同时存在强调了对能够 delineate 这些单模态域中潜在对应关系的模块的需求。这些模块被称为"适配器"（adapters），旨在促进视觉和文本域之间的互操作性。通用 MLLMs 中使用了一系列不同的适配器，从线性层或多层感知机（MLP）等基础架构到基于 Transformer 的解决方案等先进方法，以 Q-Former 模型为例，以及条件添加到 LLM 的交叉注意力（cross-attention）层。

**线性和 MLP 投影**。将视觉输入投影到文本嵌入中最直接的方法是学习一个线性映射，将视觉特征转换为与文本对应物相同的维度。一些方法如 LLaMA-Adapter（Gao et al., 2023）和 FROMAGe（Koh et al., 2023b）仅使用单层线性层执行多模态连接，而 LLaVA-1.5（Liu et al., 2023d）采用两层 MLP，展示了改进的多模态能力。尽管在早期 MLLMs 中被广泛采用，线性投影已被证明即使在具有更先进视觉输入理解的最新方法（Chen et al., 2023f; Lin et al., 2023a; Wang et al., 2023c; You et al., 2023; Zhao et al., 2023a）中也非常有效。因此，这是一种简单而有效的技术，用于将视觉特征与文本对应物对齐。另一种方法（Cha et al., 2023）提议用卷积层替换线性层，展示了适度的改进。

**Q-Former**。它是 BLIP-2（Li et al., 2023g）中提出的基于 Transformer 的模型，随后被多种其他方法（Chen et al., 2023d; Dai et al., 2023; Hu et al., 2024）采用。其特点是其灵活的架构，由两个共享互相关注层的 Transformer 块组成，促进了视觉和文本表示之间的对齐过程。它涉及一组可学习的查询，在自注意力层内交互，并通过交叉注意力机制与视觉特征接口。文本和视觉元素通过模块内的共享自注意力进行通信。

受 Q-Former 启发，引入了各种修改版本。在这方面，mPLUG-Owl 模型（Ye et al., 2023c,d）简化了 Q-Former 架构，并提出了一种视觉抽象器组件，通过将视觉信息压缩为不同的可学习标记来运行，以导出更具语义丰富性的视觉表示。同样，Qwen-VL（Bai et al., 2023b）使用单层交叉注意力模块与可学习查询压缩视觉特征，该查询还整合了 2D 位置编码。

**额外的交叉注意力层**。这种方法由 Flamingo（Alayrac et al., 2022）提出，在 LLM 的现有预训练层中集成了密集交叉注意力块。新添加的层通常与零初始化的 tanh 门控机制结合使用，以确保在初始化时，条件模型的行为与其原始版本相同。使用额外的交叉注意力层增加了从头训练它们的需要，与其他替代方案相比增加了可训练参数的数量。为了降低计算成本，Flamingo 在冻结的 LLM 上执行这些操作。

### 2.4 训练策略

MLLMs 的训练可以大致分为两个主要阶段。首先，视觉编码器和适配器经过预训练或联合训练，以将视觉信息映射到 LLM 的词嵌入空间。其次，模型在视觉指令数据上进行微调，以学习遵循指令并与用户进行对话的能力。一些方法还执行额外的第三阶段，使用高质量数据或特定任务数据进一步改进模型。

**单阶段训练**。一些方法（Zhu et al., 2023a; Liu et al., 2023e）在视觉指令微调数据上直接训练整个模型，而无需单独的视觉对齐阶段。这种策略更简单，但可能需要更多数据和计算资源。

**两阶段训练**。在两阶段训练的第一个阶段，目标是将图像特征与文本嵌入空间对齐。此阶段之后，输出往往支离破碎且不连贯。因此，第二步是提高多模态对话能力。LLaVA（Liu et al., 2023e）采用了这种方法，首先对齐视觉和语言特征，然后在视觉指令数据上微调 LLM，使其能够执行有意义的对话。其他方法也遵循类似的范式，使用对比损失或生成损失进行初始对齐，然后使用指令遵循数据进行微调。

**全模型微调与部分参数更新**。根据可用资源，一些方法选择更新所有模型组件，也可能使用仅文本数据来保留 LLM 的对话能力。

**两阶段训练**。在两阶段训练的第一个阶段，目标是将图像特征与文本嵌入空间对齐。此阶段之后，输出往往支离破碎且不连贯。因此，第二步是提高多模态对话能力。LLaVA（Liu et al., 2023e）采用了这种方法，首先对齐视觉和语言特征，然后在视觉指令数据上微调 LLM，使其能够执行有意义的对话。其他方法也遵循类似的范式，使用对比损失或生成损失进行初始对齐，然后使用指令遵循数据进行微调。

## 2.4 多模态训练

从预训练大语言模型（LLM）出发，多模态大语言模型（MLLM）的训练经历单阶段或两阶段过程。两种情况下均采用标准交叉熵损失进行下一 token 预测，作为自回归目标。

**单阶段训练。** LLaMA-Adapter（Gao et al., 2023）探索了这种可能性，它引入了额外的可训练参数来封装视觉知识，并同时管理纯文本指令学习。为此，该模型使用图像-文本对和指令进行联合训练，在独立参数上运行。与此同时，（Koh et al., 2023b）提出的模型通过为图像-文本检索加入两个对比损失来调整最终损失函数，训练过程中仅更新三个线性层。另一条线路上，Kosmos-1（Huang et al., 2023b）采用冻结的视觉骨干网络，从头训练 13 亿参数的语言模型。

Flamingo（Alayrac et al., 2022）及其开源变体（Awadalla et al., 2023; Laurençon et al., 2024）则训练交叉注意力层和基于 Perceiver 的组件，将视觉特征与冻结的 LLM 模块相连接。此外，Otter（Li et al., 2023b）扩展了 Flamingo 的训练以增强其上下文学习能力。鉴于当前可用的训练数据量，SPHINX-X（Gao et al., 2024）等方法选择执行单阶段一体化训练。值得注意的是，Flamingo 及其衍生模型（Awadalla et al., 2023; Laurençon et al., 2023e,d）是最早引入视觉指令遵循训练方案的模型之一，该方案作为第二阶段训练，同时更新多模态适配器和 LLM 的参数；而在第一阶段，仅多模态适配器可训练。与之不同，MiniGPT-4（Zhu et al., 2023a）的显著特点是在两个阶段中均仅训练负责多模态对齐的线性层，第二阶段使用模型自身在第一阶段后收集和过滤的数据。

另一种方法如 InstructBLIP（Dai et al., 2023）所示，涉及冻结视觉编码器和 LLM。在两个训练阶段中，只有 Q-Former 和连接模块可训练。与先前方法中视觉骨干网络保持冻结不同，mPLUG-Owl（Ye et al., 2023c,d）在初始阶段更新视觉骨干网络，有利于同时捕获低层和高层视觉信息。同样，在第二阶段联合使用纯文本和多模态数据以增强对齐。与之不同，Shikra（Chen et al., 2023f）在两个阶段中更新所有权重，唯一的例外是视觉骨干网络保持冻结。

**训练数据。** 在第一（或单）训练阶段，数据集主要由大规模、公开可用且未经策划的数据组成。例如，Conceptual Captions 3M（CC3M）数据集（Sharma et al., 2018）由 300 万张图像与专门为图像描述系统设计的文本描述配对组成。与广泛使用且经过策划的 MS-COCO（Lin et al., 2014）数据集（用途类似）不同，CC3M 中的图像和描述从网络收集，展现出更广泛的风格和内容范围。类似地，LAION 系列（Schuhmann et al., 2021, 2022）代表了从网页来源获取的大规模非策划图像-文本对的扩展集合，为预训练多模态语言模型提供了丰富的资源。此外，COYO-700M（Byeon et al., 2022）数据集作为重要资源尤为突出，包含 7.47 亿图像-文本对。值得注意的是，COYO-700M 中的每个替代文本（alt-text）都链接到 HTML 文档中的一张图像。此外，DataComp（Gadre et al., 2023）呈现了从通用爬虫（common crawl）来源的 128 亿经过过滤的图像-文本对的庞大资源池。

需要强调的是，训练初始阶段使用的数据集（通常由大规模非策划数据组成）与后续精炼阶段选定的数据集之间存在区别：前者强调多样性和规模，后者则关注特异性和任务相关性，从而促进更精细的模型优化方法。特别是在单训练阶段方法中，某些方法（Alayrac et al., 2022; Laurençon et al., 2024）也利用交错数据集（interleaved datasets），其中图像与来自网络的文本交错排列，旨在扩充大型模型的数据集规模（Hoffmann et al., 2022）。这些数据集中的图像可以位于句首或句中，允许模型支持任意交错的图像和文本序列作为输入，从而通过融合文本和视觉元素增强输入格式的灵活性。这些数据集中最常用的是 WebLI（Chen et al., 2023j），由 100 亿图像和图像-文本对组成；MMC4（Zhu et al., 2023d），作为纯文本 C4（Raffel et al., 2020）数据集的扩展，由 3.65 亿份文档和 1560 亿个涉及不同概念的标记组成；以及 OBELICS（Laurençon et al., 2024），一个开放且经过策划的交错图像-文本网页文档集合，包含 1.41 亿份文档、1150 亿文本标记和 3.53 亿张图像。

在构成 MLLM 第二阶段训练的视觉指令微调（visual instruction tuning）背景下，可用数据量有限。这一限制主要源于创建过程耗时且不够规范化。在此阶段，使用不同的数据集来提升一系列下游任务的性能。其中，LLaVA-Instruct（Liu et al., 2023e）是 GPT-4 生成的多模态指令遵循数据的集合，包含 15.8 万个独特的语言-图像描述，涵盖多种任务类型，包括 5.8 万个对话、2.3 万个详细描述和 7.7 万个复杂推理。类似地，LRV-Instruction（Liu et al., 2023c）最初由 GPT-4 生成的 40 万个视觉指令组成，最近又新增了 30 万个视觉指令。为了增强指令调优中的鲁棒性，LRV-Instruction 还包含跨三个语义层次组织的负指令，表明在该数据集上进行指令调优的 MLLM 相比原始版本受幻觉影响更小。此外，LLaVAR（Zhang et al., 2023i）利用公开可用的 OCR 工具，从 LAION 数据集的 42.2 万张富含文本的图像上收集结果。该流程首先收集 42.2 万张含噪声的富文本图像，然后通过 OCR 模型提取文本，在 GPT-4 的帮助下，将提取结果和描述用于创建 1.6 万段对话，同时加入特定问题以创建复杂指令，有助于提升模型在新任务上的表现。

## 3 使用 MLLM 解决视觉任务

标准 MLLM 能够处理视觉理解任务，如视觉问答（VQA）、描述生成和多轮对话。然而，最近的研究兴趣转向解决更细粒度的视觉任务，如视觉定位（visual grounding）和图像生成。

### 3.1 视觉定位

MLLM 的视觉定位能力对应于与用户进行包含内容定位的对话能力，也称为指称对话（Chen et al., 2023f）。具体而言，You et al.（2023）引入 referring 作为理解输入区域内容的能力，可在区域描述（region captioning）和指称表达式生成（referring expression generation）等任务上进行评估。相反，grounding 与定位给定文本描述的区域相关联，对应于指称表达式理解（REC）、指称表达式分割（RES）、短语定位（phrase grounding）和带定位的描述生成（grounded captioning）等任务。使 MLLM 具备这些能力需要两个主要组件：一种区域到序列（region-to-sequence）方法来处理输入区域，以及一种序列到区域（sequence-to-region）方法来定位名词和短语。表 2 总结了具备视觉定位能力的 MLLM。

**区域即文本（Region-as-Text）。** 输出区域最常用的方式是直接将它们以一系列坐标的形式插入到生成文本中，以数字或专门的定位分箱（location bin）特殊标记表示。Shikra（Chen et al., 2023f）、Kosmos-2（Peng et al., 2023）、MiniGPT-v2（Chen et al., 2023e）、Ferret（You et al., 2023）、CogVLM（Wang et al., 2023c）、SPHINX（Lin et al., 2023b）、Qwen-VL（Bai et al., 2023b）和 Griffon（Zhan et al., 2023）通过指示两个点将边界框转换为文本。VisionLLM（Wang et al., 2023e）、VistaLLM（Pramanick et al., 2023）、LLaFS（Zhu et al., 2023c）和 ChatSpot（Zhao et al., 2023b）允许 MLLM 通过将多边形表示为一系列点来处理多边形。

**嵌入即区域（Embedding-as-Region）。** 另一种方案是通过区域编码器读取输入区域，并将从 MLLM 最后一层提取的嵌入提供给解码器作为输出区域。对于输入区域，GLaMM（Rasheed et al., 2023）、GPT4RoI（Zhang et al., 2023g）、ASM（Wang et al., 2023d）和 ChatterBox（Tian et al., 2024b）利用图像编码器的特征在边界框上执行 ROI 对齐（ROI align），而 PVIT（Chen et al., 2023a）则利用 RegionCLIP（Zhong et al., 2022）。PixelLLM（Xu et al., 2023a）和 LLaVA-G（Zhang et al., 2023d）分别使用 SAM（Kirillov et al., 2023）和 Semantic-SAM（Li et al., 2023e）的提示编码器。对于输出区域，LISA（Lai et al., 2023）、GLaMM、GSVA（Xia et al., 2023b）、NeXt-Chat（Zhang et al., 2023a）和 LISA++（Yang et al., 2023b）将对应特殊标记的嵌入发送到 SAM 的掩码解码器，LLaVA-G 发送到 OpenSeeD（Zhang et al., 2023c），Lenna（Wei et al., 2023）发送到 Grounding-DINO（Liu et al., 2023i），PixelLM（Ren et al., 2023b）发送到自定义轻量像素解码器。

与此不同，ContextDET（Zang et al., 2023）引入了一个解码器，接收具有可学习查询的名词潜在嵌入，执行与图像特征的交叉注意力，然后使用分割头。ChatterBox（Tian et al., 2024b）结合了 iTPN-B 编码器（Tian et al., 2023）和 MLLM 的特征，并将它们提供给 DINO 检测器（Zhang et al., 2022a）。GELLA（Qi et al., 2024）在 Mask2Former（Cheng et al., 2022）中提出了一个融合模块，基于多模态图像特征生成掩码，并提出了一个关联模块来分配潜在嵌入。PaLI-3（Chen et al., 2023i）通过 VQ-VAE（Van Den Oord et al., 2017）解码器将嵌入转换为分割掩码。

**文本到定位（Text-to-Grounding）。** 其他方法基于接受文本类别作为输入的开集（open-vocabulary）模型。DetGPT（Pi et al., 2023）为 Grounding-DINO 生成一个类别列表。BuboGPT（Zhao et al., 2023c）利用 RAM、Grounding-DINO 和 SAM 的组合，将标签与输出序列中的名词进行匹配。

### 3.2 图像生成与编辑

虽然最初的 MLLM 擅长从视觉数据中提取信息，但近期研究已涵盖视觉输出的生成。这一进展通过将 MLLM 与图像生成机制（主要由 Stable Diffusion（SD）（Rombach et al., 2022）模型体现）集成而实现。这些模型具有在文本或视觉嵌入上调节的去噪 U-Net（Ronneberger et al., 2015）架构，通过交叉注意力层实现。表 3 列出了所分析模型的完整清单。

**将 MLLM 与扩散模型连接。** GILL（Koh et al., 2023a）是将冻结 LLM 的输出嵌入空间映射到冻结扩散模型输出嵌入空间的先驱。具体而言，受 Q-Former 启发，通过最小化语言模型的图像输出表示与 SD 的期望调节嵌入之间的 ℓ₂ 距离来训练映射组件。

虽然 GILL 不对 LLM 和扩散 U-Net 进行微调，但其他方法对语言模型进行微调以扩展其多模态生成能力。在这方面，Kosmos-G（Pan et al., 2023）通过整合 LLM 输出与编码器-解码器结构的训练方案开发而成，利用重建损失和 CLIP 文本嵌入内的距离最小化。类似地，MiniGPT-5（Zheng et al., 2023）除加入 GILL 的对齐损失外，还包含扩散模型的重建损失。此外，它将整体训练过程分为两个不同阶段：初始阶段专注于文生图生成，后续阶段专注于交错的视觉-语言生成。不同地，研究人员研究了从输入图像提取的离散视觉标记（Jin et al., 2023; Ge et al., 2023a,b）和连续视觉标记（Zhu et al., 2023b）与 SD 调节嵌入之间的对齐。这通常通过对文本模型（Zhu et al., 2023b; Ge et al., 2023a,b）进行微调来实现。相反，Jin et al.（2023）对 LLM 和 SD U-Net 都进行微调。

## A 处理不同分辨率和宽高比的图像

大多数现有多模态大语言模型（Multimodal Large Language Model, MLLM）以低分辨率和固定的方形宽高比感知图像。一些工作（Liu et al., 2023d; You et al., 2023; Chen et al., 2023j）已证明，采用在更高分辨率上训练好的视觉骨干网络（visual backbone），可以减少幻觉并提升多模态理解能力，从而在需要细粒度细节的任务上表现更优。然而，将 MLLM 扩展到任意输入分辨率和宽高比会引发两个重要问题：（i）从训练时见到的较小图像切换到推理时的较大图像时的适应问题；（ii）由于视觉编码器和 LLM 中 token 数量的增加所带来的计算成本——这源于注意力机制的二次方复杂度。下面，我们区分三种不同的方法来解决这些问题。

**位置编码插值（Positional-Encoding Interpolation）。** 这些模型对其在低分辨率下训练好的视觉骨干网络的位置编码进行插值，以处理高分辨率图像。虽然方法简单，但这些方法容易出现适应问题。因此，它们至少执行一个高分辨率训练阶段来部分缓解这个问题。为了降低输入到 LLM 的序列长度，从而减少计算成本，MiniGPT-v2（Chen et al., 2023e）和 VILA（Lin et al., 2023a）提出将多个视觉 token 投影到 LLM 嵌入空间中的同一个 token 中。出于同样的原因，mPLUG-Owl2（Ye et al., 2023d）和 Qwen-VL（Bai et al., 2023b）使用可学习的查询（learnable queries），将视觉特征压缩为固定长度的序列，与分辨率无关。后者还利用窗口注意力机制进一步节省了 ViT 骨干网络（ViT backbone）大部分层的计算量。

**子图像切片（Sub-Images Slicing）。** 为了避免适应问题，一些方法提出将高分辨率图像切片为多个固定大小的子图像。Flamingo（Alayrac et al., 2022）将输入图像调整为与训练分辨率相同的固定方形分辨率，然后每个子图像独立由视觉骨干网络处理，同时将整体图像缩小到相同分辨率，并将特征拼接以获得全局表示。SPHINX（Lin et al., 2023b）在视觉骨干网络的训练分辨率处将输入图像划分为方形网格子图像（即 2×2 或 3×3）。此外，为了处理矩形宽高比，SPHINX 对图像进行填充以达到所需的方形尺寸。对于极端宽高比，填充会导致子图像完全由填充像素组成。因此，在 SPHINX-X（Gao et al., 2024）中引入了 skip token 来替换仅与填充子图像相关的噪声 token，并减少提供给 LLM 的序列长度，提高效率。类似地，LLaVA-NeXT（Liu et al., 2024b）忽略了仅由填充组成的子图像，并通过引入一种特殊的 token 来指示一行子图像的结束位置，从而处理不同形状的网格。Monkey（Li et al., 2023l）使用类似 Perceiver 的重采样器（Perceiver-like resampler）从每个子图像中提取固定长度的序列，并在由多个集成于 ChatGPT 的视觉专家模型筛选的图像-文本数据集上进行训练。InfiMM-HD（Liu et al., 2024a）提出了一个动态分辨率适应训练阶段，将图像尺寸增加到 1344 像素。它采用门控交叉注意力层（gated cross-attention layers，同 Flamingo）将视觉特征注入 LLM，而不会增加输入序列长度。LLaVA-UHD（Xu et al., 2024）找到了最优的分区方案，使子图像最接近视觉编码器的原生分辨率和宽高比。视觉 token 的数量通过类似 Perceiver 的适配器进行压缩，并以空间模式排列，使 LLM 能够理解子图像的网格结构。

**其他（Others）。** 另一种解决方案，即 OtterHD（Li et al., 2023a），可以无缝处理任意分辨率或宽高比，因为它直接将 30×30 像素的大图像块馈送到 LLM，而无需视觉编码器。而 LLaVA-HR（Luo et al., 2024）则引入了混合分辨率适应（mixture-of-resolution adaptation），将通过 CNN 提取的高分辨率特征与 ViT 本身产生的低分辨率特征融合到 ViT 层中。

## B 额外训练数据

特定的训练数据集对于赋予 MLLM 视觉定位（visual grounding）和图像生成能力至关重要。下面我们简要介绍该领域的常见选择。

**视觉定位（Visual Grounding）。** 为了实现视觉定位，MLLM 可以使用预定的指令模板直接在任务特定数据上进行训练。例如，CoinIt（Pramanick et al., 2023）是一个统一的 14 个基准数据集集合，被转换为指令微调（instruction-tuning）格式，涵盖从单图像粗粒度到多图像区域级的各种任务。通常，还需要在指令微调数据集（如 LLaVa-Instruct（Liu et al., 2023h））上执行额外的训练步骤，以保留 MLLM 的对话能力。然而，一些方法创建了自定义数据集来同时提升定位和对话能力。具体而言，Shikra（Chen et al., 2023f）、DetGPT（Pi et al., 2023）、ChatSpot（Zhao et al., 2023b）和 PVIT（Chen et al., 2023a）利用 LLM（Achiam et al., 2023; OpenAI, 2022）从同时包含两种标注的数据集（如 COCO）中组合区域和描述。不同地，Kosmos-2（Peng et al., 2023）和 Ferret（You et al., 2023）利用开放词汇检测器（open-vocabulary detector）（Li et al., 2022）对从描述中解析出的名词短语进行定位，然后重建指称表达（referring expressions）。ASM（Wang et al., 2023d）、GLaMM（Rasheed et al., 2023）和 LLaVA-G（Zhang et al., 2023d）提出了基于现成模型的自动化流程，包含多个步骤，用于生成大量与其对应图像相关的对话语料库。

**图像生成与编辑（Image Generation and Editing）。** 要执行图像生成，需要同时包含文本描述和图像的数据集，如第 2.4 节中提到的那类数据集（例如 LAION-400M、COYO-700M 和 COCO）。要实现交错文本-图像生成（interleaved text-image generation），MMC4、OBELICS 和 VIST（Huang et al., 2016）是常用的选择。而对于图像编辑任务，通常还需要使用如 InstructPix2Pix（Brooks et al., 2023）和 MagicBrush（Zhang et al., 2023e）中引入的额外数据集。

## C 评估

MLLM 在不同基准上接受评估，涵盖从经典的视觉理解和识别技能到高级多模态对话能力的各个方面。表 4 展示了最常见 MLLM 在标准视觉问答（VQA）和描述生成（captioning）数据集以及专门为 MLLM 评估设计的基准上的性能。

### C.1 标准基准（Standard Benchmarks）

MLLM 最重要的能力之一是基于给定的输入图像有效回答问题的能力。这一能力通过多个视觉问答数据集进行定量评估，衡量 MLLM 所提供答案的准确率（Antol et al., 2015）。

- VQAv2（Goyal et al., 2017）是 VQA（Antol et al., 2015）的扩展和平衡版本，通过为同一问题收集相似图像但答案不同的方式构建。这使得那些忽略视觉信息、仅依赖语言先验来回答问题的模型难以表现良好。报告的结果对应于 test-dev 划分。

- GQA（Hudson and Manning, 2019）基于 Visual Genome 场景图标注（Krishna et al., 2017），包含 113k 张图像和 22M 个问题，侧重于场景理解和组合性。我们报告 test 划分的结果（占总图像的 10%）。

- OKVQA（Marino et al., 2019）是一个用于研究视觉-语言模型如何处理视觉问题（其答案无法完全从图像中找到）的基准，鼓励系统也依赖外部知识。测试集包含 14,055 个开放式问题。

- VizWiz（Gurari et al., 2018）来源于视力障碍者拍摄图像并表述相关询问的真实场景，每个样本附带 10 个回答。验证划分包含 4,319 张图像与其对应的问题配对，而测试划分大约涵盖 8,000 个实例。

- ScienceQA（SQA）（Lu et al., 2022）评估模型在三个学科（即自然科学、语言科学和社会科学）、26 个主题、127 个类别和 379 个技能方面的具有挑战性的多模态选择题。每个问题都附有与相关课程链接的解释。测试集包含 4,241 个示例。

- Visual Spatial Reasoning（VSR）（Liu et al., 2023a）包含来自 COCO 的图像，每张图像配有一个描述两个概念及其空间关系的标题。模型必须根据图像判断给定的标题是真还是假。MLLM 通常在零样本测试划分的 616 个样本上进行评估。

- IconQA（Lu et al., 2021）在三种类型的问题上测试视觉-语言模型的视觉推理能力：多图像选择、多文本选择和填空。该数据集源自数学教科书中的现实问题，侧重于抽象图像（即图标）。共有 107,439 个问题，其中 20% 构成测试划分。

- TextVQA（VQAT）（Singh et al., 2019）是一个基于 Open Images（Kuznetsova et al., 2020）图像的数据集，挑战视觉-语言模型的 OCR 能力。测试集包含 5,734 个示例。

- OCR-VQA（Mishra et al., 2019）通过解释图像中的文本，在视觉问答中引入了一个新任务，涉及 207,572 张书籍封面图像，附带超过 1M 个问答对。

全面描述视觉输入是 MLLM 所需的另一项重要技能。为此，通常使用各种图像描述数据集进行评估。关于评估指标，CIDEr 分数（Vedantam et al., 2015）作为该任务的参考指标，用于比较生成的图像描述与真实描述（ground-truth captions）。

- COCO（Lin et al., 2014）包含超过 120k 张图像，每张图像配有五个人工生成的描述。对于描述生成任务，通常采用 Karpathy and Fei-Fei（2015）定义的划分，训练、验证和测试分别使用 113k、5k 和 5k 张图像。

- Flickr30k（Young et al., 2014）包含 31,783 张图像，描绘了多样化的日常活动、事件和场景。补充这些图像的是通过众包技术获得的 158,915 条描述。

- nocaps（Agrawal et al., 2019）代表了一个新颖物体描述生成（novel object captioning）的基准，拥有近 400 个新颖物体类别的广泛集合（相比 COCO 数据集）。验证集和测试集分别包含来自 Open Images（Kuznetsova et al., 2020）的约 4.5k 和 10.6k 张图像。每张图像配有 11 个人工生成的描述。验证集和测试集进一步分为域内（in-domain）、近域（near-domain）和域外（out-of-domain），其中域外子集中的图像包含 COCO 中从未出现过的物体类别。

- TextCaps（Sidorov et al., 2020）包含 28k 张图像对应的 145k 条描述。目标是识别和理解图像中的文本，并提供描述整个视觉内容的有效描述。这要求模型具备 OCR 能力和图像描述技能。

### C.2 MLLM 专用基准（MLLM-Specific Benchmarks）

全面评估 MLLM 具有挑战性，仍然是一个开放的前沿领域。虽然在标准数据集上进行评估是有效的选择，但许多专为 MLLM 设计的基准已被最近提出。它们需要非常强的感知和认知技能才能取得成功，并且通常查询深度领域特定知识。为了方便评估，许多工作提出利用最先进的特有模型（例如 ChatGPT（OpenAI, 2022）、GPT-4（Achiam et al., 2023））来自动评判候选答案。在表 4 中，我们报告了部分模型在这些新基准子集上的性能。

- POPE（Li et al., 2023k）是评估 MLLM 中物体幻觉（object hallucination）挑战的宝贵基准。该数据集包含几个不同的子集，即随机（random）、流行（popular）和对抗（adversarial），它们利用多种采样方法生成。总体而言，它是一个二分类查询数据集，包含 8,910 个条目，便于对物体幻觉现象进行综合研究。

- MME（Yin et al., 2023）是一个综合基准，用于定量评估感知和认知能力，而不受指令模板的影响。它由 14 个子任务组成，总共包含 2,344 个问答对。

- MMB（Liu et al., 2023f）探索了感知和推理能力，涵盖八个维度，包括室外场景、坐标、.count、实例属性、比较、全局场景、画内文字和逻辑推理。它在八个子类别中使用混合问题类型，总共包含约 1,500 个样本。

- SEED-Bench（Li et al., 2023j）由 19k 个多模态选择题组成，涵盖 8 种评估维度，包括图像和视频理解。它在 2D 和 3D 之间提供了对空间理解的细粒度评估，总共包含约 14k 个样本。

- LLaVA-Bench（W）（Liu et al., 2023h）通过提示将开放式问题分为两部分：场景描述（$Q_1$）和详细对话（$Q_2$），以评估模型关于视觉输入和对话生成的能力。作者使用 GPT-4 根据参考解决方案和给定的评分标准来评分答案。

- MM-Vet（Yu et al., 2023b）评估 MLLM 在 16 个任务上的表现，这些任务涵盖六项基础视觉-语言能力，如识别、OCR、知识、语言生成、空间感知和数学。该基准包含 200 张图像和 218 个问题。评估分数通过少样本提示（few-shot prompting）由 GPT-4 获得。

- MathVista（MathV）（Lu et al., 2023b）探究 MLLM 在视觉问答中的数学推理技能。共有 6,141 个问题，但只有 5,141 个用于评估。在计算准确率之前，作者建议使用 GPT-4 等 LLM 来解析答案。

- MMMU（Yue et al., 2023）是一个针对多模态模型领域特定知识的具有挑战性的基准。它包含 10.5k 个测试样本，来自大学教科书或在线课程，涵盖六个主要学科。问题可能包含图像与文本交错的内容。使用精确匹配和词汇匹配来评估多选题答案的正确性。

# C.3 视觉定位评估（Visual Grounding Evaluation）

多模态大语言模型（MLLM）的视觉定位能力评估涵盖多种标准定位任务，包括区域描述（region captioning）、指代表达生成（Referring Expression Generation, REG）和区域级视觉问答（region-level question answering），以及定位任务如指代表达理解（Referring Expression Comprehension, REC）、指代表达分割（Referring Expression Segmentation, RES）和附注描述（grounded captioning）。在评估指标方面，对于指代表达理解（REC），通过假设预测与真实标注的交并比（Intersection over Union, IoU）高于 0.5 的预测为正确来计算准确率（Acc@0.5）。对于指代表达分割，采用累积交并比（cumulative IoU, cIoU）作为评估指标；而对于区域描述，通常使用 METEOR（Banerjee and Lavie, 2005）和 CIDEr（Vedantam et al., 2015）作为评估指标。

然而，很少有方法引入自己的基准来评估更真实场景下的性能，这些场景可能涉及多轮附注对话（grounded conversations）。指代表达理解（REC）、指代表达分割（RES）和区域描述任务上的定量结果分别报告在表 5、表 6 和表 7 中。

RefCOCO 和 RefCOCO+（Mao et al., 2016）是基于 COCO 数据集图像的指代表达集合。它们通过 ReferItGame（Kazemzadeh et al., 2014）收集，这是一种双人游戏：第一位玩家检视一幅包含分割目标物体的图像，并形成描述该物体的自然语言指代表达；第二位玩家只能看到图像和指代表达，选择对应的物体。如果玩家回答正确则交换角色，否则获得新的物体和图像进行描述。RefCOCO 数据集对自然语言没有约束，由 19,994 张图像中的 50,000 个物体构成 142,209 条指代表达。

相比之下，在 RefCOCO+ 中，玩家不允许在指代表达中使用方位词，该数据集包含 19,992 张图像中的 49,856 个物体的 141,564 条指代表达。两个数据集的评估均在验证集、testA 和 testB 分集对应的 1,500、750 和 750 张图像上进行。

RefCOCOg（Yu et al., 2016）由一组标注员收集，他们为图像中的物体编写自然语言指代表达，另一组标注员则选择与给定指代表达对应的物体。当选择的物体正确时，相应的指代表达被插入数据集。该数据集包含 26,711 张图像中的 54,822 个物体的 85,474 条指代表达。评估在对应验证集和测试集分集的 1,300 和 2,600 张图像上进行。

Visual Genome（Krishna et al., 2017）将结构化图像概念与语言关联，包含 108,077 张图像及其所有物体的详细描述，提供 5.4M 条区域描述和 1.7M 对视觉问答（visual question-answering）数据。该数据集通常用于区域级描述和问答任务。

Visual7W（Zhu et al., 2016）是一个视觉问答数据集，通过物体级定位将文本描述与图像区域结合起来。它包含 47k 张 COCO 图像上的 328k 对问答数据，以及 1.3M 个人工生成的多项选择题和超过 560k 条来自 36,579 个类别的物体定位标注。

GRIT（Peng et al., 2023）是一个大规模附注图像-文本对数据集（即与图像区域关联的名词短语或指代表达），基于 COYO-700M 和 LAION-2B 的子集构建。其构建流程包括两个步骤：(i) 从标题中提取名词块，并使用开放词汇检测器（如 GLIP）将其定位到边界框；(ii) 利用原始标题中的依存关系将名词块扩展为指代表达。生成的数据集包含 91M 张图像、115M 个文本片段和 137M 个关联边界框。

ReasonSeg（Lai et al., 2023）是为推理分割任务引入的一个基准，该任务涉及为复杂和隐式查询文本提供分割掩码。图像来自 OpenImages（Kuznetsova et al., 2020）和 ScanNetv2（Dai et al., 2017），并使用文本指令和对应的分割掩码进行标注。生成的数据集包含 1,218 对图像-指令数据。评估指标与指代表达分割标准基准相同。两个扩展变体 ReasonDet（Wei et al., 2023）和 ReasonSeg-Inst（Yang et al., 2023b）分别针对推理检测和推理实例分割任务引入。

附注一切数据集（Grounded-anything Dataset, GranD）（Rasheed et al., 2023）是为附注对话生成（Grounded Conversation Generation, GCG）任务设计的数据集，旨在构建图像级标题，其中包含与图像中分割掩码关联的短语。该数据集通过四阶段自动标注流程构建：(i) 使用对应的语义标签、分割掩码、属性和深度信息进行物体定位；(ii) 提取检测到的物体之间的关系；(iii) 组合之前收集的关系以生成密集标题；(iv) 用上下文信息丰富标题。它包含 1,100 万张 SAM（Kirillov et al., 2023）图像的标注。

另一个数据集 GranDf 是为进一步微调和评估 GCG 任务而引入的。它通过扩展 Flickr30k（Young et al., 2014）、RefCOCOg 和 PSG（Yang et al., 2022）并利用 GPT-4 以及手动标注一组样本来收集。它包含 214k 对附注图像-文本数据，其中 2.5k 验证样本和 5k 测试样本。评估指标包括描述的 METEOR 和 CIDEr、定位的类别无关掩码平均精度（class-agnostic mask AP）、分割的交并比（IoU）以及附注描述的掩码召回率（mask recall）。

附注基准（Grounded-Bench）（Zhang et al., 2023d）是为评估 MLLM 进行附注视觉对话的能力而引入的基准。它基于 LLaVA-Bench（Liu et al., 2023h），包含使用 GPT-4 生成的对话数据和来自 COCO 的实例标注。它通过从 COCO 中标注的 1,000 张图像和 7,000 个实体进行扩展。

# C.4 图像生成与编辑评估（Image Generation and Editing Evaluation）

为了评估图像生成和编辑结果，通常使用一组不同的基准。在评估指标方面，Fréchet 初始距离（Fréchet Inception Distance, FID）（Heusel et al., 2017）是评估生成图像的参考指标。它定量评估了合成生成图像的分布与真实图像分布之间的一致性。FID 分数的降低表示生成图像在视觉质量和真实感方面的增强。

其他指标衡量生成图像与输入提示及对应真实图像的连贯性。具体而言，CLIP-I 和 DINO 分数通过利用 CLIP（Radford et al., 2021）和 DINO（Caron et al., 2021）作为视觉主干网络，计算生成图像与真实图像之间的余弦相似度。而 CLIP-T 则通过使用 CLIP 对图像和文本提示进行编码，计算输入标题与生成图像之间的余弦相似度来测量图像-文本对齐。

COCO 数据集用于评估文本到图像生成。评估使用原始验证集（包含 41k 个样本）或从同一集合中随机选择的 30k 样本子集进行。表 8 报告了具有图像生成能力的 MLLM 在该数据集上的结果。

VIST（Huang et al., 2016）专门为交错图像-文本生成任务设计。它包含 34k 个训练样本和 5k 个评估样本。每个样本是由 5 张图像和 5 段文本叙述组成的序列，共同构成一个连贯的故事。

MagicBrush（Zhang et al., 2023e）是图像编辑领域的一个基准，包含 10,000 个人工标注的三元组，每个三元组由源图像、编辑指令和对应的目标图像组成。该基准上的性能报告在表 9 中。

DreamBench（Ruiz et al., 2023）是一个评估模型在主体驱动生成方面生成能力的基准。具体而言，它包含 30 个主体，每个主体用 4 到 6 张图像展示，以及 25 个模板提示，用于修改和装饰给定主体。该基准上的结果如表 10 所示。

## 3D 理解（3D Understanding）

MLLM 也应用于 3D 数据，用于解决 3D 视觉问答（3D VQA）、3D 对话和 3D 密集描述等复杂任务。不同
3D 场景编码与对齐。不同于利用 2D 预训练嵌入（embedding）的标准视觉编码方式，在 3D 数据语境下，研究者设计了合适的策略将 3D 特征投影（project）到大语言模型（Large Language Model, LLM）空间。在 3D-LLM（Hong et al., 2023）中，3D 场景以不同视角渲染，3D 特征由连接微调 BLIP-2 模型的 EVA-CLIP 骨干网络构建。类似地，Xu et al.（2023b）使用预训练的 PointBERT（Yu et al., 2022）作为 3D 编码器，并进行两阶段训练：最初通过 MLP 投影层对齐输入特征，随后进行模型指令微调（instruction tuning）阶段。不同之处在于，Point-Bind（Guo et al., 2023）将 3D 点云与 ImageBind（Girdhar et al., 2023）对齐，并利用 I2P-MAE（Zhang et al., 2023f）作为 3D 编码器。这种对齐使得引入新任务成为可能，如任意到 3D 生成（any-to-3D generation）和 3D 嵌入空间算术（3D embedding-space arithmetic）。最近，LL3DA（Chen et al., 2023g）引入了 Interactor3D 模块，该模块由冻结的 3D 场景编码器、视觉提示编码器（visual prompt encoder）和 Q-Former 组成，用于解决 3D 描述生成（3D captioning）和视觉问答（Visual Question Answering, VQA）。

任意模态模型（Any-Modality Models）。多项研究专注于通过纳入视频、3D 和音频等多种模态来扩展多模态大语言模型（Multimodal Large Language Model, MLLM）的推理能力。一条研究方向调查了使用专用通路（dedicated pathway）将不同模态输入到 LLM 的方法。UniVAL（Shukor et al., 2023）通过共享表示空间将各模态编码器的特征映射为 LLM 的表示，而 EncLLM（Diao et al., 2023）和 AnyMAL（Moon et al., 2023）则分别为每种模态使用专用的 Q-Former 或 Perceiver 适配器。为最大化特征兼容性，AnyMAL（Moon et al., 2023）使用了一种已经与文本嵌入空间对齐的编码器，涵盖各模态（包括 IMU 信号），以及一个专用适配器（adapter）：视觉模态采用 Perceiver，其他模态采用线性层。另一方面，PandaGPT（Su et al., 2023）和 NExT-GPT（Wu et al., 2023c）则利用单一的冻结多模态编码器（即 ImageBind）从不同模态提取特征。OneLLM（Han et al., 2024）通过混合多个图像投影模块和一个模态路由器（modality router）构建了统一通用编码器和通用投影模块，以将输入信号与语言对齐。CAT（Ye et al., 2024）增加了一个线索聚合器（clue aggregator），用于聚合感知问题的视听隐藏特征并生成提供给 LLM 的线索标记（clue token）。

除处理输入 LLM 的不同模态外，一些研究还探讨了生成不同模态输出的方法。例如，NExT-GPT（Wu et al., 2023c）在 LLM 中引入信号标记（signal token），用于指示基于扩散（diffusion-based）的解码器是否需要针对特定模态被激活。此外，这些信号标记被提供给基于 Transformer 的输出投影器（output projector），以调节生成过程。类似地，M2UGen（Hussain et al., 2023）通过使用对应信号标记的 LLM 输出以及来自音乐编码器的单模态音乐特征来处理音乐模态。LLMBind（Zhu et al., 2024）通过将文本包裹在特殊标记中来指示图像、视频或音频的生成条件文本，随后将该文本提供给对应模态的基于扩散的专用解码器。Unified-IO2（Lu et al., 2023a）使用 VQ-GAN 解码器解码图像和音频模态的输出离散标记，并能够为输入图像生成法线、深度和分割掩码。AnyGPT（Zhan et al., 2024）将输入和输出中的所有连续非文本模态解释为离散标记，分别使用多模态分词器（multimodal tokenizer）和反分词器（de-tokenizer）。为支持 3D 模态，LAMM（Yin et al., 2023d）引入了一个新的指令微调数据集和基准，包含图像-文本和点云-文本指令-响应对，涵盖广泛的 2D 和 3D 任务。

交互式与组合式系统（Interactive and Compositional Systems）。另一趋势是构建能够组合多个工具（即现有的纯视觉或视觉与语言模型）的系统，通常通过 ChatGPT 或其他 LLM 实现。特别地，这些方法旨在让用户与 LLM 交互，由 LLM 负责选择有用工具以执行复杂任务。在此背景下，一些解决方案研究了如何提示（prompt）ChatGPT（Wu et al., 2023a; Yang et al., 2023c）调用视觉基础模型。相比之下，GPT4Tools（Yang et al., 2023a）使用开源 LLM（如 LLaMA 和 OPT），通过 PEFT 技术微调以使用工具执行广泛的视觉任务。不同地，Liu et al.（2023l）通过在图像或视频上引入基于鼠标的指向指令，实现了更复杂的用户-聊天机器人交互。尽管在所有这些方法中 LLM 并不直接处理视觉输入（而是由其他外部工具处理），但在 LLaVA-Plus（Liu et al., 2023h）中，查询图像直接被输入到 MLLM（即 LLaVA），因此在根据用户需求选择和调用最有用工具时直接参与。这一目标的实现还得益于引入了一个新的"遵循指令使用工具"数据集，该数据集用于微调 MLLM。
