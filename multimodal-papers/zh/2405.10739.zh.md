多模态大语言模型（Multimodal Large Language Model, MLLM）在视觉问答（visual question answering）和图像描述（image captioning）等多种任务中表现出色。然而，MLLM 在需要复杂识别的任务（如人群计数和小字符光学字符识别（Optical Character Recognition, OCR））中面临相当大的挑战。解决这些挑战的直接方法是提高图像分辨率，即增加视觉 token 的数量。然而，这种策略给 MLLM 带来了巨大的计算负担，主要是因为 Transformer 架构中计算成本随输入 token 数量的二次增长。受此挑战驱动，旨在减少大量 token 所导致的过高计算预算的视觉 token 压缩技术，已成为高效 MLLM 的核心研究方向。我们将通过几种关键技术来探讨这一主题，包括多视角输入（multi-view input）、token 处理（token processing）、多尺度信息融合（multi-scale information fusion）、视觉专家代理（vision expert agents）以及视频特定方法（video-specific methods）。

**多视角输入（Multi-view Input）** 直接使用高分辨率视觉编码器进行细粒度感知的成本极高，且不符合实际使用需求。因此，为了在利用低分辨率视觉编码器的同时使 MLLM 感知到细节信息，一种常见方法是输入多视角高分辨率（High-Resolution, HR）图像，即：全局视角（global view）——通过缩放获得的低分辨率图像，以及局部视角（local view）——由分割产生的图像块（image patch）。例如，LLaVA-UHD [35] 提出了一种图像模块化策略，将原生分辨率图像划分为更小的可变尺寸切片，以实现高效且可扩展的编码。此外，InternLM-XComposer2-4KHD [90] 引入了一种自动布局排列的动态分辨率调整策略，该策略不仅保持了图像原始宽高比，还自适应地改变 patch 布局和数量，从而提高了图像信息提取的效率。通过对不同分辨率图像实施自适应输入策略，可以在感知能力和效率之间取得平衡。

**Token 处理技术（Token Processing Techniques）** 旨在处理冗长的视觉 token 序列，这些技术对高效 MLLM 至关重要，因为它们解决了保留细粒度细节和降低计算复杂性的双重挑战。LLaVA-UHD [35] 提出了一种新颖的方法来处理高分辨率图像相关的计算负担。它提出了两个关键组件：(1) 一个压缩模块，进一步压缩来自视觉编码器的图像 token，显著降低计算负载；(2) 一个空间架构（spatial schema），用于为 LLM 组织切片 token。值得注意的是，LLaVA-UHD 通过支持 6 倍更大分辨率的图像，且推理计算仅占先前模型的 94%，展示了其效率。此外，该模型可以在学术环境中高效训练，在 8 块 A100 GPU 上仅需 23 小时即可完成训练。LLaVA-PruMerge [41] 和 MADTP [42] 提出了一种自适应视觉 token 缩减方法，在保留相当模型性能的同时显著减少了视觉 token 的数量。TinyChart [37] 和 TextHawk [36] 专注于面向文档的任务，前者采用视觉 Token 合并（Vision Token Merging）模块，后者引入了重采样与重排列（ReSampling and ReArrangement）模块。这些模块可以增强细粒度视觉感知和信息压缩能力。

**多尺度信息融合（Multi-Scale Information Fusion）** 利用多尺度图像信息对于视觉特征提取至关重要。这种方法使模型能够同时捕获较小尺度中存在的细粒度细节和较大尺度中可用的更广泛上下文。Mini-Gemini [26] 包含双编码器，一个用于高分辨率图像，另一个用于低分辨率视觉嵌入。它提出了一种 Patch 信息挖掘（Patch Info Mining）方法，利用低分辨率视觉嵌入作为查询，通过交叉注意力从高分辨率候选者中检索相关的视觉线索。尺度上的尺度（Scaling on Scales, S²）[40] 证明了多尺度小模型具有与大模型相当的学习能力，并且使用 S² 预训练小模型可以在 MLLM 基准测试中匹配甚至超过大模型的优势，同时计算效率更高。将大图像分割成小子图像后，S²-wrapper 处理各个子图像而不是使用窗口注意力（window attention），这允许使用不支持窗口注意力的预训练模型，并避免了从头开始训练额外参数。然后将大特征图插值到常规大小，确保视觉 token 数量保持可接受。

**视觉专家代理（Vision Expert Agents）** 大多数 MLLM 由于其非无损图像标记化（non-lossless image tokenization），难以完全捕获文本和物体的复杂细节。利用视觉专家代理是解决单一视觉编码器在内容丰富的细节上泛化能力有限这一问题的方案。P2G [38] 使用专家代理进行实时定位（real-time grounding），通过多模态提示实现高效且有目的推理。这一创新框架促进了对富含自然视觉和文本的高分辨率场景中推理的即插即用地面的实现。它通过利用代理来增强文本和视觉的地面感知和感知能力来实现这一点，如 OCR 代理（文本）或地面代理（图像）。MoVA [43] 通过引入专家路由策略来解决单个视觉编码器在不同内容上泛化能力下降的问题。这种方法能够灵活有效地利用来自多个任务特定视觉专家的表示，从而增强泛化能力。

**视频特定方法（Video-Specific Methods）** 视频理解同样需要处理大量帧，这在 LLM 的上下文窗口中可能构成巨大的计算挑战。Elysium [92] 在性能和视觉 token 消耗之间提供了一种权衡，其中 T-Selector 作为一种视觉 token 压缩网络被引入，使 LLM 能够在减少视觉 token 使用的同时区分单个帧。VideoLLaVA [44] 基于 LanguageBind [93]，将视觉表示统一到语言特征空间中，以推动基础 LLM 向统一的语言-视觉 LLM 发展，且不带来巨大的计算负担。

### 4.5 高效结构（Efficient Structures）

高效结构主要探索三个方向：混合专家（Mixture-of-Experts）、Mamba 和推理加速（Inference Acceleration）。

**混合专家（Mixture of Experts）** MoE 通过在保持激活参数不变的情况下调整模型参数总数来增强模型容量，因此不会显著损害推理速度。MoE-LLaVA [25] 提出了一种基于 MoE 的稀疏 MLLM 框架，在不损害计算效率的情况下有效增加了参数数量。此外，它引入了 MoE-Tuning，一种旨在将 MoE [89] 适配到 MLLM 并防止由稀疏性引起的模型退化的三阶段训练策略。MM1 [30] 设计了两种 MoE 模型变体。第一种是 3B-MoE 模型，采用 64 个专家，每两层将一层密集层替换为稀疏层。第二种是 7B-MoE 模型，使用 32 个专家，每四层将一层密集层替换为稀疏层。

**Mamba** Cobra [13] 将高效的 Mamba [77] 语言模型纳入视觉模态，并探索了不同的模态融合方案，以开发有效的多模态 Mamba。实验表明，它不仅实现了与最先进高效方法相当的性能，而且由于其线性序列建模，速度更快。它在克服封闭集挑战性预测基准中的视觉错觉和空间关系判断方面也表现出色，并且仅使用 43% 的参数就达到了与 LLaVA 相当的性能。VL-Mamba [18] 将基于 Transformer 的主干语言模型替换为预训练的 Mamba 语言模型。它探索了如何有效地实现用于多模态学习的 2D 视觉选择性扫描（2D vision selective scan）机制，以及不同视觉编码器和预训练 Mamba 语言模型变体的组合。

**推理加速（Inference Acceleration）** SPD [45] 提出了一种仅使用语言模型（language-only model）的投机解码（speculative decoding）方法以提高推理效率。通过使用仅语言模型作为投机解码的草稿模型（draft model），绕过图像 token 及其相关处理组件的需求。FastV [46] 发现大多数图像 token 在第二个解码器层之后获得的注意力效率低下，并通过在推理阶段消除冗余视觉 token 来实现计算减少，且不牺牲性能。VTW [47] 断言视觉 token 在 MLLM 的深层中不是必需的。它在特定层策略性地移除所有视觉 token，仅允许文本 token 参与后续层。VTW 的这种方法可以在各种多模态任务中将计算开销减少 40% 以上，且不损害性能。

### 3.1 紧凑架构（Compact Architecture）

紧凑架构是指在高性能保持下游任务性能的同时，设计轻量级且高效的模型。它包含各种策略和方法，在不损害性能的前提下减少模型大小、计算复杂度和内存占用。这些策略可以 broadly 分为三类：1）架构设计方法（Architecture Design Methods），2）架构搜索方法（Architecture Search Methods），以及 3）注意力机制优化方法（Optimization of Attention Mechanisms Methods）。

**架构设计方法** 涉及创建新架构 [133] 或调整现有架构 [134] 以实现紧凑性，同时不牺牲性能。例如，Reformer [96] 在注意力机制中引入了局部敏感哈希（locality-sensitive hashing）以降低复杂度，同时使用可逆残差层（reversible residual layers）更高效地存储激活值。此外，EfficientFormer [97] 分析了基于 ViT 的模型架构和算子，引入了一种维度一致的纯 Transformer 范式，并使用延迟驱动的剪枝（latency-driven slimming）来生成优化模型。此外，EfficientFormerV2 [98] 提出了一种低延迟、高参数效率的超网（supernet）。

**架构搜索方法** 涉及使用神经架构搜索算法（neural architecture search algorithms）[113] 来探索和发现针对特定任务或约束定制的紧凑架构。例如，Autoformer [99] 交织了层内的权重，实现了数千个子网的充分训练。NASViT [100] 引入了梯度投影算法（gradient projection algorithm）、可切换层缩放（switchable layer scaling）和简化数据增强，提高了收敛性和性能。此外，TF-TAS [101] 研究了免训练架构搜索方法，提出了一种高效的方案。UniNet [102] 引入了上下文感知的下采样模块（context-aware down-sampling modules），改善了 Transformer 和 MLP 算子的信息容纳能力。

**注意力机制优化方法** 侧重于通过引入自适应注意力、学习稀疏注意力模式以及动态调整注意力机制来降低计算复杂度。Fayyaz 等人 [135] 通过评分和自适应采样重要 token 来实现自适应注意力。PatchMerger [103] 提取区域 token 之间的全局信息，并通过自注意力（self-attention）用区域 token 之间的信息交换局部自注意力。DynamicViT [104] 提出了一种注意力掩蔽策略，通过阻断与其他 token 的交互以可微分的方式修剪 token。此外，SepViT [105] 使用深度可分离自注意力（depthwise separable self-attention）在窗口内部和跨窗口进行局部-全局信息交互。这些方法共同优化了注意力机制，提高了计算效率和性能。

### 3.2 剪枝（Pruning）

剪枝涉及从视觉 Transformer 模型中移除较不重要的权重，通常分为非结构化剪枝、结构化剪枝和混合剪枝技术。

**非结构化剪枝（Unstructured Pruning）** 侧重于在不考虑模型内部结构排列的情况下消除单个权重。Rao 等人 [104] 引入了一种动态 token 稀疏化框架，用于基于输入对冗余 token 进行渐进式自适应剪枝，集成了一个轻量级预测模块来估计 token 重要性得分，并采用注意力掩蔽策略来区分 token 交互并以端到端方式优化预测模块。CAP [106] 提出了一种新颖的、具有理论依据的剪枝器（pruner），能够在剪枝过程中准确高效地处理复杂的权重相关性，并配合一种有效的微调过程进行压缩后恢复。CaiT [107] 引入了非对称 token 合并（asymmetric token merging）来高效地整合相邻 token，同时保留空间结构，配合一致的动态通道剪枝（consistent dynamic channel pruning）对 Vision Transformers 中不重要的通道进行统一剪枝，增强模型压缩能力。

**结构化剪枝（Structured Pruning）** 旨在根据预定义标准移除结构组件，如注意力头或层。例如，WDPruning [108] 使用二元掩码（binary mask）根据参数大小来识别不重要的参数。此外，Yu 等人 [136] 提出了一个集成剪枝的统一框架来生成紧凑的 Transformer。X-Pruner [109] 利用端到端学习的可解释性感知掩码（explainability-aware mask）来衡量每个单元对预测目标类的贡献，并自适应地搜索逐层阈值（layer-wise thresholds），在确定剪枝率的同时保留最具信息量的单元。此外，VTP [110] 通过整合控制系数来降低嵌入维度，同时移除系数可忽略的神经元。Tang 等人 [111] 通过首先识别最后一层中的有效 patch，然后利用它们指导先前层的选择过程来消除冗余 patch，其中对最终输出特征影响最小的 patch 随后被丢弃。

**混合剪枝（Hybrid Pruning）** 如 [137] 研究了非结构化和结构化稀疏性，引入了一种用于注意力头移除的一阶重要性近似方法（first-order importance approximation approach）。SPViT [112] 开发了一种基于动态注意力的多头 token 选择器（multi-head token selector），用于自适应实例级 token 选择，以及一种软剪枝技术（soft pruning technique），将信息量较少的 token 合并成包 token 而不是丢弃它们。ViT-Slim [113] 使用可学习的统一稀疏度约束（learnable and unified sparsity constraint）和预定义因子，在各种维度的连续搜索空间中表示全局重要性。

### 3.3 知识蒸馏（Knowledge Distillation）

知识蒸馏是一种技术，其中较小的模型从较大、更复杂的模型中学习以复制其性能，从而在保持预测准确性的同时实现高效部署 [139]。视觉 Transformer（ViT）的知识蒸馏（KD）技术可以分为两大类：1）同构知识蒸馏（homomorphic KDs）和 2）异构知识蒸馏（heteromorphic KDs）。

**同构知识蒸馏** 可进一步分为 logit 级 [114, 115]、patch 级 [117]、模块级 [116] 和特征级 KD [118]。对于 logit 级方法，在 DeiT [114] 中，一个蒸馏 token 被纳入自注意力模块，以模拟教师模型推断的类标签，促进学生注意力与层之间的交互，从而在后向传播（back-propagation）期间学习硬标签（hard labels）。TinyViT [115] 在预训练期间应用蒸馏，其中大型教师模型的 logit 被预存储在硬件中，从而在将知识转移到缩小的学生 Transformer 时实现内存和计算效率。像 DeiT-Tiny [117] 这样的 patch 级技术训练一个小型学生模型，使其在 patch 级结构上匹配预训练的教师模型，然后使用分解流形匹配损失（decomposed manifold matching loss）进行优化以降低计算成本。模块级方法涉及将教师模块从预训练的统一模型中分离出来，并将学生模块从模块化模型中分离出来。在 m2mKD [116] 中，这些模块与共享的元模型（meta-model）相结合，允许学生模块模拟教师模块的行为。如 MiniViT [118] 所示的特征级 KD 方法，结合了连续 Transformer 块的权重。这涉及跨层共享权重，同时引入变换以增强多样性。此外，还利用自注意力上的权重蒸馏（weight distillation），将知识从大规模 ViT 模型转移到具有多重权重的紧凑模型。

**异构知识蒸馏** 涉及在架构不同的模型之间传递知识。例如，DearKD [119] 提出了一种新颖的两阶段框架 DearKD，偏离了 ViT 架构的传统方法。在第一阶段，他们使用 vanilla KD 策略将 CNN 特征转移到 ViT 学生模型，代表了异构迁移。在后续阶段，如果真实样本有限，他们引入了一种边界保持的类内散度损失（boundary-preserving intra-divergence loss）来增强该过程。类似地，CiT [120] 提出了一种异构 KD 策略，知识从不同的模型（如 CNN 和神经网络逆卷积（involution neural network））转移，从而提高了 ViT 学生模型的性能。

### 3.4 量化（Quantization）

ViT 量化是降低 ViT 模型中数值表示精度的过程，通常从浮点数运算过渡到定点运算 [140]。这种精度降低旨在减少内存使用、计算复杂度和能源消耗，同时将模型精度保持在可接受的水平。当前研究主要可分为训练后量化（post-training quantization）、量化感知训练（quantization-aware training）和硬件感知量化（hardware-aware quantization）。

**训练后量化（Post-Training Quantization, PTQ）** 通过将训练好的 ViT 模型参数从高精度浮点数转换为低精度定点数（如 8 位整数）来压缩模型。例如，Liu 等人 [141] 引入了一种排序损失方法（ranking loss method），用于识别权重和输入的最优低位量化区间，确保注意力机制的功能。他们还进行了分析，以了解不同层中的量化损失与特征多样性之间的关系，探索了利用每个注意力图和输出特征的核范数（nuclear norm）的混合精度量化方法。此外，PTQ4ViT [121] 引入了双均匀量化方法（twin uniform quantization method），以最小化 softmax 和 GELU 函数之后的激活值的量化误差，并引入了一种 Hessian 引导度量（Hessian-guided metric）来增强校准精度。APQ-ViT [122] 提出了一种统一的底部消除块校准（Bottom-elimination Blockwise Calibration）方案来优化校准度量，优先处理关键量化误差，并设计了用于 Softmax 的马太效应保持量化（Matthew-effect Preserving Quantization）以保持幂律特征和注意力机制的功能。NoisyQuant [123] 提出向量化值添加固定的均匀噪声偏置（Uniform noisy bias），在某些条件下量化误差显著减少。该技术成功地修改了重尾激活分布以适应给定的量化器。

**量化感知训练（Quantization-Aware Training, QAT）** 将量化整合到训练周期中。这种集成在缩小到极低比特精度（如 4 位或更低）时特别有利，此时 PTQ 会遭遇显著的性能损失。例如，Quantformer [124] 利用熵信息（entropy information）来保持自注意力排名的-致性，并引入了一种可微分搜索机制（differentiable search mechanism）以最优方式分组 patch 特征维度，减少了舍入和剪裁不准确。Q-ViT [126] 引入了一个蒸馏 token 和信息校正模块（Information Rectification Module, IRM）来抵消量化注意力模块中改变的分布。TerViT [127] 和 Bit-shrinking [125] 在调节锐度（sharpness）的同时逐步降低模型位宽，以在整个量化过程中保持准确性。PackQViT [129] 减轻了量化过程中的异常值影响。BiViT [128] 引入了 Softmax 感知二值化（Softmax-aware Binarization）来调整二值化过程，最小化二值化 softmax 注意力值的误差。Xiao 等人 [142] 整合了一种梯度正则化方案（gradient regularization scheme）来抑制二值化训练过程中的权重振荡，并引入了一种激活偏移模块（activation shift module）以减少激活中的信息失真。此外，BinaryViT [130] 将 CNN 的基本架构元素整合到纯 ViT 框架中，增强了其能力。

**硬件感知量化（Hardware-Aware Quantization）** 针对特定硬件平台（如 GPU [131]、FPGA [132]）优化神经网络模型的量化过程。它调整精度级别和量化策略，以在推理过程中最大化性能和能源效率。例如，Yu 等人 [131] 提出了一种利用 GPU 友好的 2:4 细粒度结构化稀疏和量化的压缩方案。他们使用 2:4 结构化剪枝将密集模型剪枝为稀疏模型，利用 GPU 加速。然后，通过稀疏蒸馏感知的量化感知训练（sparse-distillation-aware quantization-aware training）将稀疏模型量化为定点表示，利用 GPU 加速。在整个过程中，他们采用混合策略知识蒸馏，支持监督学习和无监督学习风格。Auto-ViT-Acc [132] 提出了一个旨在将 ViT 架构量化以在 FPGA 驱动的设备上运行推理的框架。他们特别将先前研究中的量化函数应用于注意力块内的前馈神经网络（FNN）模块，旨在优化 FPGA 资源利用率并加速推理。

### 4.1 注意力（Attention）

在标准自注意力机制中，时间复杂度为 O(n²)，其中 n 为序列长度。这种二次复杂度源于所有输入 token 之间的两两交互，这在处理 LLM 中的长序列时可能导致可扩展性问题。为了解决这个问题，研究人员开发了加速注意力机制和降低时间复杂度的技术，例如基于共享的注意力（sharing-based attention）、特征信息缩减（feature information reduction）、核化或低秩（kernelization or low-rank）、固定和可学习模式策略（fixed and learnable pattern strategies），以及硬件辅助注意力（hardware-assisted attention）。

**基于共享的注意力（Sharing-based Attention）** 基于共享的注意力旨在推理过程中通过跨多个键值头（Key-Value heads）共享计算资源来加速注意力计算。例如，Llama-2 [91] 采用了一种称为分组查询注意力（Grouped-Query Attention, GQA）[143] 的技术来优化自回归解码过程中的内存带宽。GQA 是一种基于共享的注意力技术，旨在在性能与效率之间取得平衡，定位于多头注意力和多查询注意力 [144] 机制之间。在多头注意力中，每个头使用不同的线性变换参数集来处理查询、键和值。相反，多查询注意力在所有查询之间共享一组键值头。GQA 将所有查询头划分为若干组，每组查询头共享一个公共的键值头，从而在有效性和计算成本之间建立了严格的平衡。

**特征信息缩减（Feature Information Reduction）** 特征信息缩减，如 Funnel-Transformer [145] 和 Set Transformer [146] 等模型所证明的那样，解决了注意力机制中对计算效率的关键需求，具体是通过在保留数据中嵌入的基本信息的同时降低输入特征的维度或数量。这种策略背后的关键动机源于在 Transformer 模型的所有层中维护全长隐藏表示的潜在冗余性。Funnel-Transformer [145] 通过逐步降低自注意力中隐藏表示的序列大小来解决这个问题
## 4.1.3 近似注意力

近似注意力（Approximate Attention）使模型在处理长文本时能够高效聚焦于任务相关信息。近似注意力中的两个核心概念是核化（Kernelization）和低秩（Low-Rank）。核化，例如 [148]，涉及将问题转换为基于核的框架，目标是将原始问题转化为高维空间中更易处理的形式。核化主要用于将文本序列映射到高维空间，在此空间中任务相关信息能够更容易地被捕捉。在这个新空间中，文本序列中的每个词被表示为高维向量，这些向量之间的距离用于衡量它们的相似度。低秩 [147] 旨在将一个高维矩阵分解为两个低维矩阵的乘积。因此，通过计算这两个低维矩阵的逆，可以获得注意力矩阵的近似逆，从而显著降低计算复杂度。

## 4.2 框架

### 混合专家模型

混合专家（Mixture of Experts, MoE）[89] 的核心思想是将一个大规模模型分解为若干个小模型，每个小模型专注于学习输入数据的特定部分。在训练过程中，每个专家被分配一个权重，该权重决定了它在整体模型中的重要性。在推理阶段，给定一个输入，所有专家按相关性排序，并选择最相关的专家进行计算。这种方法大大减少了计算量，因为只有一部分专家参与计算。通过在不同专家之间分配计算任务，MoE 在训练和推理阶段都实现了更高效的计算资源利用。在 MoE 中，每个专家都有自己的一组参数；然而，这些参数在训练过程中是共享的。这种参数共享策略减少了模型的整体参数量，从而降低了存储和计算成本。

GShard [149] 是由一组轻量级标注 API 和 XLA 编译器扩展组成的模块，它以优雅的方式表达各种并行计算模式，同时只需对现有模型代码进行最少的修改。它使我们能够利用稀疏门控混合专家，将多语言神经机器翻译 Transformer 模型扩展到超过 6000 亿参数，并实现自动分片。Switch Transformer [150] 用 MoE 路由层替换了标准 Transformer 中的前馈网络（Feed Forward Network, FFN）层，其中每个专家独立地作用于序列中的词元。在相同的计算资源下，其训练速度是 Google 之前开发的最大模型 T5-XXL 的四倍。所提出的训练技术消除了训练过程中的不稳定性，证明了大型稀疏模型也可以以低精度格式（如 bfloat16）进行训练。

### Transformer 替代结构

尽管 Transformer 是当前大规模语言模型中的主导架构，但 RWKV [151] 和 Mamba [77] 等模型已成为实现更高效率和长文本处理的热门方案。这些创新模型展现出了与 Transformer 相似的特性，包括处理长距离依赖和并行计算的能力。RWKV 模型利用线性注意力机制，使我们能够将模型表述为 Transformer 或循环神经网络（Recurrent Neural Network, RNN）。这种方法在训练期间并行化计算，并在推理期间保持恒定的计算和内存复杂度。

状态空间模型（State Space Models, SSMs）[152] 可以表述为一种 RNN，用于高效的自回归推理，并已成为注意力机制的一个有前景的替代方案，其计算复杂度接近线性，而注意力的计算复杂度为二次型。SSMs 表述为：

x'(t) = Ax(t) + Bu(t), y(t) = Cx(t) + Du(t)

将一维输入信号 u(t) 映射到 N 维潜在状态 x(t)，然后再投影到一维输出信号 y(t)，其中 A、B、C 和 D 是通过梯度下降 [152] 学习的参数。已有多种技术被提出用于增强 SSMs，例如结构化状态空间序列模型（Structured State Space sequence model, S4）[152]，它通过对矩阵 A 施加低秩修正来改进 SSMs；对角状态空间（Diagonal State Space, DSS）模型 [153]，它提出状态空间的完全对角参数化以获得更高的效率。H3 将两个 SSM 堆叠在一起，使其输出与输入投影进行交互，弥合了 SSM 与注意力之间的差距，同时适应现代硬件。

Mamba [77] 是一种选择性状态空间模型，已被引入作为大语言模型中 Transformer 架构的有力竞争者。Mamba 结合了一种选择机制来消除不相关的数据，并为循环操作开发了一种硬件感知的并行算法。这在与同容量 LLM 相当的性能下，实现了更快的推理速度，且推理时间呈线性扩展、内存使用恒定。总之，状态空间模型通过提供近线性的计算复杂度和有效捕捉长距离依赖的能力，作为注意力机制的替代方案展现出巨大的潜力。随着持续的进步和完善，SSM 有望成为深度学习和序列处理领域中有影响力的方法。

## 4.3 微调

微调作为使 LLM 适应下游任务以及训练多模态大语言模型（MLLLM）遵循视觉指令的主要阶段，在提升 LLM 效率方面发挥着至关重要的作用。

### 参数高效微调

参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）是一种旨在以较少参数在大语言模型（LLM）中实现高性能的方法。适配器微调（Adapter-based Tuning）和低秩自适应（Low-Rank Adaptation）等技术提供了有效的解决方案，以减轻与微调 LLM 相关的计算和内存挑战，同时保持其表达能力和泛化能力。适配器微调将轻量级适配器模块引入预训练模型的架构中。这些适配器模块通常由参数数量较少的前馈神经网络组成，被插入到原始模型的各层之间。在微调过程中，仅更新适配器参数，而预训练模型的参数保持固定。这种方法显著减少了可训练参数的数量，从而加快了训练和推理速度，且不影响模型的性能。

LLM-Adapters [154] 提出了一个将各种适配器集成到大语言模型中的框架，支持多样化任务的参数高效微调。该框架涵盖了最先进的开源大语言模型以及多种广泛使用的适配器。(IA)³ [155] 引入了一种新颖的参数高效微调方法——通过抑制和放大内部激活来注入适配器（Infused Adapters by Inhibiting and Amplifying Inner Activations），它通过学习向量与激活相乘来为模型参数加权，从而在推理期间实现鲁棒的少样本性能和任务混合，而无需手动调整模型结构。

低秩自适应 [161] 采用矩阵分解技术来减少模型中的参数数量。通过将原始权重矩阵分解为低秩矩阵，低秩自适应捕捉模型表示中最重要的组成部分，同时丢弃不太重要的信息。这 resulting 在一个参数更少的紧凑模型中，可以更高效地进行微调。在 LoRA-FA [156]（LoRA 的一种变体）中，第一个低秩矩阵在初始化后被冻结，用作随机投影，而另一个则被训练。这使参数数量减少了一半，同时保持了与传统 LoRA 技术相当的性能。DyLoRa [157] 引入了一种动态低秩自适应技术，能够为一系列秩（而非单一秩）训练 LoRA 块，这是通过在对不同秩进行训练期间对适配器模块学习的表示进行排序来实现的。

### 全参数微调

全参数微调是一种在微调过程中更新预训练模型所有参数的方法。该方法旨在通过利用预训练模型的全部容量，在特定下游任务上实现最优性能。虽然全参数微调通常能产生最先进的结果并提高任务特定性能，但它对计算能力和内存消耗有更高的资源要求。为减轻训练负担，许多研究集中在提高全参数微调期间的内存效率。这种策略方法有效地减少了曾经阻碍该研究领域进展的障碍。

LOMO [158] 引入了一种源自随机梯度下降（Stochastic Gradient Descent, SGD）的低内存优化（Low-Memory Optimization）技术来减少内存消耗。通常使用 ADAM 优化器，但该方法中的优化器状态占用了大量内存。通过使用修改后的基于 SGD 的 LOMO，可以减少内存使用。虽然 SGD 本身面临三个挑战，但这些问题在模型微调期间往往会自行解决。具体修改涉及在梯度计算内更新参数，而不是在整个层之后更新。MeZO [159] 提出了一种仅通过两次前向传播来计算梯度的优化器，使得 LLM 的微调内存占用与推理时相同。在 55GB 的 GPU 内存需求下，它允许对 300 亿参数模型进行全面微调。

## 5 训练

高效多模态大语言模型（MLLM）的训练过程是决定其在下游任务上性能以及处理多样模态能力的关键方面。在本节中，我们概述了各种训练方法，包括预训练、指令微调、多样训练步骤和参数高效迁移学习策略。这些方法旨在优化不同模态之间的对齐、在特定任务上微调模型，并减少与迁移学习过程相关的计算和参数成本。图 14 展示了开发高效 MLLM 所涉及的不同训练阶段的示意图。在以下小节中，我们将深入探讨这些方面的每一个，并讨论它们在高效 MLLM 背景下的意义。

## 5.1 预训练

在预训练阶段，主要关注在嵌入空间中对齐不同的模态，使语言模型能够接受来自各种模态的输入。这一阶段的训练主要涉及大规模的文本配对数据， predominantly 以图像-标题对的形式。一个图像-标题对 (X, Y) 通常被扩展为单轮对话 (X<sup>instruct</sup>, X<sup>a</sup>)，其中 X<sup>instruct</sup> 包含一张图像 X<sup>v</sup> 和从一组指令中随机采样的问题 X<sup>q</sup>，该指令要求助手简要描述图像，X<sup>a</sup> 是原始图像描述。给定这样的对话，模型被训练为自回归地预测图像描述。因此，我们可以计算在 X<sup>v</sup> 和 X<sup>instruct</sup> 条件下预测 X<sup>a</sup> 的概率，并使用标准交叉熵损失函数对其进行优化：

$$
\max_\theta \sum_{i=1}^{L} \log p_\theta(x_i | X_v, X_{instruct}, X_a, <i)
$$

其中 L 是 X<sup>a</sup> 的长度，θ 表示可训练参数。为了更好地在预训练阶段对齐不同模态的知识并避免灾难性遗忘，θ 通常仅包括可学习的模态接口，即视觉-语言投影器（Vision-Language Projector）。

**应该解冻哪部分？** 考虑到在使用小语言模型（SLM）时仅训练连接器可能不足以很好地对齐视觉和文本信息，TinyLlava [23] 也选择部分解冻预训练模块（即视觉编码器和小语言模型），以激活更多参数用于学习对齐。VILA [49] 揭示，在预训练阶段更新基础 LLM 对于继承一些有吸引力的 LLM 特性（如上下文学习）至关重要。ShareGPT4V [55] 发现，解冻更多参数，特别是在视觉编码器后半层的参数，在学习更大更多样化的数据集时被证明是有益的，表明训练方案的选择与数据质量密切相关。

**多阶段预训练** 为最大化计算效率，Idefics2 [48] 将预训练分解为两个阶段。第一阶段，将最大图像分辨率限制为 384 像素并使用大批次大小。第二阶段引入 PDF 文档，将图像分辨率提高到最大 980 像素以确保文本可读。

## 5.2 指令微调

指令微调（Instruction Tuning, IT）是高效 MLLM 的关键方面，旨在通过利用任务特定指令在特定任务上微调模型。这种方法建立在 MLLM 能够理解和遵循自然语言提供的指令这一概念之上，从而增强其在目标任务上的性能。IT 在高效 MLLM 中的好处是多方面的。首先，它使模型能够以最小的架构或训练数据变化适应广泛的任务。这使其成为一种灵活高效的方法，适用于多样化任务的微调。其次，IT 允许更好的泛化，因为模型学习遵循指令并将其知识应用于新的、未见过的任务。

IT 阶段通常在监督微调（Supervised Fine-Tuning, SFT）范式内进行。SFT 数据集通常来自预训练数据的一部分，被转换为基于指令的格式，以单轮或多轮对话结构呈现。给定一张图像 X<sup>v</sup> 及其标题，可以生成对话数据 (X<sup>q1</sup>, X<sup>a1</sup>, ..., X<sup>qT</sup>, X<sup>aT</sup>)，其中 T 是总轮数。通常，可以按照 [7] 将数据组织为一系列指令和响应，其中第 t 轮的指令 X<sup>instruct</sup><sub>t</sub> 为：

$$
X_{instruct}^t = \begin{cases} 
\text{随机选择 } [X_q^1, X_v] \text{ 或 } [X_v, X_q^1], & \text{第一轮 } t = 1 \\
X_q^t, & \text{剩余轮次 } t > 1
\end{cases}
$$

利用这种多模态指令遵循序列，可以使用与预训练阶段相同的自回归训练目标进行 IT。一种普遍的策略是在 IT 过程中保持视觉编码器权重固定，同时继续更新投影器和小语言模型（SLM）的预训练权重。

**高效指令微调** 当前的 IT 解决方案成本极高，需要优化大量参数和额外的大规模训练。LaVIN [50] 为 MLLM 的高效指令微调引入了一种创新且成本效益高的解决方案。LaVIN 中的混合模态适配（Mixture-of-Modality Adaptation, MMA）使用轻量级模块来弥合 LLM 与视觉语言（VL）任务之间的差距。这也促进了视觉和语言模型的联合优化。实现 LaVIN 的实际成本非常低，例如，它仅需 1.4 小时训练时间和 380 万可训练参数。HyperLLaVA [51] 研究了 MLLM 中尚未充分探索的动态微调策略，并在两阶段训练中利用视觉和语言引导的动态微调来调整投影器和 LLM。

## 5.3 多样训练步骤

传统的两阶段策略需要手动将各种可调参数和数据集组合分配到不同的训练阶段，这可能是一项繁琐的任务。为缓解这一问题，SPHINX-X [14] 设计了一种单阶段、全面的训练流水线，平等对待所有收集的数据集，并一致地将它们转换为多模态、多轮对话格式。在这个统一的训练阶段，SPHINX-X 中除视觉编码器外的所有参数都被激活。Cobra [13] 也认为预对齐的初始阶段可能不是必需的，即使在微调后模型仍然欠拟合。因此，它放弃了预对齐阶段，选择直接对整个小语言模型（SLM）骨干网络和投影器进行微调。TinyGPT-V [28] 的训练过程包括四个阶段：一个用于视觉语言理解的初始预训练阶段、一个用于细化图像模态处理的第二阶段、一个通过微调进行类人学习的第三阶段，以及一个用于增强其作为聊天机器人对话能力的多任务学习第四阶段。

## 5.4 参数高效迁移学习

多项研究采用参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）技术进行迁移学习，如 LoRA [161]，以防止预训练知识的丢失。高效注意力跳跃（Efficient Attention Skipping, EAS）模块 [52] 提出了一种新颖的参数和计算高效的微调方法，使 MLLM 在下游任务上保持高性能并减少参数和计算开销。MemVP [53] 认为这种迁移学习范式仍然存在低效率，因为它显著增加了语言模型的输入长度。MemVP 中的视觉提示与前馈网络的权重拼接，以实现视觉知识注入，减少了微调后 MLLM 的训练时间和推理延迟，并超越了之前的 PEFT 方法。

## 6 数据与基准

在本节中，我们概述了用于训练和评估高效 MLLM 的数据和基准。我们讨论了预训练数据、指令微调数据以及用于评估这些模型性能的基准的重要性。讨论强调了多样化和高质量数据集在实现稳健和准确的 MLLM 中的重要性，以及用于生成和完善这些数据集的各种策略。此外，我们提供了跨已建立基准的 MLLM 性能综合比较，强调了进行彻底评估的必要性，以确保这些模型在真实世界应用中的有效性。

## 6.2 指令微调数据

指令微调（Instruction Tuning, IT）是提升高效 MLLM 准确解释用户指令和有效执行期望任务能力的关键步骤。该过程与多任务提示（multi-task prompting）的概念密切相关。常用预训练数据集的摘要见表 3。高质量的 IT 数据可以从任务特定的数据集中获得。例如，考虑视觉问答（Visual Question Answering, VQA）数据集中的一个样本，其中输入包含一张图像和一个自然语言问题，输出是基于该图像的问题文本答案。这可以轻松形成指令样本的多模态输入和响应。指令或任务描述可以通过人工创建或在 GPT 的帮助下通过半自动生成获得。

除利用公开可用的任务特定数据集外，SPHINX-X [14] 从互联网上广泛来源的 PDF 数据中组装了一个专注于光学字符识别（Optical Character Recognition, OCR）的数据集。具体而言，它首先从网络上收集大规模的 PDF 数据集，然后获取 PDF 文件中每一页的渲染结果，同时保存所有文本标注及其各自的边界框。最终，这些元素被转换为统一的问题-回答格式。

虽然多任务数据集提供了丰富的数据来源，但它们可能并不总是适合复杂的现实场景，如参与多轮对话。为应对这一挑战，一些研究探索了利用大语言模型从有限数量的人工标注样本中生成基于文本或多模态指令跟随数据的方法。SPHINX-X [14] 组装了一个具有图像和文本之间细粒度对应关系的丰富多领域数据集。它从多样化的来源收集图像，然后使用标注在原始图像上应用各种标记。通过使用这些标记图像和量身定制的领域特定准则提示 GPT-4V，系统生成提供图像概述、区域细节和对象关系洞察的标题。在训练过程中，SPHINX-X 使用未更改的图像而非标记的图像。ALLaVA [29] 提出在单个会话中为一张图像生成一个标题和一个问答对。具体而言，它向 GPT-4V 提示一张图像，并要求它首先生成一个细粒度标题，然后生成一个视觉问答对。

此外，除多模态教学数据外，用户与助手之间纯粹基于语言的对话可以显著增强模型的对话专业知识和对指令的响应能力。例如，VILA [49] 的研究表明，在微调过程中将纯文本教学数据与图像-文本数据整合，不仅能减轻纯文本任务性能的下降，还能提高 MLLM 相关任务的准确性。

## 7 应用场景

从前面的分析可以看出，许多高效 MLLM 方法在一系列场景中评估其性能，如视觉问答（Visual Question Answering, VQA）、视觉定位（visual grounding）、图像分割（image segmentation）等。然而，在成熟的基准任务上探索这些高效架构以实现其极限性能也至关重要。因此，我们选择介绍几个下游任务，如医学分析（medical analysis）、文档理解（document understanding）和视频理解（video comprehension）。

### 7.1 生物医学分析

由于标注生物医学数据的成本高昂，基础模型（foundation models）有望成为生物医学领域的新范式，在许多应用上达到最先进（state-of-the-art）的结果，包括医学问答 [194] 和医学图像分类 [195]。最近，多模态生成式人工智能（multimodal generative AI）已成为生物医学领域激动人心的前沿，将应用范围从单模态扩展到多模态，如视觉问答和放射学报告生成（radiology report generation）。专家混合微调（Mixture of Expert Tuning, MoE Tuning）已有效提升了通用 MLLM 的性能且参数量更少，但其在资源受限的医疗环境中的应用尚未得到充分探索。MoE-TinyMed [64] 是一个为医疗应用量身定制的模型，显著降低了参数需求。LLaVA-Rad [63] 是一个最先进的工具，在私有环境下单张 V100 GPU 上即可快速运行，非常适合真实临床场景。它采用模块化方法，集成单模态预训练模型并强调轻量适配器（lightweight adapter）的训练。因此，LLaVA-Rad 在标准指标上超越了 GPT-4V 和 Med-PaLM 等更大规模的模型，展现了其卓越的效率和有效性。

### 7.2 文档理解

文档或图表是重要的信息来源，以直观的形式可视化数据。它们已成为信息传播、商业决策和学术研究中不可或缺的一部分。然而，当前的图表理解模型仍面临两个主要局限性：（1）参数量庞大使得训练和部署困难。例如，ChartLlama [196] 是一个 130 亿参数的模型，难以在单张消费级 GPU 上部署。（2）这些模型难以高效编码高分辨率图像，因为视觉 Transformer 倾向于产生过长的特征序列。
为了应对面向文档的 MLLM 在细粒度视觉感知和视觉信息压缩方面的挑战，TinyChart [37] 采用思维链编程学习（Program-of-Thoughts, PoT learning）和视觉词元合并（Visual Token Merging）策略，在超越多个 13B MLLM 的同时，在更快的推理速度上也表现出色。TextHawk [36] 通过设计四个专用组件来应对面向文档的任务所带来的挑战，探索了高效的细粒度感知。HRVDA [66] 和 Monkey [65] 也是为应对视觉文档理解任务中高分辨率需求所带来的挑战而设计的大型多模态模型。

### 7.3 视频理解

视频对人类持续感知视觉世界的方式提供了令人印象深刻的准确表征。智能视频理解对于多种真实应用至关重要，包括视频分类（video category classification）、视频描述生成（video captioning）和视频-文本检索（video-text retrieval）。videoChat [197] 和 Video-LLaMA [198] 等工作是基于大语言模型的大型多模态模型，用于端到端的以对话为中心的视频理解。然而，这些方法只能接收有限数量的帧用于短视频理解。
为了应对因视觉词元数量过多而处理长视频带来的计算挑战，已开发了多种方法。mPLUG-video [67] 专为视频理解任务设计，首先采用基于 TimeSformer 的视频编码器从稀疏采样的视频帧中有效提取特征，随后通过视觉抽象器（visual abstractor）模块来缩短序列长度。Video-LLaVA [44] 在投影之前将图像和视频的视觉表征统一到单一语言特征空间中，在多种视频理解任务中表现出色。这种方法使其能够通过 LanguageBind [93] 有效学习多模态交互。LLaMA-VID [69] 通过用两个不同的词元表示每一帧来解决这一问题，即上下文词元（context token）和内容词元（content token）。上下文词元基于用户输入编码整体图像上下文，而内容词元则封装每一帧中的视觉线索。这种双词元策略显著降低了长视频的负担，同时保留了关键信息。与大多数现有工作尝试同时处理更多帧不同，MA-LMM [68] 提出以在线方式处理视频，并将过去的视频信息存储在记忆库（memory bank）中，以供长期分析时参考历史视频内容，同时不超过大语言模型的上下文长度约束或 GPU 内存限制。

## 8 讨论与结论

### 8.1 局限性与未来工作

高效 MLLM 的发展仍处于起步阶段，仍有很大的改进空间。我们总结当前状况如下：

- 目前，高效 MLLM 在处理扩展上下文多模态信息方面面临挑战，通常仅限于接受单张图像。这阻碍了能够处理更多多模态词元的更复杂模型的发展。这类模型将有助于理解长视频和分析包含图像和文本混合内容的大文档等应用，从而创建更通用和更强大的系统。
- 主流高效 MLLM 主要支持两种输入模态——图像和文本，以及一种输出模态——文本。然而，现实世界涵盖了更广泛的模态。通过将高效 MLLM 的范围扩展到适应更丰富的输入模态多样性，并增强其生成能力，我们可以显著提升其多功能性并扩大其应用范围。
- 加强高效 MLLM 模型有两条主要途径。首先，引入更多样化的轻量级 LLM 可以使 MLLM 的设计更具适应性，便于定制以满足广泛的需求。其次，利用高质量的指令微调数据集可以使高效 MLLM 更好地理解和执行大量指令，从而增强其零样本学习（zero-shot learning）能力。
- 能够在边缘设备上部署的具身智能体（embodied agents）的发展代表了高效 MLLM 的一个关键应用前景。拥有专业知识并能够与现实世界交互的智能体具有深远的意义，可能彻底改变机器人技术、自动化和人工智能等领域。

### 8.2 结论

本研究深入探讨了高效 MLLM 文献领域，全面审视了其核心主题，包括基础理论及其扩展。我们的目标是识别和突出需要进一步研究的领域，并提出未来研究的潜在方向。我们希望提供对当前高效 MLLM 状态的全面视角，希望能激发更多研究。鉴于该领域的动态特性，一些最新进展可能未被完全涵盖。为此，我们建立了一个专门的网站，通过众包（crowdsourcing）来追踪最新进展。该平台旨在作为持续更新的信息来源，促进该领域的持续发展。由于篇幅限制，我们无法深入涵盖所有技术细节，但对该领域的主要贡献提供了简要概述。未来，我们计划持续更新和增强我们网站上的信息，随时添加新的见解。
