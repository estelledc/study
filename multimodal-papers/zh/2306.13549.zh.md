# 多模态大语言模型综述

Shukang Yin* , Chaoyou Fu*†, Sirui Zhao*, Ke Li, Xing Sun, Tong Xu, and Enhong Chen, Fellow, IEEE

## 摘要

近年来，以 GPT-4V（Generative Pre-trained Transformer 4 with Vision）为代表的多模态大语言模型（Multimodal Large Language Model, MLLM）已成为一个新兴的研究热点，它利用强大的大语言模型（Large Language Models, LLM）作为"大脑"来执行多模态任务。MLLM 展现出了一些令人惊讶的涌现能力，例如基于图像编写故事和无光学字符识别（OCR-free）的数学推理，这些能力在传统多模态方法中极为罕见，暗示了通向人工通用智能（Artificial General Intelligence, AGI）的可能路径。为此，学术界和工业界都在努力开发能够与甚至超越 GPT-4V 的 MLLM，以惊人的速度推动研究前沿。本文旨在追踪和总结 MLLM 的最新进展。首先，我们给出了 MLLM 的基本公式，并厘清了其相关概念，包括架构、训练策略与数据，以及评估方法。随后，我们介绍了 MLLM 如何扩展以支持更细粒度的交互、更多模态、更多语言和更多场景。接着，我们讨论了多模态幻觉（multimodal hallucination）问题和扩展技术，包括多模态上下文学习（Multimodal In-Context Learning, M-ICL）、多模态思维链（Multimodal Chain of Thought, M-CoT）以及大语言模型辅助视觉推理（LLM-Aided Visual Reasoning, LAVR）。最后，我们讨论了现有挑战并指出了有前景的研究方向。鉴于 MLLM 时代才刚刚开始，我们将持续更新本综述，希望能激发更多研究。一个收集最新论文的关联 GitHub 链接可在 https://github.com/BradyFU/Awesome-Multimodal-Large-Language-Models 获取。

**索引词** — 多模态大语言模型（Multimodal Large Language Model）、视觉语言模型（Vision Language Model）、大语言模型（Large Language Model）。

## 1 引言

近年来，LLM 取得了显著的进展[1], [2], [3], [4], [5]。通过扩大数据规模和模型规模，这些 LLM 展现出了非凡的涌现能力，典型地包括指令遵循（instruction following）[5], [6]、上下文学习（In-Context Learning, ICL）[7] 和思维链（Chain of Thought, CoT）[8]。尽管 LLM 在大多数自然语言处理（Natural Language Processing, NLP）任务上表现出了令人惊讶的零样本/少样本推理性能，但它们本质上对视觉是"盲"的，因为它们只能理解离散的文本。同时，大型视觉模型（Large Vision Models, LVM）能够清晰地"看"[9], [10], [11], [12]，但通常在推理方面有所欠缺。

正是考虑到这种互补性，LLM 和 LVM 相互靠近，催生了多模态大语言模型（MLLM）这一新领域。形式化地，它指的是基于 LLM 的模型，具有接收、推理和输出多模态信息的能力。在 MLLM 出现之前，已有大量关于多模态的研究工作，可分为判别式（discriminative）[13], [14], [15] 和生成式（generative）[16], [17], [18] 范式。CLIP[13] 作为前者的代表，将视觉和文本信息投影到统一的表征空间，为下游多模态任务搭建了桥梁。相比之下，OFA[16] 是后者的代表，它以序列到序列（sequence-to-sequence）的方式统一了多模态任务。按照序列操作方式分类，MLLM 属于后者，但它展现出两个相较于传统方法的代表性特征：(1) MLLM 基于具有数十亿规模参数的 LLM，这是先前模型所不具备的。(2) MLLM 采用新的训练范式来释放其全部潜力，例如使用多模态指令微调（multimodal instruction tuning）[19], [20] 来鼓励模型遵循新的指令。凭借这两个特征，MLLM 展现出了一些新的能力，例如基于图像编写网站代码[21]、理解表情包的深层含义[22]，以及无 OCR 的数学推理[23]。

自 GPT-4[3] 发布以来，由于其展现出的令人惊叹的多模态示例，MLLM 的研究热潮便持续升温。快速的发展由学术界和工业界的共同努力推动。初步的 MLLM 研究聚焦于基于文本提示和图像[20], [24]/视频[25], [26]/音频[27] 的文本内容生成。后续工作扩展了其能力或使用场景，包括：(1) 更好的粒度支持。开发了更精细的用户提示控制，以通过边界框（bounding box）[28] 或点击（click）[29] 来支持特定区域或某个特定对象。(2) 输入和输出模态的增强支持[30], [31]，如图像、视频、音频和点云。除输入外，像 NExT-GPT[32] 这样的项目进一步支持不同模态的输出。(3) 改进的语言支持。人们已付出努力将 MLLM 的成功扩展到其他语言（例如中文），仅使用相对有限的训练语料[33], [34]。(4) 扩展到更多领域和使用场景。一些研究将 MLLM 的强大能力转移到其他领域，如医学图像理解[35], [36], [37] 和文档解析[38], [39], [40]。此外，还开发了多模态智能体（multimodal agents）来协助现实世界的交互，例如具身智能体（embodied agents）[41], [42] 和图形用户界面智能体（GUI agents）[43], [44], [45]。MLLM 发展时间线如图 1 所示。

考虑到该领域的如此快速进展和有前景的结果，我们撰写本综述，旨在为研究人员提供对 MLLM 的基本思想、主要方法和当前进展的把握。需要注意的是，我们主要关注视觉和语言模态，但也包括涉及视频和音频等其他模态的工作。具体地，我们涵盖了 MLLM 最重要的方面，并给出了相应总结，同时开放了一个 GitHub 页面以实时更新。据我们所知，这是第一篇关于 MLLM 的综述。

本综述的后续部分结构安排如下：首先对 MLLM 的基本方面进行全面回顾，包括 (1) 主流架构（§2）；(2) 训练策略与数据的完整方案（§3）；(3) 性能评估的常见做法（§4）。随后，我们深入讨论关于 MLLM 的一些重要主题，每个主题聚焦于一个主要问题：(1) 哪些方面可以进一步改进或扩展（§5）？(2) 如何缓解多模态幻觉问题（§6）？接着，我们介绍了三种关键技术（§7），每种技术针对一个特定场景：多模态 ICL（M-ICL, §7.1）是一种在推理阶段常用的有效技术，用于提升少样本性能；另一种重要技术是多模态 CoT（M-CoT, §7.2），通常用于复杂的推理任务；之后，我们概述了一种开发基于 LLM 的系统以解决复合推理任务或处理常见用户查询的通用思路（§7.3）。最后，我们以总结和潜在研究方向结束全文。鉴于 MLLM 时代才刚刚开始，我们将持续更新本综述，并希望它能激发更多研究。一个收集最新论文的关联 GitHub 链接已在 https://github.com/BradyFU/Awesome-Multimodal-Large-Language-Models 发布。

## 2 架构

一个典型的 MLLM 可以抽象为三个模块：一个预训练的模态编码器（modality encoder）、一个预训练的 LLM 和一个连接它们的模态接口（modality interface）。以人类作为类比，模态编码器（如图像/音频编码器）类似于人类的眼睛/耳朵，负责接收和预处理光学/声学信号；而 LLM 则类似于人类的大脑，负责理解和推理处理后的信号；模态接口则用于对齐不同模态。一些 MLLM 还包含一个生成器（generator）来输出除文本之外的其他模态。架构图如图 2 所示。本节将依次介绍每个模块。

### 2.1 模态编码器

编码器将原始信息（如图像或音频）压缩为更紧凑的表征。一种常见的做法是使用经过预训练的编码器，该编码器已经与其他模态进行了对齐。例如，CLIP[13] 通过在大规模图像-文本对上进行预训练，包含了一个与文本语义对齐的视觉编码器。因此，使用这类预先对齐的编码器来与 LLM 进行对齐预训练（alignment pre-training，见 §3.1）会更加容易。

常用的图像编码器系列总结在表 1 中。除了原始的 CLIP 图像编码器[13] 之外，一些工作还探索了使用其他变体。例如，MiniGPT-4[21] 采用了 EVA-CLIP[47], [48]（ViT-G/14）编码器，该编码器使用了改进的训练技术进行训练。相比之下，Osprey[29] 引入了一个基于卷积的 ConvNeXt-L 编码器[46] 来利用更高分辨率和多层级特征。一些工作还探索了免编码器（encoder-free）架构。例如，Fuyu-8B[49] 的图像块（image patches）在送入 LLM 之前直接被投影，因此模型天然支持灵活分辨率的图像输入。

在选择编码器时，通常需要考虑分辨率、参数量和预训练语料等因素。值得注意的是，许多工作已通过经验验证，使用更高分辨率可以实现显著的性能提升[34], [50], [51], [52]。扩大输入分辨率的方法可分为直接缩放（direct scaling）和分块分割（patch-division）两种。直接缩放的方式是将更高分辨率的图像输入编码器，通常需要进一步微调编码器[34] 或用更高分辨率的预训练编码器替换[50]。类似地，CogAgent[44] 使用了一种双编码器机制，其中两个编码器分别处理高分辨率和低分辨率图像，高分辨率特征通过交叉注意力（cross-attention）注入到低分辨率分支。分块分割方法将高分辨率图像分割成多个块（patch），并复用低分辨率编码器。例如，Monkey[51] 和 SPHINX[53] 将大图像分割成更小的子图像，与下采样后的高分辨率图像一起送入图像编码器，其中子图像和低分辨率图像分别捕获局部和全局特征。相比之下，参数量和训练数据组成的重要性低于输入分辨率，这一点已被实证研究[52] 所证实。

**专家模型**。除了可学习接口之外，使用专家模型（如图像描述生成模型）也是一种弥合模态差距的可行方法[77], [78], [79], [80]。基本思想是在不进行训练的情况下将多模态输入转换为语言，这样 LLM 就能通过转换后的语言来理解多模态信息。例如，VideoChat-Text[25] 使用预训练的视觉模型来提取动作等视觉信息，并使用语音识别模型来丰富描述。尽管这种方式简单有效，但它在很大程度上依赖于外部专家模型的质量，并可能引入额外的延迟和错误。

## 3 训练策略与数据

一个成熟的 MLLM 经历三个阶段，即预训练（pre-training）、指令微调（instruction-tuning）和对齐微调（alignment tuning）。每个训练阶段需要不同类型的数据并实现不同目标。在本节中，我们讨论每个训练阶段的训练目标、数据收集和数据特征。

### 3.1 预训练

#### 3.1.1 训练细节

作为第一个训练阶段，预训练主要旨在对齐不同模态并学习多模态世界知识。预训练阶段通常需要大规模文本配对数据，例如描述数据（caption data）。通常，描述对用自然语言句子描述图像/音频/视频。这里，我们考虑一个常见场景，其中 MLLM 被训练来对齐视觉与文本。如表 3 所示，给定一张图像，模型被训练为自回归地预测该图像的描述，遵循标准交叉熵损失。预训练的常见方法是保持预训练模块（如视觉编码器和 LLM）冻结，并训练一个可学习的接口[20], [35], [72]。其思想是在不丢失预训练知识的情况下对齐不同模态。一些方法[34], [81], [82] 也解冻更多模块（如视觉编码器）以启用更多可训练参数用于对齐。需要注意的是，训练方案与数据质量密切相关。

对于简短且嘈杂的描述数据，可以采用较低的分辨率（如 224）来加快训练过程，而对于更长、更干净的数据，最好使用更高的分辨率（如 448 或更高）来减轻幻觉。此外，ShareGPT4V[83] 发现，在预训练阶段使用高质量描述数据来解锁视觉编码器能促进更好的对齐。

#### 3.1.2 数据

预训练数据主要服务于两个目的：(1) 对齐不同模态；(2) 提供世界知识。预训练语料可以根据粒度分为粗粒度和细粒度数据，我们将依次介绍。我们将常用的预训练数据集总结在表 4 中。

**粗粒度描述数据**有一些典型共同特征：(1) 数据量大，因为样本通常来源于互联网。(2) 由于网络爬取的性质，描述通常简短且嘈杂，因为它们源自网络图像的替代文本（alt-text）。这些数据可以通过自动化工具进行清洗和过滤，例如使用 CLIP[13] 模型来过滤相似度低于预定义阈值的图像-文本对。下面我们介绍一些代表性的粗粒度数据集。

**CC。** CC-3M[84] 是一个包含 330 万图像-文本对的网络规模描述数据集，原始描述来源于与图像关联的替代文本。作者设计了一个复杂的管道来清洗数据：(1) 对于图像，过滤掉具有不当内容或宽高比的图像。(2) 对于文本，使用 NLP 工具获取文本标注，根据设计的启发式规则过滤样本。(3) 对于图像-文本对，通过分类器为图像分配标签。如果文本标注与图像标签不重叠，则删除相应样本。

**CC-12M** [85] 是 CC-3M 的后续工作，包含 1240 万图像-文本对。与之前的工作相比，CC-12M 放宽并简化了数据收集管道，从而收集了更多数据。

**SBU Captions** [86]。这是一个包含 100 万图像-文本对的带描述照片数据集，图像和描述均来源于 Flickr。具体地，通过用大量查询词搜索 Flickr 网站获取初始图像集，附加在图像上的描述即作为标题。然后，为了确保描述与图像相关，保留的图像需满足以下要求：(1) 图像描述长度令人满意，由观察决定。(2) 描述包含预定义术语列表中至少 2 个词和一个通常暗示空间关系的命题词（如"在"、"下"）。

**LAION。** 该系列是大规模网络数据集，图像从互联网上爬取，关联的替代文本作为描述。为了过滤图像-文本对，执行以下步骤：(1) 删除文本过短或图像过大或过小的样本。(2) 基于 URL 进行图像去重。(3) 为图像和文本提取 CLIP[13] 嵌入，并使用嵌入来删除可能非法的内容和嵌入之间余弦相似度低的图像-文本对。以下是一些典型变体的简要总结：

- **LAION-5B** [87]：它是一个包含 58.5 亿图像-文本对的研究用途数据集。该数据集是多语言的，包含一个 20 亿英语子集。
- **LAION-COCO** [88]：它包含从 LAION-5B 英语子集中提取的 6 亿张图像。描述是合成的，使用 BLIP[89] 生成各种图像描述，并使用 CLIP[13] 为图像选择最匹配的。

**COYO-700M** [90]。它包含 7.47 亿图像-文本对，从 CommonCrawl 中提取。对于数据过滤，作者设计了以下策略：(1) 对于图像，过滤掉大小、内容、格式或宽高比不当的图像。此外，基于 pHash 值过滤图像，以移除与 ImageNet 和 MS-COCO 等公共数据集重叠的图像。(2) 对于文本，仅保存具有满意长度、名词形式和适当词汇的英文文本。将句子前后的空白字符移除，连续空白字符替换为单个空白字符。此外，出现超过 10 次的文本（如"图像用于"）将被删除。(3) 对于图像-文本对，基于（图像 pHash, 文本）元组移除重复样本。

最近，更多工作[83], [91], [92] 已探索通过提示强大的 MLLM（如 GPT-4V）来生成高质量的细粒度数据。与粗粒度数据相比，这些数据通常包含对图像更长、更准确的描述，从而实现图像和文本模态之间更细粒度的对齐。然而，由于这种方法通常需要调用商业用途的 MLLM，成本更高，数据量相对较小。值得注意的是，ShareGPT4V[83] 通过首先用 GPT-4V 生成的描述训练一个描述生成器，然后将数据规模扩展到 120 万，取得了平衡。

### 3.2 指令微调

#### 3.2.1 概述

指令（instruction）是指对任务的描述。直观地，指令微调旨在教会模型更好地理解来自用户的指令并完成要求的任务。通过这种方式微调，LLM 可以通过遵循新指令来泛化到未见过的任务，从而提升零样本性能。这个简单但有效的想法催生了后续 NLP 工作的成功，如 ChatGPT[2]、InstructGPT[95]、FLAN[19], [56] 和 OPT-IML[96]。

指令微调与相关典型学习范式的比较如图 3 所示。监督微调方法通常需要大量任务特定的数据来训练任务特定的模型。提示方法减少了对大规模数据的依赖，并可以通过提示工程完成特定任务。在这种情况下，虽然少样本性能有所提升，但零样本性能仍然相当一般[7]。相比之下，指令微调学习如何泛化到未见过的任务，而非像另外两种方法那样拟合特定任务。此外，指令微调与多任务提示（multi-task prompting）[97] 高度相关。

**图 3：三种典型学习范式的比较。** 图像来自 [19]。
(A) 预训练-微调（BERT, T5）：需要大量任务特定样本，每个任务需要一个专用模型。
(B) 提示（GPT-3）：通过少样本提示或提示工程提高性能。
(C) 指令微调（FLAN）：在多个任务上指令微调预训练模型，可执行未见过的任务。

在本节中，我们描述指令样本的格式、训练目标、收集指令数据的典型方式以及相应的常用数据集。

#### 3.2.2 训练细节

多模态指令样本通常包括可选的指令和输入-输出对。指令通常是描述任务的自然语言句子，例如"详细描述该图像"。输入可以是像视觉问答（Visual Question Answering, VQA）任务[99] 那样的图像-文本对，也可以是仅有图像。输出是答案。

多模态指令的格式示例如下：

```
下面是一个描述任务的指令。请写一个适当完成该请求的响应。
指令：<instruction>
输入：{<image>, <text>}
响应：<output>
```

**表 5：用于构建多模态指令数据的简化模板。** `<instruction>` 是对任务的文本描述。`{<image>, <text>}` 和 `<output>` 分别是输入和来自数据样本的输出。注意，某些数据集（如仅包含 `<image>` 的图像描述数据集）的输入中可能没有 `<text>`。该示例改编自 [98]。

形式上，多模态指令样本可以用三元组形式表示，即 (I, M, R)，其中 I、M、R 分别代表指令、多模态输入和真实响应。MLLM 在给定指令和多模态输入的情况下预测答案：

$$A = f(I, M; \theta)$$

这里，$A$ 表示预测的答案，$\theta$ 是模型的参数。训练目标通常是训练 LLM 使用的原始自回归目标[20], [37], [71], [101]，基于此，MLLM 被鼓励预测响应的下一个词元。目标可以表示为：

$$L(\theta) = -\sum_{i=1}^{N} \log p(R_i | I, R_{<i}; \theta)$$

其中 $N$ 是真实响应的长度。

#### 3.2.3 数据收集

由于指令数据在格式上更加灵活、任务形式更加多样，收集数据样本通常更加棘手和昂贵。在本节中，我们总结了三种大规模获取指令数据的典型方式，即数据适配（data adaptation）、自指令（self-instruction）和数据混合（data mixture）。

**数据适配。** 任务特定的数据集是高质量数据的丰富来源。因此，大量工作[60], [70], [76], [82], [101], [102], [103], [104] 已利用现有的高质量数据集构建指令格式的数据集。以 VQA 数据集的转换为例，原始样本是一个输入-输出对，其中输入包含一张图像和一个自然语言问题，输出是基于图像的条件答案。这些数据集的输入-输出对可以自然地构成指令样本的多模态输入和响应（见 §3.2.2）。指令，即任务的描述，可以源自手动设计或 GPT 辅助的半自动生成。具体地，一些工作[21], [35], [60], [70], [102], [105] 手工设计候选指令池，并在训练期间从中采样一条。我们提供了一种 VQA 数据集的指令模板示例，如表 6 所示。其他工作手动设计一些种子指令，并用它们来提示 GPT 生成更多指令[25], [82], [98]。

需要注意的是，由于现有 VQA 和描述数据集的答案通常简洁，直接使用这些数据集进行指令微调可能会限制 MLLM 的输出长度。有两种常见策略来解决这个问题。第一种是在指令中明确指定。例如，ChatBridge[104] 对简短回答数据明确声明"简短和简略"，对传统的粗粒度描述数据声明"一句话和单句"。第二种是扩展现有答案的长度[105]。例如，M3IT[105] 提出通过重述原始答案来扩展输出。

**自指令。** 虽然现有的多任务数据集可以作为丰富的数据来源，但它们通常不能很好地满足真实场景中的人类需求，例如多轮对话。为了解决这个问题，一些工作通过自指令[106] 收集样本，该方法利用 LLM 使用少量手工标注的样本生成文本指令遵循数据。具体地，手工制作一些指令遵循样本作为演示，之后用这些演示作为指导来提示 ChatGPT/GPT-4 生成更多的指令样本。LLaVA[20] 将这种方法扩展到多模态领域，通过将图像转换为描述和边界框的文本，并提示纯文本 GPT-4 根据要求和演示生成新数据。通过这种方式，构建了一个多模态指令数据集，称为 LLaVA-Instruct-150k。遵循这一思路，后续工作如 MiniGPT-4[21]、ChatBridge[104]、GPT4Tools[107] 和 DetGPT[72] 开发了满足不同需求的不同数据集。最近，随着更强大的多模态模型 GPT-4V 的发布，许多工作已采用 GPT-4V 来生成更高质量的数据，如 LVIS-Instruct4V[91] 和 ALLaVA[92]。我们将在表 7 中总结通过自指令生成的流行数据集。

**数据混合。** 除了多模态指令数据之外，纯语言用户-助手对话数据也可用于提高对话能力和指令遵循能力[81], [98], [101], [103]。LaVIN[101] 通过从纯语言和多模态数据中随机采样直接构建一个小批量。MultiInstruct[102] 探索了使用单模态和多模态数据融合进行训练的不同策略，包括混合指令微调（combine both types of data）和顺序指令微调（text data followed by multimodal data）。

#### 3.2.4 数据质量

近期研究揭示，指令微调样本的数据质量不亚于数据量。Lynx[73] 发现，在大规模但嘈杂的图像-文本对上预训练的模型，其表现不如在较小但更干净的数据集上预训练的模型。类似地，Wei 等人[108] 发现，更少但更高质量的指令微调数据可以取得更好的性能。对于数据过滤，该工作提出了一些评估数据质量的指标，以及相应的自动过滤低质量视觉-语言数据的方法。这里我们讨论数据质量的两个重要方面。

**提示多样性。** 指令的多样性已被发现对模型性能至关重要。Lynx[73] 通过实证验证，多样的提示有助于提高模型性能和泛化能力。

**任务覆盖。** 就训练数据中涉及的任务而言，Du 等人[109] 进行了一项实证研究，发现视觉推理任务优于描述和问答任务，能更有效地提升模型性能。此外，该研究表明增强指令的复杂性可能比增加任务多样性和纳入细粒度空间标注更有益。

### 3.3 对齐微调

#### 3.3.1 概述

对齐微调更常用于需要将模型与特定人类偏好对齐的场景，例如减少幻觉的响应（见 §6）。目前，基于人类反馈的强化学习（Reinforcement Learning with Human Feedback, RLHF）和直接偏好优化（Direct Preference Optimization, DPO）是对齐微调的两项主要技术。在本节中，我们依次介绍这两项技术的主要思想，并提供一些它们在解决实际问题中如何被使用的示例，最后给出相关数据集的汇总。

**表 8：对齐微调数据集汇总。** 对于输入/输出模态，I：图像，T：文本。

| 数据集 | 样本数 | 模态 | 来源 |
|--------|-------|------|------|
| LLaVA-RLHF [112] | 10K | I+T→T | 人工 |
| RLHF-V [114] | 5.7K | I+T→T | 人工 |
| VLFeedback [115] | 380K | I+T→T | GPT-4V |

#### 3.3.2 训练细节

**RLHF** [110], [111]。该技术旨在利用强化学习算法将 LLM 与人类偏好对齐，以人工标注作为训练循环中的监督。如 InstructGPT[95] 所示，RLHF 包含三个关键步骤：

1) **监督微调**。该步骤旨在微调预训练模型以呈现初步期望的输出行为。RLHF 设置中的微调模型称为策略模型（policy model）。注意，该步骤可能被跳过，因为监督策略模型 $\pi_{SFT}$ 可以从指令微调模型初始化（见 §3.2）。
2) **奖励建模**。在此步骤中，使用偏好对训练奖励模型。给定一个多模态提示（如图像和文本）$x$ 和响应对 $(y_w, y_l)$，奖励模型 $r_\theta$ 学习给偏好的响应 $y_w$ 更高的奖励，反之亦然给 $y_l$，遵循以下目标：

$$L(\theta) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} [\log(\sigma(r_\theta(x, y_w) - r_\theta(x, y_l))]$$

其中 $\mathcal{D} = \{(x, y_w, y_l)\}$ 是由人工标注者标注的比较数据集。在实践中，奖励模型 $r_\theta$ 与策略模型具有相似的结构。

3) **强化学习**。在此步骤中，采用近端策略优化（Proximal Policy Optimization, PPO）算法来优化 RL 策略模型 $\pi_\phi^{RL}$。通常会在训练目标中添加逐词元 KL 惩罚，以避免过度偏离原始策略[95]， resulting in the objective:

$$L(\phi) = -\mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\phi^{RL}(y|x)} r_\theta(x, y) - \beta \cdot D_{KL}(\pi_\phi^{RL}(y|x) || \pi_{REF}(y|x))$$

其中 $\beta$ 是 KL 惩罚项的系数。通常，RL 策略 $\pi_\phi^{RL}$ 和参考模型 $\pi_{REF}$ 都从监督模型 $\pi_{SFT}$ 初始化。期望通过此微调过程使获得的 RL 策略模型与人类偏好对齐。

研究人员已探索使用 RLHF 技术来改善多模态对齐。例如，LLaVA-RLHF[112] 收集了人类偏好数据，并基于 LLaVA[20] 用更少幻觉的响应微调模型。

**DPO** [113]。它利用简单二分类损失从人类偏好标签中学习。与基于 PPO 的 RLHF 算法相比，DPO 免于学习显式奖励模型，从而将整个流程简化为两个步骤，即人类偏好数据收集和偏好学习。学习目标如下：

$$L(\phi) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \log \sigma \left( \beta \log \frac{\pi_\phi^{RL}(y_w|x)}{\pi_{REF}(y_w|x)} - \beta \log \frac{\pi_\phi^{RL}(y_l|x)}{\pi_{REF}(y_l|x)} \right)$$

RLHF-V[114] 通过纠正模型中的幻觉来收集细粒度（片段级）偏好数据对。Silkie[115] 则通过提示 GPT-4V 收集偏好数据，并将偏好监督通过 DPO 蒸馏到指令微调模型中。

#### 3.3.3 数据

对齐微调数据收集的要点是收集对模型响应的反馈，即决定哪个响应更好。收集这类数据通常成本更高，此阶段使用的数据量通常甚至少于前几个阶段。在本部分中，我们介绍一些数据集并在表 8 中总结它们。

**LLaVA-RLHF** [112]。它包含 1 万对从关于诚实度和帮助性的人工反馈中收集的偏好对。该数据集主要服务于减少模型响应中的幻觉。

**RLHF-V** [114]。它包含通过片段级幻觉纠正收集的 5700 条细粒度人工反馈数据。

**VLFeedback** [115]。它利用 AI 对模型响应提供反馈。该数据集包含超过 38 万对由 GPT-4V 按帮助性、忠实性和伦理关注评分而得的比较对。

## 4 评估

评估是开发 MLLM 的重要组成部分，因为它为模型优化提供反馈，并帮助比较不同模型的性能。与传统多模态模型的评估方法相比，MLLM 的评估呈现出几个新特征：(1) 由于 MLLM 通常是通用的，对 MLLM 进行全面评估非常重要。(2) MLLM 展现出许多涌现能力，需要特别关注（如无 OCR 的数学推理），因此需要新的评估方案。MLLM 的评估根据问题类型大致可分为两类，包括封闭集（closed-set）和开放集（open-set）。

### 4.1 封闭集

封闭集问题是指一种问题，其中可能的答案选项是预定义的，且限于有限集合。评估通常在任务特定数据集上进行。在这种情况下，响应可以自然地通过基准指标来判断[20], [60], [70], [76], [101], [102], [103], [104]。例如，InstructBLIP[60] 报告了在 ScienceQA[116] 上的准确率，以及在 NoCaps[118] 和 Flickr30K[119] 上的 CIDEr 分数[117]。评估设置通常是零样本[60], [102], [104], [105] 或微调[20], [35], [60], [70], [76], [101], [103], [105]。

第一个设置通常选择覆盖不同一般任务的广泛数据集，并将其分为保留集（held-in）和保留外集（held-out）。在前者上微调后，在后者上评估零样本性能，使用未见过的数据集甚至未见过的任务。相比之下，第二个设置通常用于领域特定任务的评估。例如，LLaVA[20] 和 LLaMA-Adapter[76] 报告了在 ScienceQA[116] 上的微调性能。LLaVA-Med[35] 报告了在生物医学 VQA[120], [121], [122] 上的结果。

上述评估方法通常局限于小范围选定任务或数据集，缺乏全面的定量比较。为此，一些努力旨在开发专门为 MLLM 设计的新基准测试[123], [124], [125], [126], [127], [128], [129]。例如，Fu 等人[123] 构建了一个综合评估基准 MME，总共包含 14 个感知和认知任务。MME 中所有指令-回答对都是手工设计的，以避免数据泄露。MMBench[124] 是一个专门为评估模型能力多个维度而设计的基准，使用 ChatGPT 将开放响应与预定义选择进行匹配。Video-ChatGPT[130] 和 Video-Bench[131] 专注于视频领域，提出了专门的基准和评估工具。也有设计用于评估模型特定方面的评估策略[102]，如 POPE[132] 用于评估幻觉程度。

此外，一些工作[134] 利用更先进的 GPT-4V 模型来评估 MLLM 的性能。例如，Woodpecker[77] 采用 GPT-4V 基于图像来判断模型答案的响应质量。由于 GPT-4V 可以直接访问图像，预期这种评估比使用纯文本 GPT-4 更准确。一种补充方法是通过案例研究比较 MLLM 的不同能力。例如，一些研究评估了两个典型的先进商业模型 GPT-4V 和 Gemini。Yang 等人[135] 通过设计跨越从初步技能（如图像描述和物体计数）到需要世界知识和推理的复杂任务（如理解笑话和作为具身智能体进行室内导航）等各个领域和任务的样本，对 GPT-4V 进行了深入的定性分析。Wen 等人[136] 通过设计针对自动驾驶场景的样本，对 GPT-4V 进行了更有针对性的评估。Fu 等人[137] 通过将模型与 GPT-4V 进行比较，对 Gemini-Pro 进行了全面评估。结果表明，GPT-4V 和 Gemini 表现出相当的视觉推理能力，尽管响应风格不同。

### 4.2 开放集

与封闭集问题相比，开放集问题的响应可以更加灵活，其中 MLLM 通常扮演聊天机器人的角色。由于聊天内容可以是任意的，因此判断起来比封闭集输出更加棘手。评判标准可分为人工评分、GPT 评分和案例研究。人工评分需要人工评估生成的响应。这种方法通常涉及手工设计的问题，旨在评估特定维度。例如，mPLUG-Owl[81] 收集了一个视觉相关的评估集来评判自然图像理解、图表和流程图理解等能力。类似地，GPT4Tools[107] 分别为微调和零样本性能构建了两组评估集，并从思想、动作、参数和整体方面评估响应。

由于人工评估劳动密集，一些研究人员探索了使用 GPT 进行评分，即 GPT 评分。这种方法通常用于评估多模态对话的性能。LLaVA[20] 提出使用纯文本 GPT-4 从不同方面（如帮助性和准确性）对响应进行评分。具体地，从 COCO[133] 验证集中采样 30 张图像，每张图像关联一个简短问题、一个详细问题和一个复杂推理问题（通过 GPT-4 上的自指令生成）。模型和 GPT-4 生成的答案被发送给 GPT-4 进行比较。后续工作遵循这一思路，提示 ChatGPT[81] 或 GPT-4[35], [70], [101], [104], [105] 来评分结果[35], [70], [81], [101], [104] 或判断哪个更好[103]。

应用纯文本 GPT-4 作为评估器的一个主要问题是，评判者仅基于与图像相关的文本内容（如描述或边界框坐标）进行判断，而无法访问图像[35]。因此，在这种情况下将 GPT-4 设为性能上限是值得商榷的。随着 GPT 视觉接口的发布，一些工作[77], [134] 利用更先进的 GPT-4V 模型来评估 MLLM 的性能。例如，Woodpecker[77] 使用 GPT-4V 直接评估锚定在图像中的模型响应。FaithScore[162] 是一种更细粒度的指标，基于将描述性子句子分解并单独评估每个子句子的例程。基于先前的研究，AMBER[163] 是一个无 LLM 的基准，包含判别任务和生成任务，并涉及三种可能的幻觉类型（见 §6.1）。

## 5 扩展

近期研究在扩展 MLLM 的能力方面取得了重大进展，涵盖从更强大的基础能力到更广泛的场景覆盖。我们在此追踪 MLLM 的主要发展方向。

**粒度支持。** 为促进智能体与用户之间更好的交互，研究人员开发了在模型输入和输出粒度方面具有更细粒度支持的 MLLM。在输入方面，支持用户提示更精细控制的模型逐步发展，从图像到区域[28], [138], [139] 甚至像素[29], [140], [141]。具体地，Shikra[28] 支持区域级输入和理解。用户可以通过引用特定区域（以自然语言形式的边界框表示）与助手进行更灵活的交互。Ferret[141] 更进一步，通过设计混合表示方案来支持更灵活的引用。该模型支持不同形式的提示，包括点、框和草图。类似地，Osprey[29] 通过利用分割模型[9] 支持点输入。借助预训练分割模型的非凡能力，Osprey 能够通过单次点击指定单个实体或其一部分。在输出方面，定位能力随着输入支持的发展而提升。Shikra[28] 支持响应以图像中的框标注为基准，从而实现更高的精度和更细的引用体验。LISA[142] 进一步支持掩码级理解和推理，使像素级定位成为可能。

**模态支持。** 增加对模态的支持是 MLLM 研究的一个趋势。一方面，研究人员已探索将 MLLM 适配为支持更多多模态内容的输入，如 3D 点云[41], [143], [144], [145]。另一方面，MLLM 还被扩展为生成更多模态的响应，如图像[32], [146], [147], [148]、音频[32], [147], [149], [150] 和视频[32], [151]。例如，NExT-GPT[32] 提出了一个支持混合模态（具体为文本、图像、音频和视频的组合）输入和输出的框架，借助附加到 MLLM 的扩散模型[152], [153]。该框架采用编码器-解码器架构，并将 LLM 作为理解和推理的中心枢纽。

**语言支持。** 当前模型主要是单语言的，可能是由于高质量非英语训练语料稀缺。一些工作已致力于开发多语言模型，以覆盖更广泛的用户。VisCPM[33] 通过设计多阶段训练方案将模型能力转移到多语言设置。具体地，该方案以英语为枢纽语言，拥有丰富的训练语料。利用预训练的双语 LLM，通过在指令微调期间添加一些翻译样本来将多模态能力转移到中文。采用类似的方法，Qwen-VL[34] 从双语 LLM Qwen[58] 开发而来，同时支持中文和英文。在预训练期间，中文数据被混合到训练语料中，占整体数据量的 22.7%。

**场景/任务扩展。** 除了开发常见的通用助手之外，一些研究聚焦于更具体的场景，在这些场景中需要考虑实际条件，而其他研究将 MLLM 扩展到具有特定专业知识的下游任务。一个典型趋势是将 MLLM 适配为更具体的现实生活场景。MobileVLM[63] 探索了为资源受限场景开发小型 MLLM 变体。采用了一些设计和技巧以便在移动设备上部署，如更小尺寸的 LLM 和量化技术以加速计算。其他工作开发了与现实世界交互的智能体[41], [154], [155]，例如专门为图形用户界面（GUI）设计的用户友好助手，如 CogAgent[44]、AppAgent[43] 和 Mobile-Agent[45]。这些助手在规划和指导完成用户指定任务的每一步方面表现出色，充当人类-机器交互的有用智能体。另一条路线是为解决不同领域的任务（如文档理解[38], [39], [156], [157] 和医学领域[35], [36], [37]）赋予 MLLM 特定技能。对于文档理解，mPLUG-DocOwl[38] 利用各种形式的文档级数据进行微调，从而在无 OCR 的文档理解方面增强了模型。TextMonkey[39] 结合了与文档理解相关的多个任务以提高模型性能。除传统的文档图像和场景文本数据集外，还添加了位置相关任务以减少幻觉，帮助模型学会将响应锚定在视觉信息中。MLLM 还可以通过注入医学领域知识来扩展到医学领域。例如，LLaVA-Med[158] 将医学知识注入原始 LLaVA[20]，开发了一个专门用于医学图像理解和问答的助手。

## 6 多模态幻觉

多模态幻觉是指 MLLM 生成的响应与图像内容不一致的现象。尽管幻觉在 LLM 中也有出现，但多模态幻觉更复杂，因为输入和推理过程涉及多种模态。具体地，MLLM 需要同时理解视觉内容和文本指令，但在生成过程中可能倾向于语言先验而非视觉内容。在生成阶段，LLM 可能基于训练语料中的统计偏见或强语言先验"想象"出图像中不存在的物体。这可能导致看似合理但不正确的响应。此外，模型可能在不存在物体时声称其存在，或错误描述物体的属性或关系。这些问题在视觉问答和描述任务中尤为明显，因为模型需要同时处理图像内容和文本查询。多模态幻觉问题的严重性已被广泛认可，因为 MLLM 的安全性和可靠性依赖于它们生成与图像内容一致响应的能力[77]。作为一个基础且重要的问题，该问题受到了越来越多的关注。在本节中，我们简要介绍一些相关概念和研究进展。

### 6.1 预备知识

当前关于多模态幻觉的研究可进一步分为三种类型[159]：

1) **存在幻觉（Existence Hallucination）** 是最基本的形式，指模型错误地声称图像中存在某些物体。
2) **属性幻觉（Attribute Hallucination）** 指以错误方式描述某些物体的属性，例如未能正确识别狗的颜色。它通常与存在幻觉相关联，因为属性的描述应锚定在图像中存在的物体上。
3) **关系幻觉（Relationship Hallucination）** 是一种更复杂的类型，也基于物体的存在。它指对物体之间关系的错误描述，如相对位置和相互作用。

下面，我们首先介绍一些特定的评估方法（§6.2），这些方法可用于衡量缓解幻觉方法（§6.3）的性能。然后，我们将详细讨论当前减少幻觉的方法，按每个方法所属的主要类别进行分类。

### 6.2 评估方法

CHAIR[160] 是一种早期的指标，用于评估开放集描述中的幻觉水平。该指标衡量包含幻觉物体或幻觉物体的句子在所有提到的物体中所占的比例。相比之下，POPE[132] 是一种评估封闭集选择的方法。具体地，制定多个带有二元选择的提示，每个提示询问图像中是否存在特定物体。该方法还涵盖更具挑战性的设置以评估 MLLM 的鲁棒性，同时考虑数据统计。最终评估使用简单的关键词机制，即通过检测关键词"是/否"，将开放集响应转换为封闭集二元选择。采用类似的评估方法，MME[123] 提供了更全面的评估，涵盖存在性、数量、位置和颜色等方面，如[77]所示。

与之前使用匹配机制检测和决定幻觉的方法不同，HaELM[161] 提出使用纯文本 LLM 作为评判者，自动判断 MLLM 的描述是否与参考描述一致。考虑到纯文本 LLM 只能访问有限的图像上下文并需要参考标注，Woodpecker[77] 使用 GPT-4V 直接评估锚定在图像中的模型响应。FaithScore[162] 是一种更细粒度的指标，基于将描述性子句子分解并单独评估每个子句子的例程。基于先前的研究，AMBER[163] 是一个无 LLM 的基准，包含判别任务和生成任务，并涉及三种可能的幻觉类型（见 §6.1）。

### 6.3 缓解方法

根据高层次思想，当前方法大致可分为三类：预修正（pre-correction）、过程中修正（in-process-correction）和后修正（post-correction）。

**预修正。** 幻觉的一个直观且直接的解决方案是收集专门的数据（如负数据）并使用这些数据进行微调，从而产生幻觉响应更少的模型。

LRV-Instruction[164] 引入了一个视觉指令微调数据集。除了常见的正面指令外，该数据集还纳入了在不同语义层级精心设计的负面指令，以鼓励响应忠于图像内容。LLaVA-RLHF[112] 收集人类偏好对，并使用强化学习技术微调模型，使模型与幻觉更少的答案更加对齐。

**过程中修正。** 另一条路线是在架构设计或特征表示方面进行改进。这些工作试图探索幻觉的原因并设计相应的 remedies，在生成过程中减轻幻觉。

HallE-Switch[159] 对物体存在幻觉的可能因素进行了实证分析，并假设存在幻觉源于未被视觉编码器锚定的物体，它们实际上是基于嵌入在 LLM 中的知识推断的。基于这一假设，引入了一个连续控制因子和相应的训练方案，在推理期间控制模型输出中想象的程度。

VCD[165] 指出物体幻觉源于两个主要原因，即训练语料中的统计偏见和嵌入在 LLM 中的强语言先验。作者注意到，当向图像注入噪声时，MLLM 倾向于在响应生成中偏向语言先验而非图像内容，导致幻觉。相应地，该工作设计了一种"放大后对比"（amplify-then-contrast）解码方案来抵消错误的偏差。

HACL[166] 研究视觉和语言的嵌入空间。基于观察，设计了一种对比学习方案，将配对的跨模态表征拉近，同时将非幻觉和幻觉的文本表征推远。

**后修正。** 与之前的范式不同，后修正以事后补救的方式减轻幻觉，并在输出生成后纠正幻觉。Woodpecker[77] 是一个无需训练的通用幻觉纠正框架。具体地，该方法结合专家模型来补充图像的上下文信息，并设计了一个逐步纠正幻觉的流程。该方法具有可解释性，因为每个步骤的中间结果可以被检查，且物体被锚定在图像中。另一种方法 LURE[167] 训练一个专门的修正器，以屏蔽描述中高不确定性的物体并重新生成响应。

## 7 扩展技术

### 7.1 多模态上下文学习

ICL 是 LLM 的重要涌现能力之一。ICL 有两个良好的特征：(1) 与传统的有监督学习范式从大量数据中学习隐式模式不同，ICL 的关键是从类比中学习[168]。具体地，在 ICL 设置中，LLM 从一些示例和可选指令中学习，并将知识外推到新问题，从而以少样本方式解决复杂和未见过的任务[22], [169], [170]。(2) ICL 通常以无需训练的方式实现[168]，因此可以灵活地集成到推理阶段的不同框架中。与 ICL 密切相关的一项技术是指令微调（见 §3.2），实证表明指令微调可以增强 ICL 能力[19]。

在 MLLM 的背景下，ICL 已被扩展到更多模态，从而产生了多模态 ICL（M-ICL）。基于（§3.2）中的设置，在推理时，M-ICL 可以通过将一组演示样本（即一组上下文样本）添加到原始样本来实现。在这种情况下，模板可以扩展为表 9 所示。注意，我们列出了两个上下文示例以供说明，但示例的数量和顺序可以灵活调整。事实上，模型通常对演示的排列顺序很敏感[168], [171]。

#### 7.1.1 对 ICL 能力的改进

近期，越来越多的工作集中在各种场景下增强 ICL 性能。在本节中，我们追踪该领域的发展并总结一些相关工作。

MIMIC-IT[172] 通过构建一个用多模态上下文格式化的指令数据集，将上下文学习与指令微调相结合。在引入的数据集上指令微调的模型在描述任务中表现出改进的少样本性能。Emu[173] 通过在模型生成和相应的训练语料中引入额外模态，扩展了 Flamingo[74] 的思路。借助引入的视觉解码器（即 Stable Diffusion），模型从额外的视觉监督中学习，并支持输出格式和上下文推理中更大的灵活性。具体地，除纯文本回答外，模型还可以以图像形式给出响应。Sheng 等人[174] 采用了类似的思路，尝试将输出模态扩展到文本和图像两者。该工作没有采用专门的图像编码器，而是采用了一种统一的量化方案和共享嵌入层。

其他一些工作探索了在特定设置下改进少样本学习性能。Link-context 学习[175] 侧重于加强图像-标签对之间的因果联系，通过制定正负图像-描述对来进行对比训练。MMICL[176] 旨在增强多张相关图像的推理能力。为加强图像和文本之间的联系，该工作提出了一种上下文方案，将交错的图像-文本数据转换为统一格式。Jeong[177] 发现，当插入一小部分不连贯的图像/文本作为噪声时，MLLM 可能被误导给出与上下文不一致的响应。基于这一观察，该工作相应地提出了一种预过滤方法，以移除不相关的上下文并促进更连贯的响应。

#### 7.1.2 应用

在多模态方面，M-ICL 主要用于两个场景：(1) 解决各种视觉推理任务[22], [74], [178], [179], [180]；(2) 教会 LLM 使用外部工具[169], [170], [181]。前者通常涉及从一些任务特定的示例中学习，并泛化到新的但类似的问题。从指令和演示中提供的信息中，LLM 了解到任务在做什么、输出模板是什么，并最终生成期望的答案。相比之下，工具使用的示例更加精细。它们通常包含一系列可按顺序执行的步骤来完成任务。因此，第二个场景与 CoT（见 §7.2）密切相关。

### 7.2 多模态思维链

正如开创性工作[8] 所指出的，CoT 是"一系列中间推理步骤"，已被证明在复杂的推理任务中是有效的[8], [182], [183]。CoT 的主要思想是提示 LLM 不仅输出最终答案，还输出导致答案的推理过程，类似于人类的认知过程。

受 NLP 中成功的启发，多项工作[184], [185], [186], [187] 已提出将单模态 CoT 扩展到多模态 CoT（M-CoT）。我们首先介绍获取 M-CoT 能力的不同范式（§7.2.1）。然后，我们描述 M-CoT 的更具体方面，包括链配置（§7.2.2）和生成模式（§7.2.3）。

#### 7.2.1 学习范式

学习范式也是一个值得研究的方面。获取 M-CoT 能力大致有三种方式，即通过微调和无需训练的少样本/零样本学习。这三种方式的样本量需求依次递减。

直观地，微调方法通常涉及为 M-CoT 学习策划特定的数据集。例如，Lu 等人[116] 构建了一个包含讲座和解释的科学问答数据集 ScienceQA，可以作为学习 CoT 推理的来源，并微调模型。

除微调外，少/零样本学习在计算上更高效。它们之间的主要区别在于，少样本学习通常需要手工制作一些上下文示例，以便模型更容易学会逐步推理。相比之下，零样本学习不需要任何特定的 CoT 学习示例。在这种情况下，模型通过嵌入知识和推理能力来学习，无需显式指导，只需通过提示设计的指令如"让我们逐帧思考"或"这两帧之间发生了什么"[184], [186]。类似地，一些工作[22], [188] 用任务描述和工具使用的描述来提示模型，将复杂任务分解为子任务。

#### 7.2.2 链配置

结构和长度是推理链的两个关键方面。就结构而言，当前方法可分为单链（single-chain）和树形（tree-shape）方法。单链推理是一种在各种方法中广泛使用的范式[116], [185]。具体地，逐步推理过程形成单一的"问题-推理-答案"链。最近，一些方法探索了使用更复杂的方案，即树形链，进行推理。具体地，DDCoT[189] 将一个问题分解为多个子问题，每个子问题由 LLM 本身或视觉专家解决以生成推理过程。然后 LLM 聚合推理过程并进行推理以形成最终答案。就链长度而言，可分为自适应和预定义两种配置。前者配置要求 LLM 自行决定何时停止推理链[22], [116], [169], [170], [185], [188]，而后者设置以预定义的长度停止链[79], [184], [186], [187]。

#### 7.2.3 生成模式

链如何构建是一个值得研究的问题。我们将当前工作归纳为 (1) 基于填充（infilling-based）的模式和 (2) 基于预测（predicting-based）的模式。具体地，基于填充的模式要求在周围上下文（前一步和后一步）之间推断步骤以填补逻辑空白[184], [186]。相比之下，基于预测的模式要求在指令和先前推理历史等条件下扩展推理链[22], [116], [169], [170], [185], [188]。这两种类型的模式都有一个共同要求，即生成的步骤应一致且正确。

### 7.3 基于 LLM 的视觉推理

#### 7.3.1 概述

受工具增强 LLM 成功[190], [191], [192], [193] 的启发，一些研究探索了调用外部工具[22], [107], [169], [170] 或视觉基础模型[22], [79], [80], [188], [194], [195], [196] 进行视觉推理任务的可能性。以 LLM 作为具有不同角色的助手，这些工作构建了任务特定[79], [197], [198] 或通用[22], [169], [170], [181], [188] 的视觉推理系统。

与传统视觉推理模型[199], [200], [201] 相比，这些工作表现出几个良好的特征：(1) **强大的泛化能力。** 借助大规模预训练中学习的丰富开放世界知识，这些系统可以轻松泛化到未见过的对象或概念，表现出显著的零/少样本性能[169], [170], [195], [197], [198], [202]。(2) **涌现能力。** 借助 LLM 强大的推理能力，这些系统可以执行复杂的任务。例如，给定一张图像，MM-REACT[22] 可以解释表面之下的含义，如解释为什么一个表情包有趣。(3) **更好的交互性和可控性。** 传统模型通常允许有限的控制机制，并经常涉及昂贵的精心策划数据集[203], [204]。相比之下，基于 LLM 的系统具有以用户友好界面（如点击和自然语言查询）进行精细控制的能力[79]。

对于这部分，我们首先介绍构建基于 LLM 的视觉推理系统所采用的不同训练范式（§7.3.2）。然后，我们深入探讨 LLM 在这些系统中扮演的主要角色（§7.3.3）。

#### 7.3.2 训练范式

根据训练范式，基于 LLM 的视觉推理系统可分为两种类型，即无需训练（training-free）和微调（finetuning）。

**无需训练。** 凭借预训练 LLM 中存储的丰富先验知识，一种直观且简单的方式是冻结预训练模型，直接提示 LLM 来完成各种需求。根据设置，推理系统可进一步分为少样本模型[22], [169], [170], [181] 和零样本模型[79], [197]。少样本模型需要少量手工制作的上下文样本（见 §7.1）来引导 LLM 生成程序或执行步骤序列。这些程序或执行步骤作为对应基础模型或外部工具/模块的指令。零样本模型更进一步，直接利用 LLM 的语言/语义知识或推理能力。例如，PointCLIP V2[197] 提示 GPT-3 生成具有 3D 相关语义的描述，以便与相应图像更好地对齐。在 CAT[79] 中，LLM 被指示根据用户查询优化描述。

**微调。** 一些工作采用进一步的微调来改善关于工具使用[107] 或系统的定位能力[142], [205] 的规划能力。例如，GPT4Tools[107] 引入指令微调方法（见 §3.2）。相应地，收集了一个新的工具相关指令数据集，并用于微调模型。

#### 7.3.3 功能

为进一步考察 LLM 在基于 LLM 的视觉推理系统中究竟扮演什么角色，现有相关工作分为三种类型：
- **LLM 作为控制器（Controller）**
- **LLM 作为决策者（Decision Maker）**
- **LLM 作为语义精炼器（Semantics Refiner）**

前两种角色与 CoT 相关（见 §7.2）。它们经常被使用，因为复杂任务需要分解为中间更简单的步骤。当 LLM 作为控制器时，系统通常在一轮中完成任务，而多轮在决策者情况下更为常见。我们在以下部分描述 LLM 如何扮演这些角色。

**LLM 作为控制器。** 在这种情况下，LLM 充当中央控制器，(1) 将复杂任务分解为更简单的子任务/步骤，(2) 将这些任务分配给适当的工具/模块。第一步通常通过利用 LLM 的 CoT 能力来完成。具体地，明确提示 LLM 输出任务规划[181]，或更直接地，要调用的模块[107], [169], [170]。例如，VisProg[170] 提示 GPT-3 输出视觉程序，其中每行程序调用一个模块来执行子任务。此外，LLM 需要输出模块输入的参数名称。为处理这些复杂要求，使用了一些手工制作的上下文示例作为参考[169], [170], [181]。这与推理链的优化密切相关（见 §7.2），或更具体地说，与最小到最多提示（least-to-most prompting）[206] 技术有关。通过这种方式，复杂问题被分解为依次解决的子问题。

**LLM 作为决策者。** 在这种情况下，复杂任务以多轮方式解决，通常是迭代式的[195]。决策者通常履行以下职责：(1) 总结当前上下文和历史信息，并判断当前步骤可用的信息是否足以回答问题或完成任务；(2) 组织和总结答案，以用户友好的方式呈现。

**LLM 作为语义精炼器。** 当 LLM 被用作语义精炼器时，研究人员主要利用其丰富的语言和语义知识。具体地，通常指示 LLM 将信息整合为一致、流畅的自然语言句子[202]，或根据不同特定需求生成文本[79], [197], [198]。

## 8 挑战与未来方向

MLLM 的发展仍处于初级阶段，因此有较大的改进空间，我们总结如下：

- **当前 MLLM 在处理长上下文的多模态信息方面有限。** 这限制了具有更多多模态词元的先进模型的发展，例如长视频理解和图像与文本交错的长文档。
- **MLLM 应升级以遵循更复杂的指令。** 例如，生成高质量问答对数据的主流方法仍然是提示闭源的 GPT-4V，因为它具有先进的指令遵循能力，而其他模型通常无法达到。
- **M-ICL 和 M-CoT 等技术仍有很大的改进空间。** 目前关于这两项技术的研究仍处于初级阶段，MLLM 的相关能力较弱。因此，对底层机制和潜在改进的探索是很有前景的。
- **开发基于 MLLM 的具身智能体是一个热门话题。** 开发能够与现实世界交互的此类智能体是有意义的。这些努力需要具有关键能力的模型，包括感知、推理、规划和执行。
- **安全问题。** 与 LLM 类似，MLLM 可能容易受到精心设计的攻击[177], [207], [208]。换句话说，MLLM 可能被误导输出有偏见的或不期望的响应。因此，提高模型安全性将是一个重要课题。

## 训练策略与数据

一个成熟的多模态大语言模型（Multimodal Large Language Model, MLLM）需要经历三个阶段的有监督训练，即预训练（pre-training）、指令微调（instruction-tuning）和对齐微调（alignment tuning）。每个训练阶段需要不同类型的数据并实现不同的目标。在本节中，我们讨论各训练阶段的训练目标、数据收集及数据特性。

### 3.1 预训练

#### 3.1.1 训练细节

作为第一个训练阶段，预训练的主要目标是（1）对齐不同模态并（2）学习多模态世界知识。预训练阶段通常需要大规模文本配对数据，例如描述性文本（caption）数据。通常，描述性文本配对以自然语言句子描述图像/音频/视频。在此，我们考虑一个MLLM被训练以对齐视觉与文本的常见场景。如表3所示，给定一张图像，模型按照标准交叉熵损失被训练以自回归（autoregressively）地预测该图像的描述性文本。预训练的一种常见方法是冻结预训练模块（例如视觉编码器（visual encoders）和大语言模型（Large Language Models, LLMs）），并训练一个可学习的接口（learnable interface）[20]、[35]、[72]。其理念是在不损失预训练知识的前提下对齐不同模态。一些方法[34]、[81]、[82]也会解冻更多模块（例如视觉编码器），以实现更多可训练参数的对齐。需要注意的是，训练方案与数据质量密切相关。对于短且有噪声的描述性文本数据，可以采用较低的分辨率（例如224）来加速训练过程；而对于更长、更干净的数据，最好使用更高的分辨率（例如448或更高）来减轻幻觉（hallucinations）。此外，ShareGPT4V [83]发现，在预训练阶段使用高质量的描述性文本数据，并解锁视觉编码器（unlocking the vision encoder），可以促进更好的对齐。

### 3.1.2 数据

预训练数据主要服务于两个目的：（1）对齐不同模态和（2）提供世界知识。预训练语料库可以按粒度分为粗粒度（coarse-grained）和细粒度（fine-grained）数据，我们将依次介绍。我们在表4中总结了常用的预训练数据集。

粗粒度描述性文本数据通常具有一些典型特征：（1）数据量大，因为样本通常来源于互联网；（2）由于网络爬取的性质，描述文本通常较短且噪声较多，因为它们源自网络图像的替代文本（alt-text）。这些数据可以通过自动化工具进行清洗和过滤，例如使用CLIP [13]模型来过滤掉相似度低于预设阈值的图像-文本对。接下来，我们介绍一些具有代表性的粗粒度数据集。

**CC（Conceptual Captions）数据集**。CC-3M [84]是一个包含330万（3.3M）图像-文本对的网络规模描述性文本数据集，其中原始描述性文本来源于与图像关联的替代文本。作者设计了一个复杂的数据清洗流程：（1）对于图像，过滤掉包含不当内容或长宽比不合适的样本；（2）对于文本，使用自然语言处理（NLP）工具获取文本标注，并根据设计的启发式规则过滤样本；（3）对于图像-文本对，通过分类器为图像分配标签。如果文本标注与图像标签不重叠，则剔除相应样本。

**CC-12M [85]** 是CC-3M的后续工作，包含1240万（12.4M）图像-文本对。与之前的工作相比，CC-12M放宽并简化了数据收集流程，从而收集了更多的数据。

**SBU Captions [86]**。这是一个包含100万（1M）图像-文本对的带描述照片数据集，图像和描述均来源于Flickr。具体来说，初始图像集是通过使用大量查询词在Flickr网站上查询获取的。因此，附加在图像上的描述作为描述性文本。然后，为确保描述与图像相关，保留的图像需满足以下要求：（1）图像的描述具有令人满意的长度，由观察决定；（2）图像描述包含预定义术语列表中至少2个词，以及一个通常暗示空间关系的介词性词语（例如"on"、"under"）。

**LAION数据集系列**。该系列是大规模网络级数据集，图像从互联网上爬取，关联的替代文本作为描述。为了过滤图像-文本对，执行以下步骤：（1）删除文本过短或图像尺寸过大或过小的样本；（2）基于URL进行图像去重；（3）提取图像和文本的CLIP [13]嵌入（embeddings），并使用这些嵌入来删除可能非法内容以及嵌入之间余弦相似度（cosine similarity）较低的图像-文本对。以下是几种典型变体的简要介绍：

- **LAION-5B [87]**：这是一个包含58.5亿（5.85B）图像-文本对的研究用途数据集。该数据集为多语言数据集，包含一个20亿（2B）英语子集。
- **LAION-COCO [88]**：它包含6亿（600M）从LAION-5B英语子集中提取的图像。描述性文本是合成的，使用BLIP [89]生成多种图像描述，并使用CLIP [13]选择最符合图像的描述。

**COYO-700M [90]**。它包含7.47亿（747M）图像-文本对，从CommonCrawl中提取。对于数据过滤，作者设计了以下策略：（1）对于图像，过滤掉尺寸、内容、格式或长宽比不合适的图像。此外，基于pHash值过滤图像，以移除与ImageNet和MS-COCO等公共数据集重叠的图像；（2）对于文本，仅保留具有满意长度、名词形式且用词恰当的中文文本。删除句子前后的空白，并将连续的空白字符替换为单个空白。此外，出现超过10次的文本（例如"image for"）将被删除；（3）对于图像-文本对，基于（图像pHash，文本）元组删除重复样本。

最近，更多工作[83]、[91]、[92]探索了通过提示（prompting）强大的MLLM（例如GPT-4V）生成高质量细粒度数据。与粗粒度数据相比，这些数据通常包含对图像更长、更准确的描述，从而实现图像与文本模态之间更细粒度的对齐。然而，由于这种方法通常需要调用商业用途的MLLM，成本较高，且数据量相对较小。值得注意的是，ShareGPT4V [83]通过先用GPT-4V生成的10万（100K）数据训练一个描述生成器，然后使用预训练的描述生成器将数据量扩展到120万（1.2M），从而取得了平衡。

## 3.2 指令微调

### 3.2.1 引言

指令（Instruction）是指对任务的描述。直观地，指令微调旨在教会模型更好地理解来自用户的指令并完成要求的任务。以这种方式进行微调，LLM可以通过遵循新指令来推广到未见过的任务，从而提升零样本（zero-shot）性能。这个简单而有效的想法引发了后续自然语言处理（Natural Language Processing, NLP）工作的成功，例如ChatGPT [2]、InstructGPT [95]、FLAN [19]、[56]和OPT-IML [96]。

指令微调与相关典型学习范式的比较如图3所示。有监督微调（supervised fine-tuning）方法通常需要大量任务特定数据来训练任务特定模型。提示（prompting）方法减少了对大规模数据的依赖，并可以通过提示工程（prompt engineering）完成特定任务。在这种情况下，尽管少样本（few-shot）性能得到了提高，但零样本性能仍然相当平均[7]。不同地，指令微调学习如何推广到未见过的任务，而不是像另外两种方法那样拟合特定任务。此外，指令微调与多任务提示（multi-task prompting）[97]密切相关。

在本节中，我们描述指令样本的格式、训练目标、收集指令数据的典型方式以及相应的常用数据集。

### 3.2.2 训练细节

多模态指令样本通常包括一个可选的指令和一个输入-输出对。指令通常是描述任务的自然语言句子，例如"详细描述这张图像"。输入可以是如图像-文本对（如视觉问答（Visual Question Answering, VQA）任务[99]），也可以仅是图像，输出是给定输入条件下对指令的回答（answer）。

形式化地，一个多模态指令样本可以用三元组形式表示，即 $(I, M, R)$，其中 $I$、$M$、$R$ 分别代表指令、多模态输入和真实回答（ground truth response）。MLLM在给定指令和多模态输入的情况下预测一个回答：

$$A = f(I, M; \theta) \quad (1)$$

这里，$A$ 表示预测的回答，$\theta$ 是模型的参数。训练目标通常是训练LLM时使用的原始自回归（auto-regressive）目标[20]、[37]、[71]、[101]，基于该目标，MLLM被鼓励预测回答的下一个词元（token）。该目标可以表示为：

$$\mathcal{L}(\theta) = -\sum_{i=1}^{N} \log p(R_i \mid I, R_{<i}; \theta) \quad (2)$$

其中 $N$ 是真实回答的长度。

### 3.2.3 数据收集

由于指令数据格式更灵活、任务形式更加多样化，收集数据样本通常更加复杂且成本更高。在本节中，我们总结了三种大规模获取指令数据的典型方式，即数据适配（data adaptation）、自指令生成（self-instruction）和数据混合（data mixture）。

**数据适配**。任务特定的数据集是高质量数据的重要来源。因此，大量工作[60]、[70]、[76]、[82]、[101]、[102]、[103]、[104]利用现有的高质量数据集来构建指令格式的数据集。以VQA数据集的转换为例，原始样本是一个输入-输出对，其中输入包含图像和自然语言问题，输出是基于图像条件对问题的文本回答。这些数据集的输入-输出对可以自然地构成指令样本的多模态输入和回答（见§3.2.2）。指令即任务的描述，可以来自人工设计或由GPT辅助的半自动生成。具体来说，一些工作[21]、[35]、[60]、[70]、[102]、[105]手工制作一个候选指令池，并在训练期间从中采样。我们在表6中给出了VQA数据集的指令模板示例。其他工作手工设计一些种子指令（seed instructions），并用这些指令提示GPT生成更多指令[25]、[82]、[98]。

需要注意的是，由于现有VQA和描述性文本数据集的回答通常比较简洁，直接使用这些数据集进行指令微调可能会限制MLLM的输出长度。处理这个问题有两种常见策略。第一种是在指令中明确说明。例如，ChatBridge [104]对短答案数据明确声明"简短和简洁"，对传统粗粒度描述性文本数据声明"一句话和单句"。第二种是扩展现有回答的长度[105]。例如，M3 IT [105]提出通过使用原始问题、回答和图像的上下文信息（例如描述和光学字符识别（Optical Character Recognition, OCR））提示ChatGPT来重写原始回答。

**自指令生成（Self-Instruction）**。尽管现有的多任务数据集可以提供丰富的数据来源，但它们通常不能很好地满足现实世界场景中的人类需求，例如多轮对话（multi-round conversations）。为了解决这个问题，一些工作通过自指令生成[106]收集样本，该方法利用大语言模型（LLM）使用少量人工标注的样本来生成文本形式的指令遵循数据。具体来说，一些指令遵循样本被手工制作作为演示，之后用这些演示作为指导来提示ChatGPT/GPT-4生成更多指令样本。LLaVA [20]将这种方法扩展到多模态领域，通过将图像转换为描述性文本和边界框（bounding box）文本，并提示纯文本的GPT-4根据要求和演示的指导生成新数据。通过这种方式，构建了一个多模态指令数据集，称为LLaVA-Instruct-150k。遵循这一思路，后续工作如MiniGPT-4 [21]、ChatBridge [104]、GPT4Tools [107]和DetGPT [72]开发了针对不同需求的不同数据集。最近，随着更强大的多模态模型GPT-4V的发布，许多工作已采用GPT-4V生成更高质量的数据，例如LVIS-Instruct4V [91]和ALLaVA [92]。我们在表7中总结了通过自指令生成生成的流行数据集。

**数据混合**。除了多模态指令数据外，仅文本的用户-助手对话数据也可用于提升对话能力和指令遵循能力[81]、[98]、[101]、[103]。LaVIN [101]通过从仅文本和多模态数据中随机采样来直接构造一个小批量（minibatch）。MultiInstruct [102]探索了使用单模态和多模态数据融合进行训练的不同策略，包括混合指令微调（混合两种类型的数据并随机打乱）和顺序指令微调（先文本数据后多模态数据）。

### 3.2.4 数据质量

最近的研究揭示，指令微调样本的数据质量不亚于数据数量。Lynx [73]发现，在大规模但噪声较大的图像-文本对上预训练的模型，其性能不如在更小但更干净的数据集上预训练的模型。同样，Wei等人[108]发现，较少但质量更高的指令微调数据可以实现更好的性能。对于数据过滤，该工作提出了一些评估数据质量的指标，以及相应的方法来自动过滤低质量的视觉-语言数据。在此，我们讨论两个与数据质量相关的重要方面。

**提示多样性（Prompt Diversity）**。指令的多样性已被发现对模型性能至关重要。Lynx [73]经验性地验证了多样化的提示有助于提升模型性能和泛化能力。

**任务覆盖度（Task Coverage）**。就训练数据中涉及的任务而言，Du等人[109]进行了一项实证研究，发现视觉推理任务优于描述生成（captioning）和问答（QA）任务，能更有效地提升模型性能。此外，该研究表明，增强指令的复杂性可能比增加任务多样性和引入细粒度空间标注更有益。

## 3.3 对齐微调

### 3.3.1 引言

对齐微调（Alignment tuning）更多地用于需要将模型与特定人类偏好对齐的场景，例如减少幻觉（hallucinations）的回答（见§6）。目前，基于人类反馈的强化学习（Reinforcement Learning with Human Feedback, RLHF）和直接偏好优化（Direct Preference Optimization, DPO）是对齐微调的两项主要技术。在本节中，我们依次介绍这两项技术的主要思想，并提供一些它们如何被用于解决实际问题的示例，最后给出相关数据集的汇总。

### 3.3.2 训练细节

**基于人类反馈的强化学习（RLHF）[110]、[111]**。这项技术旨在利用强化学习算法将LLM与人类偏好对齐，在训练循环中以人工标注作为监督。如InstructGPT [95]所示，RLHF包含三个关键步骤：（1）有监督微调（Supervised Fine-Tuning, SFT）；（2）训练一个奖励模型（reward model）以拟合人类偏好；（3）使用强化学习（reinforcement learning）算法优化LLM，以最大化奖励模型的输出。尽管使用专家模型进行奖励标注是直截了当的，但采用可学习的接口可能不如前者灵活。将外部模态转换为文本会导致信息损失。例如，将视频转换为文本描述会扭曲时空关系[25]。

## 4 评估

评估是多模态大语言模型（Multimodal Large Language Model, MLLM）开发的重要组成部分，因为它为模型优化提供反馈并帮助比较不同模型的性能。与传统多模态模型的评估方法相比，MLLM的评估呈现出若干新特征：（1）由于MLLM通常具有通用性，对其进行全面评估非常重要。（2）MLLM展现出许多涌现能力，需要特别关注（例如无OCR数学推理），因此需要新的评估方案。MLLM的评估按照问题类型大致可分为闭集和开集两类。

### 4.1 闭集

闭集问题是指可能答案选项被预先定义且限制于有限集合内的一类问题。评估通常在特定任务的数据集上进行。在这种情况下，可以通过基准指标自然地评判回复 [20]、[60]、[70]、[76]、[101]、[102]、[103]、[104]。例如，**InstructBLIP [60]** 在 **ScienceQA [116]** 上报告准确率，以及在 **NoCaps [118]** 和 **Flickr30K [119]** 上报告 **CIDEr [117]** 分数。评估设置通常是零样本 [60]、[102]、[104]、[105] 或微调 [20]、[35]、[60]、[70]、[76]、[101]、[103]、[105]。第一种设置通常选取涵盖不同通用任务的广泛数据集，并将其划分为保留集（held-in）和留出集（held-out）。在前者上微调后，在后者（未见过的数据集甚至未见过的任务）上评估零样本性能。相比之下，第二种设置常见于领域特定任务的评估。例如，**LLaVA [20]** 和 **LLaMA-Adapter [76]** 在 **ScienceQA [116]** 上报告微调性能。**LLaVA-Med [35]** 在生物医学视觉问答（Visual Question Answering, VQA）[120]、[121]、[122] 上报告结果。

上述评估方法通常仅限于选取的少数任务或数据集，缺乏全面的定量比较。为此，一些研究致力于开发专门为MLLM设计的新基准 [123]、[124]、[125]、[126]、[127]、[128]、[129]。例如，**Fu et al. [123]** 构建了一个包含总共14项感知和认知任务的综合评估基准 **MME**。MME中的所有指令-答案对均为人工设计，以避免数据泄露。**MMBench [124]** 是专门为评估模型能力多个维度而设计的基准，使用ChatGPT将开放回复与预定义选项进行匹配。**Video-ChatGPT [130]** 和 **Video-Bench [131]** 聚焦视频领域，提出专门的基准和评估工具。还有针对模型特定方面设计的评估策略 [102]，例如用于评估幻觉程度的 **POPE [132]**。

### 4.2 开集

与闭集问题相对，开集问题的回复更加灵活，MLLM通常扮演聊天机器人的角色。由于聊天内容可以是任意的，因此评判比闭集输出更困难。评判标准可分为人工评分、GPT评分和案例研究三类。人工评分需要人工评估生成的回复。这类方法通常涉及手工设计用于评估特定维度的问题。例如，**mPLUG-Owl [81]** 收集了一个与视觉相关的评估集，用于评判自然图像理解、图表和流程图理解等能力。类似地，**GPT4Tools [107]** 分别构建了微调和零样本性能的两套评估集，从思维、动作、参数和整体四个维度评估回复。

由于人工评估劳动密集，一些研究者探索使用GPT进行评分，即GPT评分。这种方法常用于评估多模态对话性能。**LLaVA [20]** 提出通过仅文本的GPT-4从不同方面（如有用性和准确性）对回复进行评分。具体而言，从COCO [133]验证集中采样30张图像，每张图像通过GPT-4上的自指令生成一个简短问题、一个详细问题和一个复杂推理问题。模型和GPT-4生成的回复均被发送给GPT-4进行比较。后续工作沿用这一思路，提示ChatGPT [81]或GPT-4 [35]、[70]、[101]、[104]、[105]对结果进行评分 [35]、[70]、[81]、[101]、[104]或判断哪条更好 [103]。

将仅文本的GPT-4作为评估器的一个主要问题是，评判仅基于图像相关文本内容（如标题或边界框坐标），无法访问图像本身 [35]。因此，在这种情况下将GPT-4设为性能上界可能是有疑问的。随着GPT视觉接口的发布，一些工作 [77]、[134]采用更先进的GPT-4V模型来评估MLLM的性能。例如，**Woodpecker [77]** 采用GPT-4V根据图像评判模型回复的质量。预期这种评估比使用仅文本的GPT-4更准确，因为GPT-4V可以直接访问图像。

一种补充方法是通过案例研究比较MLLM的不同能力。例如，一些研究评估了两个典型的先进商业模型GPT-4V和Gemini。**Yang et al. [135]** 通过设计涵盖不同领域和任务的一系列样本，对GPT-4V进行了深入的定性分析，从标题和物体计数等初步技能到需要世界知识和推理的复杂任务（如笑话理解和作为具身智能体的室内导航）。**Wen et al. [136]** 通过设计针对自动驾驶场景的样本，对GPT-4V进行了更有针对性的评估。**Fu et al. [137]** 通过将模型与GPT-4V进行比较，对Gemini-Pro进行了全面评估。结果表明，尽管回复风格不同，GPT-4V和Gemini展现出可比的视觉推理能力。

## 5 扩展

近期研究在扩展MLLM能力方面取得了显著进展，从更强大的基础能力到更广泛的场景覆盖。本节追踪MLLM在此方面的主要发展脉络。

**粒度支持（Granularity Support）**。为促进智能体与用户之间更好的交互，研究者开发了在模型输入和输出粒度方面提供更细粒度支持的MLLM。在输入方面，支持从用户提示词进行更精细控制的模型逐步发展，从图像发展到区域 [28]、[138]、[139]甚至像素 [29]、[140]、[141]。具体而言，**Shikra [28]** 支持区域级输入和理解。用户可以通过引用特定区域（以自然语言形式的边界框表示）更灵活地与助手交互。**Ferret [141]** 更进一步，通过设计混合表示方案支持更灵活的引用。该模型支持不同形式的提示词，包括点、框和草图。类似地，**Osprey [29]** 通过利用分割模型 [9]支持点输入。借助预训练分割模型的出色能力，Osprey实现了通过单次点击指定单个实体或其一部分。在输出方面，定位能力随输入支持的发展而提升。**Shikra [28]** 支持基于图像边界框标注的回复定位，从而实现更高的精度和更细的引用体验。**LISA [142]** 进一步支持mask级别的理解和推理，使得像素级定位成为可能。

**模态支持（Modality Support）**。增加对模态的支持是MLLM研究的一个趋势。一方面，研究者探索使MLLM支持更多模态内容的输入，如3D点云 [41]、[143]、[144]、[145]。另一方面，MLLM也被扩展为生成更多模态的回复，如图像 [32]、[146]、[147]、[148]、音频 [32]、[147]、[149]、[150]和视频 [32]、[151]。例如，**NExT-GPT [32]** 在附加到MLLM的扩散模型 [152]、[153]的帮助下，提出了一种支持文本、图像、音频和视频混合模态（即这些模态的组合）输入和输出的框架。该框架采用编码器-解码器（encoder-decoder）架构，将大语言模型（Large Language Model, LLM）作为理解和推理的中心枢纽。

**语言支持（Language Support）**。当前模型 predominantly 为单语言模型，可能由于高质量非英语训练语料稀缺。一些工作致力于开发多语言模型以覆盖更广泛的用户。**VisCPM [33]** 通过设计多阶段训练方案将模型能力转移到多语言环境。具体而言，该方案以英语为枢纽语言，因其拥有丰富的训练语料。利用预训练的双语LLM，通过在指令微调期间添加一些翻译样本，将多模态能力转移到中文。采用类似方法，**Qwen-VL [34]** 从双语LLM **Qwen [58]** 开发而来，支持中英双语。在预训练期间，将中文数据混合到训练语料中以保留模型的双语能力，占总体数据量的22.7%。

**场景/任务扩展（Scenario/Task Extension）**。除开发通用的通用助手外，一些研究聚焦于更具体的场景（需考虑实际条件），另一些则将MLLM扩展至具有特定专业知识的下游任务。

一种典型趋势是将MLLM适配到更具体的现实场景。**MobileVLM [63]** 探索为资源受限场景开发小尺寸的MLLM变体。采用一些设计和技巧以部署在移动设备上，如更小尺寸的LLM和量化技术以加速计算。其他工作开发与现实世界交互的智能体 [41]、[154]、[155]，例如专为图形用户界面（Graphical User Interface, GUI）设计的用户友好型助手，以**CogAgent [44]**、**AppAgent [43]**和**Mobile-Agent [45]**为代表。这些助手擅长规划和引导用户完成指定任务的每一步，作为人机交互的有用智能体。另一条路线是为MLLM添加特定技能以解决不同领域的任务，例如文档理解 [38]、[39]、[156]、[157]和医学领域 [35]、[36]、[37]。对于文档理解，**mPLUG-DocOwl [38]** 利用多种形式的文档级数据进行微调，从而在免OCR（OCR-free）文档理解方面增强了模型性能。**TextMonkey [39]** 整合了与文档理解相关的多项任务以提升模型性能。除传统的文档图像和场景文本数据集外，还添加了与位置相关的任务以减少幻觉并帮助模型学习将回复定位在视觉信息中。MLLM也可通过注入医学领域知识扩展到医学领域。例如，**LLaVA-Med [158]** 将医学知识注入到基础版**LLaVA [20]**中，开发了专用于医学图像理解和问答的助手。

## 6 多模态幻觉

多模态幻觉（Multimodal Hallucination）是指MLLM生成的回复与图像内容不一致的现象，尤其是生成的内容在图像中找不到依据 [77]。作为一个基础且重要的问题，该问题受到越来越多的关注。本节简要介绍一些相关概念和研究进展。

### 6.1 预备知识

当前关于多模态幻觉的研究可进一步分为三类 [159]：

1）**存在幻觉（Existence Hallucination）**是最基本的形式，指模型错误地声称图像中存在某些物体。

2）**属性幻觉（Attribute Hallucination）**指以错误方式描述某些物体的属性，例如未能正确识别狗的颜色。由于属性的描述应以图像中存在的物体为依据，因此属性幻觉通常与存在幻觉相关联。

3）**关系幻觉（Relationship Hallucination）**是更复杂的类型，同样基于物体的存在。指对物体之间关系（如相对位置和交互）的错误描述。

下文首先介绍一些具体的评估方法（§6.2），这些方法有助于衡量缓解幻觉方法（§6.3）的性能。然后，我们将根据每种方法所属的主要类别详细讨论当前减少幻觉的方法。

### 6.2 评估方法

**CHAIR [160]** 是一种早期指标，用于评估开放标题中的幻觉程度。该指标测量所有提到的物体中包含幻觉物体或幻觉句子的比例。相比之下，**POPE [132]** 是一种评估闭集选项的方法。具体而言，构建多个带有二元选择的提示词，每个提示词查询特定物体是否存在于图像中。该方法还涵盖更具挑战性的设置以评估MLLM的鲁棒性，并考虑数据统计。最终评估使用简单的关键词机制，即通过检测关键词"yes/no"，将开放回复转换为闭集二元选择。采用类似评估方法，**MME [123]** 提供了更全面的评估，涵盖存在、数量、位置和颜色等方面，如 [77] 中所例示。

与先前使用匹配机制检测和判断幻觉的方法不同，**HaELM [161]** 提出使用仅文本的LLM作为评判器，自动决定MLLM的标题是否与参考标题一致。鉴于仅文本LLM只能访问有限的图像上下文且需要参考标注，**Woodpecker [77]** 使用GPT-4V直接评估基于图像的模型回复。**FaithScore [162]** 是一种更细粒度的指标，基于将描述性子句子拆解并单独评估每个子句子的例程。基于先前研究，**AMBER [163]** 是一个免LLM的基准，涵盖判别任务和生成任务，并涉及三种可能的幻觉类型（见§6.1）。

### 6.3 缓解方法

从高层思路来看，当前方法大致可分为三类：预校正（pre-correction）、过程内校正（in-process-correction）和后校正（post-correction）。

**预校正**。幻觉的一种直观且直接的解决方案是收集专门数据（如负数据）并使用该数据进行微调，从而得到幻觉回复更少的模型。

**LRV-Instruction [164]** 引入了一个视觉指令微调数据集。除常见的正面指令外，该数据集在不同语义层次融入了精心设计的负面指令，以鼓励回复忠实于图像内容。**LLaVA-RLHF [112]** 收集人类偏好对，并使用强化学习技术微调模型，从而得到更对齐且幻觉回复更少的模型。

**过程内校正**。另一条路线是在架构设计或特征表示方面进行改进。这些工作尝试探索幻觉产生的原因并设计相应的 Remedies 以在生成过程中缓解幻觉。

**HallE-Switch [159]** 对物体存在幻觉的可能因素进行了实证分析，并假设存在幻觉源于未被视觉编码器定位的物体，实际上它们是基于嵌入在LLM中的知识推断出来的。基于这一假设，引入了一个连续控制因子及相应的训练方案，在推理过程中控制模型输出中的想象程度。

**VCD [165]** 指出物体幻觉源于两个主要原因，即训练语料中的统计偏差和嵌入在LLM中的强语言先验。作者注意到一种现象：当向图像中注入噪声时，MLLM倾向于偏向语言先验而非图像内容来生成回复，从而导致幻觉。相应地，该工作设计了一种放大-对比解码方案（amplify-then-contrast decoding scheme）来抵消错误的偏差。

**HACL [166]** 研究视觉和语言的嵌入空间。基于观察，设计了一个对比学习方案，将配对的跨模态表示拉近，同时将非幻觉和幻觉文本表示推远。

**后校正**。与前述范式不同，后校正以事后补救的方式缓解幻觉，并在输出生成后纠正幻觉。**Woodpecker [77]** 是一个免训练的通用幻觉校正框架。具体而言，该方法引入专家模型补充图像的上下文信息，并设计了一个逐步校正幻觉的流水线。该方法的可解释性在于可以检查每步的中间结果，并且物体被定位在图像中。另一种方法**LURE [167]**训练一个专门的修正器，对描述中不确定性高的物体进行掩码处理并重新生成回复。

## 7 扩展技术

### 7.1 多模态上下文学习（Multimodal In-Context Learning, M-ICL）

上下文学习（In-Context Learning, ICL）是大语言模型（Large Language Model, LLM）的重要涌现能力之一。ICL有两个优点：（1）与传统监督学习范式（tra-ditional supervised learning paradigms）从大量数据中学习隐式模式不同，ICL的核心是从类比中学习 [168]。具体而言，在ICL设置中，LLM从少量示例以及可选的指令中学习，并外推到新问题，从而以少样本方式解决复杂且未见过的任务 [22]、[169]、[170]。（2）ICL通常在训练免费（training-free）的方式下实现 [168]，因此可以灵活地集成到不同框架的推理阶段。与ICL密切相关的一种技术是指令微调（见§3.2），经验表明指令微调能够增强ICL能力 [19]。

在MLLM的背景下，ICL已扩展到更多模态，从而产生了多模态上下文学习（Multimodal ICL, M-ICL）。在§3.2所述的设置基础上，在推理时，M-ICL可以通过向原始样本添加演示集（即一组上下文样本）来实现。在这种情况下，模板可以扩展为表9所示。注意，我们列出了两个上下文示例以便说明，但示例的数量和顺序可以灵活调整。实际上，模型通常对示例的排列顺序很敏感 [168]、[171]。

#### 7.1.1 对ICL能力的改进

近期，越来越多的工作聚焦于在各种场景下提升ICL性能。本节追踪该领域的发展并总结一些相关作品。

**MIMIC-IT [172]** 通过构建以多模态上下文形式组织的指令数据集，将上下文学习与指令微调相结合。在新引入的数据集上进行指令微调的模型在标题任务中表现出改进的少样本性能。**Emu [173]** 通过在模型生成和相应训练语料中引入额外模态，扩展了**Flamingo [74]**的思路。借助引入的视觉解码器（即Stable Diffusion），该模型从额外的视觉监督中学习，并支持输出格式和上下文推理的更大灵活性。具体而言，除纯文本回复外，该模型还可以图像形式给出回复。**Sheng et al. 其他一些工作探索改进少样本学习。**CoT-PT [187]** 在该提出的数据集上学习隐式的思维链。**Multimodal-CoT [185]** 也使用ScienceQA基准，但以两步方式生成输出，即基于思维链（Chain-of-Thought, CoT）推理步骤和相应的最终答案。

### 7.1.2 应用

在多模态应用中，多模态上下文学习（M-ICL）主要在两个场景中使用：(1) 解决各种视觉推理任务 [22]、[74]、[178]、[179]、[180]；(2) 教授大语言模型（Large Language Model, LLM）使用外部工具 [169]、[170]、[181]。前者通常涉及从少量任务特定的示例中学习，并泛化到新的但相似的问题。从指令和演示中提供的信息中，LLM 能够感知任务在做什么、输出模板是什么，并最终生成预期的答案。相比之下，工具使用的示例更加细粒度，通常包含可顺序执行的步骤链以完成任务。因此，第二个场景与思维链（Chain of Thought, CoT）密切相关（见第 7.2 节）。

### 7.2 多模态思维链（Multimodal Chain of Thought）

如先驱工作 [8] 所述，CoT 是"一系列中间推理步骤"，已被证明在复杂推理任务中有效 [8]、[182]、[183]。CoT 的核心思想是提示 LLM 不仅输出最终答案，还要输出通向该答案的推理过程，这与人类的认知过程相似。

受 NLP 领域成功的启发，多项工作 [184]、[185]、[186]、[187] 已将单模态 CoT 扩展到多模态 CoT（M-CoT）。我们首先介绍获取 M-CoT 能力的不同范式（第 7.2.1 节），然后更具体地阐述 M-CoT 的各个方面，包括链配置（第 7.2.2 节）和生成模式（第 7.2.3 节）。

#### 7.2.1 学习范式

学习范式也是一个值得研究的方面。获取 M-CoT 能力主要有三种方式，即通过微调（finetuning）和免训练的小样本/零样本学习（training-free few/zero-shot learning）。三种方式的样本量需求依次递减。

直观上，微调方法通常需要为 M-CoT 学习整理特定的数据集。例如，Lu 等人 [116] 构建了一个包含讲座和解释的科学问答数据集 ScienceQA，可作为 CoT 推理学习的来源，并据此微调模型。

与微调相比，小样本/零样本学习在计算上更高效。两者的主要区别在于：小样本学习通常需要手工制作一些上下文示例，使模型更容易学会逐步推理；而零样本学习不需要任何特定的 CoT 学习示例。在这种情况下，模型学会利用嵌入的知识和推理能力，而无需通过精心设计的提示指令（如"逐帧思考"或"这两个关键帧之间发生了什么" [184]、[186]）提供明确的指导。类似地，一些工作 [22]、[188] 使用任务描述和工具使用描述来提示模型，将复杂任务分解为子任务。

#### 7.2.2 链配置

结构和长度是推理链的两个关键方面。在结构方面，当前方法可分为单链（single-chain）和树形（tree-shape）方法。单链推理是多种方法中广泛使用的范式 [116]、[185]。具体而言，逐步推理过程形成单一的问题-理由-答案链。最近，一些方法开始探索使用更复杂的树形链进行推理。具体来说，DDCoT [189] 将问题分解为多个子问题，每个子问题由 LLM 本身或视觉专家解决以生成理由，然后 LLM 聚合这些理由并进行推理以形成最终答案。关于链长度，可分为自适应（adaptive）和预定义（pre-defined）两种配置。前者要求 LLM 自行决定何时停止推理链 [22]、[116]、[169]、[170]、[185]、[188]、[188]，而后者使用预定义的链长度停止推理 [79]、[184]、[186]、[187]。

#### 7.2.3 生成模式

链如何构建是一个值得研究的问题。我们将当前工作归纳为以下两类：(1) 基于填充（infilling-based）的模式和 (2) 基于预测（predicting-based）的模式。具体而言，基于填充的模式要求在相邻上下文（前一步和后一步）之间推导步骤以填补逻辑空白 [184]、[186]。相比之下，基于预测的模式要求在给定指令和先前推理历史等条件的情况下扩展推理链 [22]、[116]、[169]、[170]、[185]、[188]。这两种模式都要求生成的步骤具有一致性和正确性。

### 7.3 LLM 辅助视觉推理

#### 7.3.1 引言

受工具增强 LLM（Tool-augmented LLM）[190]、[191]、[192]、[193] 成功的启发，一些研究探索了在视觉推理任务中调用外部工具 [22]、[107]、[169]、[170] 或视觉基础模型 [22]、[79]、[80]、[188]、[194]、[195]、[196] 的可能性。这些工作以 LLM 作为具有不同角色的辅助工具，构建了任务特定 [79]、[197]、[198] 或通用 [22]、[169]、[170]、[181]、[188] 的视觉推理系统。

与传统视觉推理模型 [199]、[200]、[201] 相比，这些工作展现出若干良好特性：(1) 强泛化能力。得益于从大规模预训练中习得的丰富开放世界知识，这些系统能够以出色的零样本/小样本性能轻松泛化到未见过的对象或概念 [169]、[170]、[195]、[197]、[198]、[202]。(2) 涌现能力（Emergent Abilities）。在 LLM 强大推理能力的辅助下，这些系统能够执行复杂任务。例如，给定一张图像，MM-REACT [22] 可以解释表面之下的含义，如解释一个梗图为何有趣。(3) 更好的交互性和可控性。传统模型通常只允许有限的控制机制，往往需要昂贵的整理数据集 [203]、[204]。相比之下，基于 LLM 的系统能够在用户友好的界面（如点击和自然语言查询）中实现精细控制 [79]。

本部分首先介绍构建 LLM 辅助视觉推理系统所采用的不同训练范式（第 7.3.2 节），然后深入探讨 LLM 在这些系统中扮演的主要角色（第 7.3.3 节）。

前两种角色与 CoT 相关（见第 7.2 节）。它们被频繁使用，因为复杂任务需要分解为中间更简单的步骤。当 LLM 作为控制器（Controller）时，系统通常以单轮方式完成任务，而当 LLM 作为决策者（Decision Maker）时，多轮方式更为常见。我们在后续部分阐述 LLM 如何扮演这些角色。

**LLM 作为控制器。** 在这种情况下，LLM 充当中央控制器，负责：(1) 将复杂任务分解为更简单的子任务/步骤；(2) 将这些任务分配给适当的工具/模块。第一步通常通过利用 LLM 的 CoT 能力来完成。具体而言，LLM 被明确提示输出任务规划 [181]，或更直接地输出要调用的模块 [107]、[169]、[170]。例如，VisProg [170] 提示 GPT-3 输出视觉程序，其中每行程序调用一个模块执行子任务。此外，LLM 还需要为模块输入输出参数名称。为了处理这些复杂需求，使用了一些手工制作的上下文示例作为参考 [169]、[170]、[181]。这与推理链的优化（见第 7.2 节）密切相关，更具体地说，与逐步最少提示（least-to-most prompting）[206] 技术相关。通过这种方式，复杂问题被分解为依次解决的子问题。

**LLM 作为决策者。** 在这种情况下，复杂任务以多轮方式解决，通常采用迭代方式 [195]。决策者通常承担以下职责：(1) 总结当前上下文和历史信息，判断当前步骤的可用信息是否足以回答问题或完成任务；(2) 组织和总结答案，以用户友好的方式呈现。

**LLM 作为语义精炼器（Semantics Refiner）。** 当 LLM 作为语义精炼器时，研究人员主要利用其丰富的语言和语义知识。具体而言，LLM 通常被指示将信息整合为一致、流畅的自然语言句子 [202]，或根据不同特定需求生成文本 [79]、[197]、[198]。

### 8 挑战与未来方向（Challenges and Future Directions）

多模态大语言模型（MLLM）的发展仍处于起步阶段，因此有很大的改进空间，我们总结如下：

- 当前 MLLM 在处理长上下文多模态信息方面能力有限。这阻碍了具有更多多模态标记的高级模型的发展，例如长视频理解以及图像与文本交错的长文档处理。
- MLLM 应升级以遵循更复杂的指令。例如，生成高质量问答对数据的主流方法仍是在提示闭源的 GPT-4V，因为其指令遵循能力先进，而其他模型通常无法达到。
- M-ICL 和 M-CoT 等技术仍有很大的改进空间。目前对这两项技术的研究仍处于起步阶段，MLLM 的相关能力也较弱。因此，对其底层机制和潜在改进的探索很有前景。
- 基于 MLLM 开发具身智能体（embodied agents）是一个热门话题。开发能够与真实世界交互的此类智能体是有意义的。此类工作对模型的关键能力有要求，包括感知、推理、规划和执行。
- 安全问题。与 LLM 类似，MLLM 也容易受到精心设计的攻击 [177]、[207]、[208]。换句话说，MLLM 可能被误导而输出有偏见或不良响应。因此，提升模型安全性将是一个重要课题。

### 9 结论（Conclusion）

本文对现有 MLLM 文献进行了综述，对其主要方向提供了广泛概述，包括基础配方和相关扩展。此外，我们强调了当前需要填补的研究空白，并指出了一些有前景的研究方向。希望本综述能为读者呈现 MLLM 的当前进展，并激励更多相关工作。
