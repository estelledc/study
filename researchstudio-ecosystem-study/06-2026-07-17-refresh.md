## 3. 唯一漂移：ppt-master

4 个提交、21 个变化文件集中在模板 authoring 与 native SVG/PPTX transport：

- 增加 model-readable `authoring_summary.json`；
- `authoring_manifest.json` 明确成为 compiler-only provenance；
- native payload 进入 content-addressed compact store；
- model-facing frame/translate 坐标压缩到最多两位小数；
- mirrored group 中的 text 做 counter-reflection，避免字形倒置；
- quality checker 在校验前 hydration native payload refs；
- 模板使用和验证文档补齐。

这不改变 ResearchStudio 的六层生态判断，但强化了传播 artifact 的一个原则：

> 模型读的摘要、编译器读的 provenance、用户看的产物应分层，不能让大机器 manifest
> 淹没模型上下文，也不能为了上下文精简丢掉可恢复证据。

### 4.2 Navigator 与 gate 矛盾

命令：

```bash
.venv311/bin/python \
  ResearchStudio-Idea/skills/idea_spark/scripts/run.py \
  next \
  --dir runs/kvcache-idea \
  --query "efficient long-context LLM inference"
```

输出：

```text
STATE: DONE — all three idea cards rendered.
```

对同一 run 执行官方 validator：

```bash
.venv311/bin/python \
  ResearchStudio-Idea/skills/idea_spark/scripts/run.py \
  validate \
  --phase1 runs/kvcache-idea/phase1/phase1_output.json \
  --phase2 runs/kvcache-idea/phase2_select/phase2_select_output.json \
  --phase3 runs/kvcache-idea/phase3_revise/final_candidate.json \
  --phase4 runs/kvcache-idea/phase4/phase4_expansion.json
```

结果：

```text
1 pass, 1 warn, 1 fail
```

关键 failure：

```text
Phase 2 winner=None not found in candidates list
```

原因是 artifact routing 错误：

- 传入的 `phase2_select_output.json` 只有 `selected_gaps` 等字段；
- kill-switch validator 期待最终 candidate 或 candidates + winner；
- Phase 3 final candidate 与 Phase 4 expansion 的内容本身存在。

因此这应归类为：

```text
contract / routing failure
```

而不是：

```text
scientific hypothesis failure
```

Navigator 只根据文件和阶段路径判断下一步，不能替代阻断 validator。

### 4.3 Paper bundle provenance 缺口

本地 PDF 实际存在：

```text
bytes = 2215244
sha256 = bdfaa68d8984f0dc02beaca527b76f207d99b666d31d1da728ee0728182df697
```

但 `manifest.json` 记录：

```json
{
  "source": {
    "pdf": null,
    "filename": null,
    "bytes": null,
    "sha256": ""
  },
  "extractor": {
    "script": null,
    "command": null,
    "skipped": true
  }
}
```

同时 bundle 中确有 text、metadata、figures、captions、sections 和 narration。

正确结论：

- 资产文件存在；
- 资产可能可用；
- manifest 不能证明它们来自当前 PDF；
- 重建命令、输入 hash 和 extractor version 缺失；
- 不能把它标为 provenance-complete artifact。

## 5. 最小 research run E2

本轮新增纯标准库实验：

```text
evidence -> idea -> experiment -> report
```

每个 artifact 必须记录：

```text
path
sha256
producer
source_artifacts
gate
gate_checks
```

8 个测试验证：

- stage 不能跳过未通过的 prerequisite；
- artifact 必须通过 named checks；
- gate 后篡改会被 hash 检出；
- artifact 只能引用已通过的 source artifact；
- retryable failure 从同一 stage 恢复；
- non-retryable scientific failure 停止 run；
- 文件存在不能直接完成 run；
-完整 provenance chain 才能完成。

输出：

```text
next=retry:idea
evidence_gate=passed
events=7
```

## 6. 四类失败不能混

| 失败类型 | 例子 | 默认动作 |
|---|---|---|
| Contract | schema 不匹配、artifact 路由错 | 修输入/适配，重新过 gate |
| Environment | provider timeout、GPU/账号缺失 | 有界重试或记录 blocker |
| Execution | 代码报错、实验进程失败 | 修代码并重跑同 stage |
| Scientific | 假设被反例或实验否定 | 停止、修订假设或保留负结果 |

把所有失败都交给 LLM “再试一次”，会把科学失败伪装成工程重试，也会浪费预算。

## 7. 稳定结论

1. Artifact 是下一阶段输入，不是输出目录里的任意文件。
2. Navigator 是 UI/调度投影，gate 才是完成权威。
3. Manifest 必须绑定输入 hash、workflow version、producer 和 source chain。
4. LLM reviewer 不能替代 schema、引用、实验、几何和编译门。
5. Degraded connector 必须显式进入 run 状态，不能静默当完整覆盖。
6. Research artifact 和 dissemination artifact 共享 provenance，但有不同专属门。
7. 负实验结果是合法科学产物，不应被 retry loop 自动抹掉。
8. 多 Agent 只扩展搜索，不能自动提高证据质量。
