# CPR 视频标注指南 v1（老师阅读版）

> 配套 schema：`schema/labeling-schema-v1.json`
> 评分依据：`cpr-eval/schema/rubric-v2.md`（8 类 28 步 100 分 / 33 个易错点 E1–E33）

## 一、为什么要标这些数据

我们在做一套 AI 自动评分系统（vea）。AI 需要"参考答案"才能学会判分。
**您标的每一个字段，都是 AI 的老师**——没有这份数据，AI 只能瞎猜。

具体来说，您标的内容会被用来回答三个问题：

1. **这次操作每一步该得几分？**（per-step 评分）
2. **如果扣分，触犯了哪个易错点？**（封闭勾选 E1–E33）
3. **每一步发生在视频的哪一段？**（时间区间）

---

## 二、您要做的 4 件事（按视频从头到尾顺序）

### 1. 给 28 步逐项打分

界面上会列 28 步，每一步显示：
- 该步标准操作（rubric-v2 原文）
- 该步满分（如 D2S5 满分 5 分）
- 该步声明的易错点（如 E13、E14）

您要填：
- **是否做了**（做了 / 没做）
- **得分**（0 / 部分 / 满分。可填具体数字）
- **触犯的易错点**（从下拉框勾，**只能勾该步声明的几个**）
- **每个易错点触发时刻**（点视频条上的时间，约 ±1 秒精度即可）
- **不确定按钮**（拿不准时勾上，不会强迫您选）

### 2. 给 28 步标时间区间

每步在视频里的 [start, end]。AI 会**先填一份草稿**（quanzhiping 项目验证可行），您只需要：
- 拖边界条调整
- 大错就重画

> **小窍门**：D8（总体评价）两步可以填 [0, duration] 全段。

### 3. 标关键时序事件（仅 7 个时序类步骤）

这 7 步分值最大（共 28 分），AI 一定要算频率，所以光有区间不够：

| 步骤 | 您要标的 |
|---|---|
| **D2S5 按压频率** | 第一次按下时刻 / 最后一次按下时刻 / 总次数（估即可） / 中断 >2s 的时段 |
| **D4S2 吹气时间** | 每次口对口吹气的时间点 + 持续秒数（标第一轮 2 次即可，其余 AI 类推） |
| **D7S3 简易呼吸器频率** | 每次挤压球囊的时间点（标前 5 次即可） |
| **D7S4 潮气量** | 与上同（持续秒数关键） |
| **D5S1 30:2 循环** | 5 轮循环每轮的起止（区间即可） |
| **D5S2 复苏时间** | 整段 5 轮的总起止 |
| **D5S3 复苏判断** | 判断动作发生的 [start, end]，应 ≤ 10s |

### 4. 标关键口述（4 句话）

学生**有没有说出**下面 4 句话？说了的话在视频条上点出时间：

| 短语 ID | 大意 | 出现在哪步 |
|---|---|---|
| `ENV_SAFE` | "现场安全" / "周围环境安全" | D1S1 |
| `MODE_MANUAL` | "手动模式" / "切换手动" | D6S1 |
| `ENERGY_200J` | "200 焦" / "选 200J" | D6S1 |
| `ALL_CLEAR` | "所有人离开" / "离开患者" | D6S5 |

> 没说就留空。说了原话不完全一致没关系，意思对就算。

## 三、几个关键约定（请仔细看）

### A. 易错点是封闭勾选，不是自由文本

为什么？AI 需要可统计的标签。如果您觉得 33 个易错点不够用，请在备注里写下来，**我们后续走"申请新增 E34、E35"流程**，不要在勾选时自由发挥。

### B. "不确定"按钮怎么用

边界模糊（如 E10 "手臂没伸直" 该多大角度算？）就勾"不确定"。这步**不会进 AI 训练数据**，但您的犹豫本身是有价值的信号——告诉我们这条易错点定义需要细化。

### C. 同人复测（重要）

您标完 15 个之后约 2 周，会让您**重标其中 3 个**（系统随机选短/中/长各 1 个），且**不会显示您第一次的答案**。
这是为了算"前后一致率"——评估单人标注稳定性的最小代价方案。请在重标时**像第一次一样认真**。

### D. AI 已经预填了一份草稿

进界面您会看到 AI draft（区间、易错点都有）。请：
- **草稿对的，您就跳过**（节省时间）
- **草稿错的，您直接改**（覆盖即可）

您和 AI 改动差异越大的步骤 = AI 越弱的环节，这是我们最想看到的诊断信号。

### E. 每个视频预计耗时

| 视频时长 | 预计标注耗时 |
|---|---|
| 短（5–6 min） | 30–45 分钟 |
| 中（7–10 min） | 45–60 分钟 |
| 长（>10 min） | 60–90 分钟 |

15 个视频估总工时 12–18 小时。建议每天 2 个视频，一周完成。

## 四、最容易标错的 6 个边界 case（cheatsheet）

| # | 场景 | 怎么标 |
|---|---|---|
| 1 | 学生跳过某步（如忘了判断脉搏） | `executed=false`，下面字段都置默认（0 分 / [] / null） |
| 2 | 一步分多次发生（如按压分 5 轮） | `interval` 标第一轮，note 写"共 5 轮" |
| 3 | 学生说了等同句（"所有人撤离"代替"所有人离开"） | 算说了，transcript 字段填实际原话 |
| 4 | 易错点同时触犯两条（如 E13 频率慢 + E14 中断长） | 都勾，时间戳分别填 |
| 5 | D8（总体评价）该怎么打？ | interval 填 [0, duration]，分数主观给即可 |
| 6 | 视频里看不清 / 听不清 | `confidence=unsure`，备注写"画面遮挡"或"音频不清" |

## 五、出错了怎么办

- 标错了 → 直接改后保存，系统有 audit log，不会丢历史
- 不知道某易错点啥意思 → 鼠标悬停看 tooltip（来自 rubric-v2）
- 系统崩了 → 您的输入每 30 秒自动存草稿，刷新即可恢复
- 易错点不够用 → 备注里写下，我们走 rubric 升级流程

## 六、和 vea 的关系（您可以不看，仅供好奇）

您标的数据 → 导出为 3 张表（gold_steps / gold_events / gold_overall）→ vea 用这些做：
- **训练**：让 AI 学会"看到这种动作 = 该扣 E10"
- **评估**：AI 跑完后对照您的答案算准确率
- **诊断**：AI 错得最多的步骤 = 下一轮要重点优化的环节

所以**您标得越细越准，AI 进步越快**。感谢您的时间。
