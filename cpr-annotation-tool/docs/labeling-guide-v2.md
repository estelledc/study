# CPR 视频标注与教学贡献指南 v2（老师阅读版）

> 配套 schema：`schema/labeling-schema-v2.json`
> 推导链：`docs/first-principles-derivation-v2.md`
> 评分依据：`cpr-eval/schema/rubric-v2.md`（8 类 28 步 100 分 / 33 个易错点 E1–E33）

## 一、为什么改成 v2

v1 只让您"批改学生考卷"。v2 让您**像教学一样贡献数据**。

类比：要让一个新医生学会评分，光让他看您批的卷子不够，还要：
- 给他看**对的范例**（"标准 EC 手法长这样"）
- 给他看**错的范例**（"E10 手肘弯曲长这样"）
- **正反对照**（"左对右错，差在手肘 15° 以上"）
- **告诉他规则的边界**（"E10 角度阈值大约是？"）

vea 也一样。所以 v2 把您的工作从 1 类扩展到 5 类。

## 二、五类贡献（您可以随选随做）

### A. 批改学生视频 `graded_video`（v1 的核心，保留）
对一段学生视频做完整 28 步评分。流程同 v1。

### B+. 上传正面示范 `reference_positive`
您觉得某个动作/口述特别标准 → 上传一段视频或一张图片。
**最常见用法**：
- 自己亲自演示（"看，标准的 EC 手法贴合面罩是这样")
- 从某学生视频里截一段做样板（"学生 7 这段按压姿势可以做范例"）

要填：
- 标题（"EC 手法标准示范"）
- 关联到哪几步（如 D7S2）
- 关键特征清单（"手肘伸直 / 掌根贴胸骨中下 1/3"）
- 解释（1-2 句话）

### B−. 上传反面教材 `reference_negative`
演示某个易错点的典型错误样子。
**最常见用法**：
- 自己演示错误动作（"这就是 E10 手肘弯曲"）
- 从学生视频里截一段（"学生 12 这段是 E13 频率慢的典型"）

要填：
- 主要演示哪个 E_code（必填，封闭从 E1–E33 选）
- 严重程度（轻 / 中 / 重）
- 关联到哪几步
- 解释

### C. 正反对照 `comparison_pair`
两个素材并排，告诉 AI 区别在哪。
**最强的教学信号**——边界都是这么学会的。

要填：
- 左 / 右各引用哪个媒体（视频或图片）
- 左是正例还是负例 / 右是正例还是负例
- 关键区别点列表（"手肘角度差 / 掌根位置差 5cm"）
- 为什么人容易混淆这两种

### D. 给 rubric 提注解 `rubric_note`
您发现 rubric 不清晰、缺失或矛盾时贡献的纯文字。
**4 种子类型**：
| proposal_kind | 何时用 |
|---|---|
| `clarify_existing` | 补充某项的边界（"E10 角度阈值约 15°"） |
| `propose_new_error_code` | 申请新增 E34、E35 |
| `propose_new_step` | 申请新 step（罕见） |
| `propose_rubric_split_or_merge` | 合并 / 拆分现有项 |

走审批后写入 rubric v3。**这条通道是 v1 死板封闭勾选的解药**。

---

## 三、媒介统一：视频和图片都收

无论哪类贡献（除了 D 纯文字），都可以上传：
- **视频**（mp4 / mov）+ 可选时间区间 [start, end]
- **图片**（jpg / png）+ 可选标注图层（圈、箭头）
- **两者混合**（一段视频 + 几张关键帧截图）

> 网站会自动从视频抽缩略图，您也可以手动上传图片标注后再附加。

---

## 四、所有贡献必须挂到 rubric

每条贡献必填 `step_links`：关联到 28 步里的哪几步。
负例和对比还要填 `error_links`：关联到 E1–E33。

**没有锚点的贡献 = vea 不知道训练时用在哪 = 进不了库**。
这是硬约束，请理解。

---

## 五、贡献状态流转

```
draft（草稿，您可以一直改）
   ↓ 您点"提交"
pending（待审）
   ↓ 至少 1 位 reviewer 批准
approved（进 vea 训练 / 检索库）
```

**MVP 阶段**：reviewer 数 = 1 即可（您团队内部互审）。
**后续**：可以调到 2 人审。

被 reject 的会带原因退回您 draft，可改后重提。

---

## 六、知情同意 + 授权

**必填三项**：
1. `subject_consent_obtained`：视频中可识别的人是否同意
2. `face_blur_applied`：是否已脱敏（可选）
3. `license`：
   - 仅团队内部使用
   - 脱敏后可用于研究
   - 可公开（CC-BY 开放共享）

**默认是仅团队内部使用**。要放宽到研究或公开必须显式勾选。

---

## 七、网站会引导您贡献到稀缺处

进首页会看到一张覆盖率热力图：
- 哪几步的正例最少
- 哪几个 E_code 还没有反例
- 哪几个边界 case 还没有对比对

按"缺口最大"排序。您可以选择跟着引导走，也可以自由贡献。

## 八、关键约定汇总

| 主题 | v2 约定 |
|---|---|
| graded_video 的同人复测 | 仍保留，3 个随机视频，2 周后重标 |
| 不确定按钮 | 仍保留，confidence=unsure 不进 gold set |
| 易错点封闭勾选 | 仍保留，但通过 rubric_note 走审批闭环 |
| 单标注者风险 | A 类靠复测；B/C/D 类靠多人审批共同缓解 |
| 媒体格式 | 视频或图片或混合，统一走 media[] |
| 数据导出给 vea | 7 个 parquet 表（gold_* + ref_* + comparison + rubric_notes） |

---

## 九、和 vea 的关系（您可以不看）

5 类贡献分别喂给 vea 不同模块：

| 贡献类型 | vea 用途 |
|---|---|
| graded_video | 评估 + 监督学习 |
| reference_positive | 检索增强：取正样本对照 |
| reference_negative | 检索增强：取负样本警示 |
| comparison_pair | 边界教学：让 AI 学分类边界 |
| rubric_note | 规则演进：让 AI 跟着 rubric v3 升级 |

**密度即上限**：每个 step / E_code 的样本越多 / 越对照鲜明，vea 越准。
您贡献的每一条，都直接拉高 vea 的天花板。
