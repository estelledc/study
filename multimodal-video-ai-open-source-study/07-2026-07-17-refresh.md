# 2026-07-17 全量刷新与证据复核

## 1. 本轮回答什么

旧材料已经解释了 9 个项目的架构。本轮不再扩充项目名单，而是重新检查四件事：

1. 固定源码是否仍对应 upstream 默认分支；
2. 每个项目的主链和限制是否仍能由真实文件支持；
3. 能否不用模型、账号和云服务，先真实跑通一条视频证据链；
4. 哪些结果是本地 E2，哪些仍只是源码 E1 或项目自述 E0。

结论：

- 9/9 本地仓库保持 clean；
- 9/9 pinned commit 与 2026-07-17 upstream HEAD 相同；
- 9/9 项目卡存在，并继续指向根目录 `projects/` 历史 clone；
- 新增实验真实生成并解码 MP4，但不调用 VLM、ASR 或外部服务；
- 真实教学视频上的识别准确率、成本和评分效度仍未验证。

## 3. 重新核对后的共同主链

9 个项目表面上分别叫 Agent、RAG、Skill 或教育助手，但都能放进同一条链：

```text
视频
  -> 采集与探测
  -> 全局压缩表示
  -> 按问题检索时间窗
  -> 回看原帧或原片段
  -> 领域规则判断
  -> 带来源的输出
```

差异主要发生在每一层由谁负责：

| 层 | 典型实现 | 主要代价 |
|---|---|---|
| 视频数据面 | VideoDB、Pixeltable、本地文件与 SQLite | 云绑定或本地运维 |
| 全局表示 | transcript、scene caption、知识树、向量 | 压缩时可能丢关键动作 |
| 检索 | FTS、向量、CLIP、层级记忆 | 召回错误会把后续推理带偏 |
| 回看 | DVD frame inspect、OmAgent Rewinder | 增加解码和模型调用成本 |
| 多工具补证 | OCR、ASR、检测、scene graph | 工具选择和跨模态冲突治理 |
| 领域判断 | protocol、rubric、步骤状态 | 需要可靠模板和人类校准 |

## 4. 三个最重要的源码复核

### 4.1 检索结果不是最终证据

`watch-skill` 的 `hybrid_search()` 合并 FTS5 与向量命中，`ask_video()` 再从命中
时间附近取帧。它说明：

```text
query
  -> 文本/向量候选
  -> 时间戳
  -> 邻近原帧
```

向量命中只负责“去哪里看”，不能单独证明“视频里发生了什么”。

源码：

- `projects/watch-skill/src/watch_skill/index/retrieval.py`
- `projects/watch-skill/src/watch_skill/index/db.py`
- `projects/watch-skill/src/watch_skill/answer/engine.py`

### 4.2 反思必须触发新观察

DeepVideoDiscovery 的提示要求找到答案后仍执行 `frame_inspect_tool` 确认；
OmAgent 的 `Rewinder` 接受时间范围和帧数，重新抽取原视频帧；ReAgent-V
在初答后生成 critical question、评价报告和多视角修订。

这三者的共同精神不是“让模型多想一次”，而是：

```text
初答
  -> 找到不确定点
  -> 新增问题相关证据
  -> 再判断
```

如果第二轮仍只看到第一轮同一段摘要，反思只是改写，不是复核。

源码：

- `projects/DeepVideoDiscovery/dvd/dvd_core.py`
- `projects/OmAgent/examples/video_understanding/agent/tools/video_rewinder/rewinder.py`
- `projects/ReAgent-V/ReAgent-V/ReAgentV.py`

### 4.3 领域评分不能由自由文本直接决定

`proteomics_lab_agent` 会把 protocol、示例、背景知识和视频交给模型，并提供
`BenchmarkDataset` / `StepAnalysis` 结构。但模型输出结构化并不自动等于评分可靠：

- protocol 可能错；
- 视频上传和模型观察过程不透明；
- `Benchmark` 字段仍需要真实标注校准；
- 同一错误类别在不同课程中可能含义不同。

对全智评更稳妥的迁移方式是：

```text
模型输出观察与证据
  -> schema gate
  -> 后端确定性规则算状态/分数
  -> 低置信或冲突进入人工复核
```

源码：

- `projects/proteomics_lab_agent/proteomics_lab_agent/sub_agents/lab_note_generator_agent/agent.py`
- `projects/proteomics_lab_agent/eval/`

## 5. 本轮 E2：真实视频证据链

实验见：

- [零基础视频证据实验](08-beginner-video-evidence-lab.md)
- `labs/video_evidence_lab.py`
- `labs/test_video_evidence_lab.py`

实验真实执行：

1. FFmpeg 生成 12 秒 MP4；
2. 三段画面依次代表 `prepare`、`add_sample`、`close_lid`；
3. `ffprobe` 读取时长、分辨率和音轨；
4. FFmpeg 从 1、5、9 秒真实解码帧；
5. 问题路由把 “sample added” 聚焦到 4-6 秒；
6. focused review 再解码 5 秒原帧；
7. rubric 检查步骤覆盖、顺序和跨模态冲突；
8. 视频与帧都写入 SHA-256 provenance。

运行结果：

```text
video: duration=12.0s size=160x120 audio=True
global: prepare@1.0s, add_sample@5.0s, close_lid@9.0s
focus: add_sample 4.0-6.0s
rubric=passed contradiction=needs_review
```

测试：

```text
9 passed
```

## 6. 这个 E2 能证明什么

可以证明：

- 真实 MP4 可以被创建、探测和按时间解码；
- 全局证据可以先持久化，再按问题复用；
- 未命中的问题不会凭空选择时间窗；
- 缺步骤、乱序和跨模态冲突不会自动通过；
- frame hash 被篡改后 provenance gate 会失败。

不能证明：

- 颜色分类能代表真实动作识别；
- VLM 能准确识别学生操作；
- ASR 和画面在真实噪声下能可靠对齐；
- 固定 1/5/9 秒采样适合任意视频；
- 该 rubric 能测量真实学习成果；
- 任一开源项目的论文数字已复现。

实验使用确定性颜色场景，是为了验证控制流和证据合同，不是替代模型效果实验。

## 9. 下一项真实验证

若要把证据等级继续提升，下一项不是再加 Agent，而是固定一个小型人工标注集：

```text
10 个短视频
  x 3 个步骤
  x 正常 / 缺失 / 乱序 / 遮挡
```

每个样本至少保存：

- 原视频 hash；
- step ground truth；
- 允许时间窗；
- 可见性；
- ASR 是否提供直接线索；
- 预测证据时间戳；
- 最终状态；
- 人工覆写与原因。

然后比较：

1. 均匀抽帧；
2. 场景抽帧；
3. 全局扫描 + 问题驱动 focused review。

只有这一步才能开始回答“主动取证是否真的改善全智评准确率和成本”。
