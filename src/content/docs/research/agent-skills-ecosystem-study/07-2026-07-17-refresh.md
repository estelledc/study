## 3. 五个漂移仓的影响

### 3.1 Anthropic Skills：内容包会快速演化

一个提交同时更新：

- `docx`、`pptx`、`xlsx` 三个 `SKILL.md`；
- Office validation、comment、recalc 和 helper；
- 新增 PPTX chart、slide、theme helper；
- 删除或移动旧 pack/unpack、redline 和 PPTX 参考文件。

影响：

- “复杂官方 Skill 由指令 + 脚本 + validator 组成”的结论更强。
- 固定提交中的 Office 文件数、具体路径和旧 helper 组织已经过期。
- 不能把“官方示例”误解为稳定 API；复制路径前必须绑定 commit。

### 3.2 Vercel Skills CLI：安装器也是安全边界

8 个提交从 `a9d8e3a` 推进到 `777599e`：

- Node.js 最低版本提高到 22.20。
- git transport 从简单判断升级为显式 allowlist，并新增 58 行测试。

影响：

- 环境兼容和 URL/protocol 验证属于安装器安全合同，不是外围细节。

## 4. 重新验收后的稳定结论

1. **规范只定义最小可移植骨架。**
   安装、发布、安全、评测和优化仍由不同项目补齐。

2. **内容型 Skill 的路径和脚本不是稳定 API。**
   Anthropic Office 更新证明官方样例也会重构。

3. **安装器必须把来源当不可信输入。**
   git 协议 allowlist 与跨平台测试属于主链。

4. **市场只证明收录和版本锚点。**
   registry 快速更新不能自动证明第三方内容安全。

5. **Harness 的复利单位是可验证的新能力。**
   `ce-handoff` 同时带 Skill、文档、设计和合同测试，而不是只多一个提示词。

6. **生成器首先是数据管线。**
   source 类型、网络恢复和 pattern precision 会直接影响最终 Skill。
