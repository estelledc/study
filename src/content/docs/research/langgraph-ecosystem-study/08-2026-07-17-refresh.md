## 3. 增量影响

### 3.1 核心内核没有漂移

LangGraph Python 与 LangGraph.js 均与 pinned commit 一致。因此以下研究主轴仍可直接复算：

- `StateGraph` 编译成可执行图。
- Pregel/BSP superstep 的“计划 -> 并行执行 -> 合并更新”。
- reducer 解决同一步并行写入。
- checkpointer 保存 thread 状态版本。
- `Command`、`Send`、interrupt、subgraph 和 stream 的职责。

这比“21 个仓都有无更新”更重要：类别锚点的执行语义没有变化。

### 3.2 Deep Agents 的产品层增长快于核心库

23 个提交主要落在 `deepagents-code`：

- plugin marketplace 和搜索。
- MCP OAuth 与非阻塞 token refresh。
- plugin reload 状态与 TUI。
- debug console、model selector、auto-approve trace。

`libs/deepagents` 只命中依赖元数据。既有“Deep Agents 在 LangChain/LangGraph 上加入 filesystem、skills、subagents、memory”的架构结论仍成立，但 CLI 产品面已经明显扩大。

### 3.3 DeerFlow 改动属于渠道层

一个提交只修复 Feishu 群聊中带 bot `@mention` 前缀的命令分发，并补 parser 测试。它不改变 DeerFlow 的 gateway/harness/agent/sandbox 分层。

这说明生产产品的 channel adapter 也需要独立测试，不能把消息能到达 gateway 当成 graph 核心能力。

### 3.4 Pydantic AI 加强 deferred control

新增：

- `DeferredToolCallEvent` / `DeferredToolResultEvent`。
- queued message delivery event。
- capability-derived durable toolset ID。
- deferred tool 内部实现和 UI event 适配。

这让 Pydantic AI 不再只适合“类型安全的直接 Agent loop”；它正在增强 deferred tool 与 durable execution 的事件语义。与 LangGraph 的差异仍在：Pydantic AI 从 typed Agent/run/capability 向外扩展，LangGraph 从 state graph/checkpoint 向上组合。

### 3.5 OpenAI Agents SDK 修复流式重试状态

两个关键修复：

- model retry 时保留 streamed session input。
- 防止复用 stale prepared-item identity。

这支持既有判断：轻量 run loop 也必须明确输入身份和重试边界；“代码少”不等于状态问题更少。

### 3.6 其他替代框架

- Microsoft AF：补 .NET session/CodeAct、Neo4j memory sample、Python workflow 内 HITL response URL。
- CrewAI：发布 1.15.3，修正 kickoff-completed event 与 hook output 同步。
- Mastra：强化每 turn eval score、Mastra Code session、可插拔 auth/sandbox factory。

这些变化扩展产品能力，没有反证原先的执行模型和持久化比较。

## 4. 重新验收后的稳定结论

1. **Graph、Agent、Harness、Product、Deployment 是不同层。**
2. **Reducer 是并行状态合并合同，不是普通 list append 语法糖。**
3. **Checkpoint 保存状态，不自动提供 exactly-once side effect。**
4. **Deferred tool 和 streamed retry 都需要稳定 identity。**
5. **渠道、UI、认证和部署故障不能归因于 StateGraph。**
6. **对照框架正在互相吸收能力，但起始抽象仍不同。**

## 5. E2 运行证据

本机通过 `uv --no-project --with-editable` 加载 pinned LangGraph 源码，运行无模型、无网络的 StateGraph：

```text
START -> source_a ----\
                       -> summarize -> END
START -> source_b ----/
```

结果：

```text
A:reducers | B:reducers
checkpoints=4
```

4 个测试验证：

- 两个并行节点通过 reducer 合并 findings。
- 无 reducer 的并行同 key 写入抛 `InvalidUpdateError`。
- InMemorySaver 保存 thread history。
- 编译后拓扑包含 START、两个 source、summarize 和 END。

首次按完整 workspace 运行时，因为 sparse checkout 未物化 `libs/cli` 而失败；改用 editable 单包隔离运行后通过。这个过程记录在[零基础实验](09-beginner-stategraph-lab.md)。
