# BERT — 双向 Transformer 预训练（零基础版）

## 一句话总结

BERT 是 Google 在 2018 年提出的一个语言模型，它的核心创新是：**让模型在读文章的时候，既能看到左边的字，也能看到右边的字，从而更好地理解语言。** 这个想法听起来简单，但在当时是前所未有的。

---

## 1. 从日常类比开始

### 1.1 填空题 vs 同传翻译

想象你在做一道填空题：

> 「今天天气很__，适合去公园。」

你会怎么填？你会看前面的「天气很」，也会看后面的「适合去公园」。两个方向的线索一起帮你判断答案是「好」。

**BERT 就像是一个特别擅长做填空题的选手。** 它读任何一句话时，都能同时看到左右两边的所有词。

对比一下另一种模型（GPT）：

> GPT 像一个同声传译员。他只能听到前面已经说出来的话，然后猜接下来会说什么。他看不到"后面"的内容。

这两种能力完全不同：

| 能力 | 类比 | 擅长任务 |
|------|------|----------|
| 双向理解（BERT） | 做填空题 | 判断句子感情、回答问题 |
| 单向生成（GPT） | 接龙游戏 | 写文章、聊天、写代码 |

### 1.2 先上大学，再做专业实习

BERT 的工作流程分两步：

1. **预训练（Pre-training）**：让模型在海量无标签文本上"上大学"——读几十亿句话，学会语言的通用规则（语法、常识、世界知识）。这一步不需要任何人标注，模型自己从文本中学。
2. **微调（Fine-tuning）**：让模型带着大学学到的知识去做"专业实习"——在某个具体任务（比如情感分类、问答）的少量标注数据上继续训练。

这个思路的关键在于：**预训练是一次性的、昂贵的，但可以被无数下游任务复用。** 就像你读完大学后，可以去很多不同的公司工作，不需要每次都重新上学。

## 2. 核心概念

### 2.1 Transformer 是什么（简要回顾）

BERT 的底层架构来自一篇 2017 年的论文《Attention Is All You Need》。如果你还没读过那篇笔记，这里只需要知道：

- Transformer 是一种神经网络架构，核心是 **self-attention** 机制
- self-attention 让每个词在处理时都能看到句子中的其他所有词
- Transformer 有 **encoder**（编码器）和 **decoder**（解码器）两部分
- **BERT 只用了 encoder 部分**

### 2.2 什么是"双向"

这是 BERT 最大的创新点。

**之前的模型要么单向、要么"浅双向"：**

- **GPT**：单向。处理第 5 个词时，只能看到第 1-4 个词。像读小说时只能往前看，不能往后翻。
- **ELMo**：浅双向。它同时跑了两个独立的单向模型（一个从左到右，一个从右到左），然后把结果拼在一起。但这不是真正的"双向融合"——两个方向的信息在最后才碰面。

**BERT 是"深双向"：**

- 从第一层到最后一层，每个词在每一步都能看到左右两边的全部信息
- 就像你读一句话时，眼睛可以同时扫视整行，而不是逐字移动

### 2.3 MLM — 掩码语言模型（Masked Language Model）

既然要让模型同时看到左右两边，那训练时怎么出题呢？

**答案：随机遮住一些词，让模型猜。**

```
原句：  我  喜欢  吃  北京  烤鸭
遮掉：  我  喜欢  [MASK]  北京  烤鸭
任务：  模型预测 [MASK] 位置应该填什么 → "吃"
```

模型必须同时利用左边的「我喜欢」和右边的「北京烤鸭」才能猜出正确答案。这就是双向的力量。

**为什么不能直接让模型预测所有词？** 因为如果模型能看到所有词（包括它自己要猜的那个词），它就变成"抄答案"了，没有真正的学习能力。遮住一部分，制造一个真正的预测任务。

**实际训练中的小技巧（80/10/10 规则）：**

被遮住的词不是简单地替换成 `[MASK]`，而是按以下比例处理：

- 80% 替换成 `[MASK]`
- 10% 替换成一个随机词（比如把「吃」换成「看」）
- 10% 保持不变

这样做是为了解决一个问题：训练时满屏幕都是 `[MASK]`，但实际使用时一个 `[MASK]` 都没有。加入随机词和保持原词，能让训练和实际的差距变小。

### 2.4 NSP — 下一句预测（Next Sentence Prediction）

BERT 的训练任务不止一个。第二个任务是：

> 给定两个句子 A 和 B，判断 B 是不是真的跟在 A 后面。

```
例子 1（真实）：
A: 我喜欢吃北京烤鸭。
B: 它的外皮酥脆可口。
→ 标签：是

例子 2（随机）：
A: 我喜欢吃北京烤鸭。
B: 量子纠缠是一种物理现象。
→ 标签：否
```

这个任务的目的是让模型学会理解句子之间的关系。BERT 会在每句话开头加一个特殊的 `[CLS]` token，这个位置的最终输出就被用来做判断。

**后来的发现：** 2019 年的 RoBERTa 论文证明 NSP 其实没那么重要。去掉它之后，BERT 的效果反而更好了。这说明 BERT 论文自己可能高估了这个贡献。

## 3. 代码示例

### 3.1 用 HuggingFace Transformers 加载预训练 BERT

```python
from transformers import BertTokenizer, BertModel

# 1. 加载分词器和预训练模型
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model = BertModel.from_pretrained('bert-base-uncased')

# 2. 输入一句话
text = "BERT is a revolutionary language model."
inputs = tokenizer(text, return_tensors='pt')

# 3. 让模型编码（不训练，只做推理）
with torch.no_grad():
    outputs = model(**inputs)

# 4. 获取输出
last_hidden_state = outputs.last_hidden_state   # 每个 token 的隐藏表示
cls_embedding = outputs.pooler_output          # [CLS] 位置的向量（768 维）

print(f"输入分词后: {tokenizer.convert_ids_to_tokens(inputs['input_ids'][0])}")
print(f"隐藏状态形状: {last_hidden_state.shape}")
# 形状说明: [1, 9, 768]
#   1     = 1 个句子
#   9     = 9 个 token（包括 [CLS] 和 [SEP]）
#   768   = 每个 token 的向量维度（BERT-base 固定为 768）
```

**这段代码做了什么？**

1. 把文字切成 token（类似"字"的最小单位）
2. 把 token 送进 BERT 的 12 层 encoder
3. 每一层都会让 token 之间互相"交流"（通过 self-attention）
4. 最后一层输出的就是 BERT 对这个句子的"理解"

### 3.2 微调 BERT 做情感分类

```python
from transformers import BertForSequenceClassification, Trainer, TrainingArguments
from datasets import Dataset
import torch

# 1. 准备少量标注数据（比如 500 条评论）
data = {
    'text': [
        '这部电影太棒了！剧情紧凑，演员出色。',
        '非常失望，浪费了两个小时。',
        '一般般吧，没什么特别的。',
        # ... 更多数据
    ],
    'label': [1, 0, 1]  # 1=正面, 0=负面
}
dataset = Dataset.from_dict(data)

# 2. 加载带分类头的 BERT（比基础版多了一层分类器）
model = BertForSequenceClassification.from_pretrained(
    'bert-base-uncased',
    num_labels=2  # 二分类：正面 / 负面
)

# 3. 配置训练参数
training_args = TrainingArguments(
    output_dir='./results',
    num_train_epochs=3,          # 过 3 遍数据就够了
    per_device_train_batch_size=16,
    learning_rate=2e-5,          # 很小的学习率（微调不能走太快）
    weight_decay=0.01,
)

# 4. 开始微调
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset,
)
trainer.train()

# 5. 用微调好的模型做预测
from transformers import pipeline
classifier = pipeline('sentiment-analysis', model=model, tokenizer=tokenizer)
result = classifier("这家餐厅的服务真好！")
print(result)
# 输出: [{'label': 'POSITIVE', 'score': 0.98}]
```

**关键数字解释：**

- `num_train_epochs=3`：微调不需要训练很多轮，3-4 轮就够了。因为预训练已经让模型学到了丰富的语言知识，微调只是"微调"。
- `learning_rate=2e-5`：微调的学习率通常比从头训练小 10 倍。走太快会把预训练学到的知识"走丢"。
- `num_labels=2`：根据任务改变。如果是 10 类分类就改成 10。

## 4. BERT 的规模

BERT 有两个常用版本：

| 版本 | 层数 | 隐藏维度 | 注意力头数 | 参数量 | 训练硬件 |
|------|------|----------|-----------|--------|----------|
| BERT-base | 12 | 768 | 12 | 1.1 亿 | 16 块 TPU × 4 天 |
| BERT-large | 24 | 1024 | 16 | 3.4 亿 | 64 块 TPU × 4 天 |

2018 年这个训练成本非常高，只有 Google 这样的大公司能负担。这也是为什么 BERT 发布后，学术界开始大量基于它的权重做研究——开源了权重比开源论文的影响力更大。

## 5. BERT 的影响

### 5.1 当时的成绩

BERT 发布后，在 **11 个 NLP 任务**上同时刷新了世界纪录：

- GLUE 基准测试得分从 76.2% 提升到 80.5%
- SQuAD 问答测试 F1 分数达到 93.2
- 多项任务实现了超过 4% 的绝对提升

### 5.2 长尾影响

1. **"预训练 + 微调"成为标准范式**：不只是 NLP，计算机视觉（ImageNet 预训练）、语音、多模态都采用了同样的思路
2. **开源预训练权重改变了学术生态**：研究者可以站在 Google 的肩膀上做研究，不再需要从头训练
3. **催生了后续一系列改进**：RoBERTa、ALBERT、DistilBERT、DeBERTa 等，各有侧重（更快、更小、更强）
4. **搜索系统的变革**：Google 搜索在 2019 年引入了 BERT，据称改善了 10% 的查询理解

## 6. 常见误区

### 误区 1：BERT 能生成文本

不对。BERT 是"理解型"模型，不是"生成型"模型。它擅长分类、抽取、判断，但不擅长写文章或对话。GPT 系列才是生成型的。

### 误区 2：BERT 越大越好

不一定。BERT-large 确实比 BERT-base 强，但代价也大得多。后来出现了 DistilBERT（只有 BERT-base 一半大小，保留 97% 性能）和 TinyBERT（更极致压缩），在资源受限场景下更实用。

### 误区 3：BERT 已经被淘汰了

不完全对。虽然生成任务被 GPT 系列主导，但在**检索、分类、信息抽取**等领域，BERT 及其衍生模型仍然是工业界的主流选择。RAG（检索增强生成）系统广泛使用 BERT 系模型做语义检索。

## 7. 学到了什么

1. **双向上下文是关键**：让模型同时看到左右两边，比单向模型理解能力强很多。这个直觉后来被反复验证
2. **预训练 + 微调是工作流胜利**：不是某个技术突破，而是组织方式的改变——一次昂贵预训练，无限次廉价微调
3. **`[MASK]` 是个巧妙的 trick**：用最简单的方式解决了"双向 LM 不能看自己"的悖论。这种"靠任务设计而非数学发明"的思路后来被 MAE、SimCLR 等工作沿用
4. **读论文要有批判性**：BERT 论文把 NSP 写成核心贡献，一年后 RoBERTa 证明它不重要。区分"真原则"和"噪声细节"是读论文的重要能力
5. **开源权重比开源论文影响更大**：BERT 的开放权重让全球研究者可以直接用它做实验，这种共享精神加速了整个领域的发展
