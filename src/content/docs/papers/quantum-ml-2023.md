# Quantum Machine Learning: A Review — 零基础学习笔记

## 一、这讲的是啥？

量子机器学习（Quantum Machine Learning, QML）是两件事的交集：

- **量子计算**：用量子物理原理来做计算
- **机器学习**：用数据训练模型来做预测

把量子计算的"武器库"引入机器学习，目标是让某些任务**跑得更快、用得更少资源**。

### 日常类比

想象你在一个巨大的图书馆里找一本书：

- **经典计算机**像是一个人，一次只能走一条走廊，一本本翻
- **量子计算机**像是有分身术，可以同时走所有走廊、同时翻所有书

QML 就是在问：如果找书的人有分身术，那么"训练一个能识别书的模型"这件事，会不会也更快？

## 二、三个必须懂的核心概念

### 1. 量子比特（Qubit）

经典比特只能是 0 **或** 1。量子比特可以是 0 和 1 的**叠加**。

```
经典比特：  0  或  1

量子比特：  α|0⟩ + β|1⟩   （同时包含 0 和 1，测量时才坍缩成一个）
```

| α|² 是测出 0 的概率
| β|² 是测出 1 的概率

一个直观类比：硬币在桌面上旋转时，它既不是正面也不是反面——它是"正+反"的叠加。只有你用手按住它（测量），才会变成其中一面。

### 2. 量子纠缠（Entanglement）

两个量子比特可以"纠缠"在一起：无论相距多远，一个被测量了，另一个瞬间就确定了对应的状态。

```
纠缠态（Bell 态）： |Φ+⟩ = (|00⟩ + |11⟩) / √2

意思是：测两个比特，要么都是 0，要么都是 1
不可能出现 01 或 10。它们是"穿一条裤子"的。
```

类比：你有两枚魔法硬币，分别放在地球和火星。 shaken 之前它们都处于叠加态。一旦你在地球上按住一枚看到"正面"，火星上的那枚**瞬间**也会变成"正面"。

### 3. 量子门（Quantum Gate）

经典计算用逻辑门（AND、OR、NOT）处理比特。量子计算用量子门处理量子比特。

常见量子门：

- **Hadamard 门（H）**：把 |0⟩ 变成叠加态 (|0⟩ + |1⟩)/√2
- **Pauli-X 门**：量子版的 NOT，把 |0⟩ 翻成 |1⟩
- **CNOT 门**：两比特门，控制比特为 |1⟩ 时才翻目标比特

## 三、QML 的两大路线

### 路线 A：用量子计算机跑机器学习算法

目标：让量子计算机本身去训练神经网络、跑支持向量机。

代表算法：

- **量子支持向量机（QSVM）**：用量子电路计算核矩阵
- **量子主成分分析（qPCA）**：量子版的 PCA，加速特征分解
- **量子神经网络（QNN）**：用量子门代替经典神经网络的权重矩阵

### 路线 B：用量子启发经典算法

目标：不一定要等量子计算机成熟，先从量子理论中汲取灵感，改进经典算法。

代表思路：

- 用量子概率模型改进自然语言处理
- 用量子退火思路优化组合问题
- 用 Tensor Network（张量网络）加速经典神经网络

## 四、代码示例

### 示例 1：用量子线路创建叠加态（Qiskit）

这个例子演示如何用 1 个量子比特创建叠加态：

```python
# 创建 1 个量子比特和 1 个经典比特
from qiskit import QuantumCircuit

qc = QuantumCircuit(1, 1)

# 应用 Hadamard 门：|0⟩ → (|0⟩ + |1⟩)/√2
qc.h(0)

# 测量
qc.measure(0, 0)

# 查看线路
print(qc.draw())
```

输出：

```
     ┌───┐┌─┐
q_0: ┤ H ├┤M├
     └───┘└╥┘
c_0: ═════╩═
          0 
```

跑 1024 次模拟，大约 50% 概率得到 0，50% 概率得到 1——这就是叠加态的效果。

### 示例 2：创建纠缠态（贝尔态）

```python
from qiskit import QuantumCircuit

# 2 个量子比特 + 2 个经典比特
qc = QuantumCircuit(2, 2)

# H 门作用在第一个比特上，创建叠加
qc.h(0)

# CNOT 门：比特 0 控制比特 1，创建纠缠
qc.cx(0, 1)

# 测量
qc.measure([0, 1], [0, 1])

print(qc.draw())
```

输出：

```
     ┌───┌─┐
q_0: ┤H├┤M├
     └─┬┘└╥┘
q_1: ──╫──╩═
       ║  0 
c_0: ═══╩═══
       0    
c_1: ═══════
          1 
```

这个线路会 100% 产生 `00` 或 `11`，不会产生 `01` 或 `10`。两个比特纠缠了。

## 五、QML 的现状

| 方面 | 现状 |
|------|------|
| **理论** | 数学框架已经建立，很多算法在纸面上证明了加速 |
| **硬件** | 量子计算机还在"早期阶段"，比特数少、噪音大 |
| **实用价值** | 尚未在真实场景超越经典算法（"量子优势"只在特定任务上） |
| **前景** | 长期看潜力巨大，短期需要耐心等待硬件进步 |

### 关键术语

- **NISQ**：Noisy Intermediate-Scale Quantum。当前量子计算机的世代——有几十到几百个比特，但噪音很大，还做不到错误纠正
- **量子优势（Quantum Supremacy）**：量子计算机在某个特定任务上超越了最强经典计算机。Google 在 2019 年用量子处理器 Sycamore 声称实现了这个目标

## 六、总结

QML 的核心想法并不复杂：

1. 量子比特能叠加，一个量子计算机能"同时处理"多种可能性
2. 量子纠缠让多个比特之间产生强关联
3. 量子门可以操作这些量子比特
4. 把这些能力用于机器学习任务，理论上可以获得加速

但现实是：量子硬件还很不成熟。当前阶段最好的做法是**先学好理论，保持关注，等硬件成熟后快速跟进**。

> 学习建议：先搞懂叠加态和纠缠态的物理直觉，再接触 Qiskit 写代码，最后读原论文中的算法证明。顺序不能颠倒。
