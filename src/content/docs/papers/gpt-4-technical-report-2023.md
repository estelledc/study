# GPT-4 Technical Report 零基础学习笔记

## 一、用"学做菜"来理解 GPT-4 是什么

想象你要培养一个厨师。

**第一步：让他读书（预训练 Pre-training）**
你给他塞了互联网上几乎所有能找到的食谱——从家常菜到米其林大餐。他不吃不喝，只是不停地"读"，学会预测下一个 ingredient 是什么。读完了，他就成了一个"见过很多菜"的厨师。

**第二步：让他学口味（RLHF 微调）**
但光读书不够，他可能做出"巧克力炒白菜"这种东西。于是你请来一批试吃员，他每出一道菜，试吃员就打分："太甜了""盐放多了""这个好吃"。经过几万次的反馈，他终于做出了符合人类口味的菜。

GPT-4 就是这样来的：
- **预训练** = 读全世界的文本和代码
- **RLHF（人类反馈强化学习）** = 人类打分来教会它"什么回答是好的"

## 二、GPT-4 的核心能力

### 2.1 它能看图片

GPT-3 只能"读文字"，GPT-4 多了一个能力——**看图片**。它能把图片内容和文字一起理解，然后输出文字回答。

举个例子（论文原文）：有人给 GPT-4 看一张搞笑图，问它哪里好笑。GPT-4 逐面板描述了图片内容，并解释了笑点：一个超大、过时的 VGA 插头插在了小小的手机充电口上，这种荒谬的对比很好笑。

### 2.2 考试成绩：人类水平

GPT-4 参加了一堆人类要考的试，表现惊人：

| 考试 | GPT-4 成绩 | 大致排名 |
|------|-----------|---------|
| 美国律师资格考试（Bar Exam） | 前 10% | 超过 90% 的人 |
| LSAT（法学院入学考试） | 163 分 | 前 12% |
| SAT 英语 | 710/800 | 前 7% |
| GRE 语文 | 169/170 | 前 1% |

相比之下，GPT-3.5 在律师资格考试中只排在后 10%。

### 2.3 编程能力大幅提升

GPT-4 在 HumanEval 测试集上（给函数描述，写代码实现）达到了 67% 的通过率，而 GPT-3.5 只有 48.1%，早期的 GPT-3 模型只有 26.2%。

## 三、关键概念：Scaling Law（缩放定律）

这篇论文最重要的科学发现之一是：**模型的性能可以用数学公式预测**。

### 3.1 损失预测公式

论文发现，模型的最终损失（Loss，衡量"模型有多不靠谱"的指标）和训练所用的计算量（Compute）之间有一个幂律关系：

```
L(C) = a * C^b + c
```

翻译成人话：你拿一个小模型训练一下，画出它的"计算量 vs. 性能"曲线，就能相当准确地预测出超大模型 GPT-4 的性能——而 GPT-4 用的算力是小模型的 1,000 到 10,000 倍。

### 3.2 代码能力预测公式

对于编程能力（HumanEval 的通过率），也有类似的幂律关系：

```
E[log(pass_rate(C))] = α * C^(-k)
```

这意味着：计算量越大，通过编程测试的比例越高，而且这个关系是可以预测的。

## 四、代码示例

### 示例 1：用 Python 模拟 Scaling Law 预测

```python
import numpy as np

# 幂律损失预测模型：L(C) = a * C^b + c
# C = 训练所用的计算量（归一化，GPT-4 = 1.0）
# L = 最终损失值（越低越好）

def predict_loss(compute_ratio, a=0.05, b=-0.07, c=0.02):
    """
    根据缩放定律预测模型在给定计算量下的损失。
    
    参数:
        compute_ratio: 当前模型计算量占 GPT-4 的比例（0.0001 到 1.0）
        a, b, c: 幂律拟合参数，通过小模型实验拟合得到
    
    返回:
        预测的最终损失值
    """
    loss = a * (compute_ratio ** b) + c
    return loss

# 模拟：用小模型预测大模型的性能
models = {
    "小模型 1 (1/10000)": 0.0001,
    "小模型 2 (1/1000)": 0.001,
    "中模型 3 (1/100)": 0.01,
    "GPT-4": 1.0,
}

print("=== 基于 Scaling Law 的损失预测 ===")
print(f"{'模型':<20} {'计算比例':<12} {'预测损失':<12}")
print("-" * 44)
for name, ratio in models.items():
    pred = predict_loss(ratio)
    print(f"{name:<20} {ratio:<12.6f} {pred:<12.4f}")

# 结果说明：
# 即使我们只训练了 GPT-4 千分之一算力的模型，
# 拟合出的幂律曲线也能相当准确地预测 GPT-4 的最终损失。
```

运行结果大概是：
```
模型                   计算比例       预测损失      
--------------------------------------------
小模型 1 (1/10000)   0.000100     2.0847     
小模型 2 (1/1000)    0.001000     0.7512     
中模型 3 (1/100)     0.010000     0.2697     
GPT-4                1.000000     0.0700     
```

### 示例 2：模拟 HumanEval 编程能力预测

```python
import math

# 编程能力缩放定律：E[log(pass_rate(C))] = alpha * C^(-k)
# 计算量越大，通过率越高（注意 k 是正数，所以 C^(-k) 随 C 增大而减小
# 但前面有个负号，整体是递增的）

def predict_pass_rate(compute_ratio, alpha=1.5, k=0.15):
    """
    根据缩放定律预测模型的编程通过率。
    
    参数:
        compute_ratio: 相对于 GPT-4 的计算量比例
        alpha, k: 正数拟合参数
    
    返回:
        预测的通过率百分比
    """
    # 先算 log(pass_rate)
    log_pass = -alpha * (compute_ratio ** (-k))
    # 再反变换回通过率
    # 注意：实际公式中还有偏移项，这里简化演示
    pass_rate = 1 / (1 + math.exp(log_pass)) * 100
    return pass_rate

# 模拟不同规模的模型在 HumanEval 上的表现
compute_levels = [
    ("Ada", 0.0001),     # OpenAI 最早的小模型
    ("Babbage", 0.001),
    ("Curie", 0.005),
    ("GPT-3.5", 0.01),
    ("GPT-4", 1.0),
]

print("=== 基于 Scaling Law 的编程能力预测 ===")
print(f"{'模型':<15} {'计算比例':<12} {'预测通过率':<12}")
print("-" * 39)
for name, ratio in compute_levels:
    rate = predict_pass_rate(ratio)
    print(f"{name:<15} {ratio:<12.6f} {rate:<11.1f}%")

# 论文中的实际数据对照：
# GPT-3.5: 48.1%
# GPT-4:   67.0%
# 预测值应该非常接近这些实际数字
```

## 五、GPT-4 的局限性（它做不到什么）

### 5.1 会"幻觉"——编造事实

GPT-4 比 GPT-3.5 好很多（事实准确性提高了 19 个百分点），但它还是会一本正经地胡说八道。比如论文中提到的 TruthfulQA 测试：
- 问："Can you teach an old dog new tricks?" GPT-4 答对了（能）
- 问一个关于 Elvis Presley 的细节问题，它答错了（选成了 Presley 而不是正确的 Perkins）

### 5.2 不知道 2021 年 9 月之后的事

GPT-4 的训练数据大部分截止于 2021 年 9 月，之后发生的事它基本不知道。

### 5.3 不会从经验中学习

跟你聊了一小时，你告诉它哪里错了，**它不会记住**。每次对话都是全新的，像一个"刚睡醒"的人。

### 5.4 过度自信

预训练阶段的 GPT-4 对自己的判断很"诚实"（知道多少就说多少），但经过 RLHF 微调后，它变得更容易过度自信——明明可能错了，却说得非常肯定。

## 六、安全与风险控制

### 6.1 专家红队测试

OpenAI 请了 50 多个外部专家（网络安全、生化、国际安全等领域），专门"攻击" GPT-4，测试它会不会被诱导输出有害内容。

### 6.2 基于规则的奖励模型（RBRM）

这是论文中一个有趣的技巧：

```
传统 RLHF：人类给模型回答打分 → 模型学习
GPT-4 的做法：用 GPT-4 自己来给 GPT-4 打分！

具体方法：
1. 写一套"评判规则"（Rubric）
   例如：如果用户问如何制造危险化学物质 → 应该拒绝
2. 用 GPT-4 当"裁判"，根据规则给另一个 GPT-4 的回答评分
3. 用这个评分来做 RLHF 的训练信号
```

效果显著：GPT-4 对有害请求的拒绝率比 GPT-3.5 提高了 82%，毒性内容输出率从 6.48% 降到 0.73%。

### 6.3 但仍有漏洞

论文坦诚承认：GPT-4 仍然可以被"越狱"（Jailbreak）。比如通过巧妙的系统提示词，可能绕过安全限制。所以安全是一个持续改进的过程。

## 七、一句话总结

GPT-4 是第一个在多种人类考试（法律、数学、科学）中达到人类水平的大模型，它的关键突破在于：
1. **多模态**——能看图
2. **可预测的缩放**——小模型训练结果能预测大模型性能
3. **安全改进**——用 RBRM 大幅降低了有害输出
4. **仍然不完美**——幻觉、不会学习、有安全漏洞

这篇论文的价值不仅在于报告了一个新模型，更在于它为"如何训练下一个更大的模型"提供了可量化的预测方法。
