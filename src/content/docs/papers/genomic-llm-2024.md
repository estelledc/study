# Large Language Models for Genomics: A New Paradigm

> 学习笔记 · 基于 arXiv:2407.11435 "Genomic Language Models: Opportunities and Challenges"
> 以及 arXiv:2311.07621 "To Transformers and Beyond: Large Language Models for the Genome"

## 从日常类比开始

### DNA 就是一本书，碱基就是字母

想象一下：人类基因组是一本由 **30 亿个字母** 写成的超级厚的书。只不过这儿的字母不是英文的 26 个，而是只有 **A、C、G、T** 四个。

在自然语言处理（NLP）中，我们用 GPT、BERT 这类大模型来"读懂"英文或中文——理解语法、语义、上下文。现在科学家做了个大胆的想法：**如果把 DNA 序列当作"文字"来训练语言模型，模型会不会也学会"读懂"基因组？**

答案：**会的，而且效果非常惊人。** 这类模型就叫 **Genomic Language Models (gLMs)**。

### 蛋白质语言的先驱

在这之前，科学家已经成功训练了 **蛋白质语言模型 (pLMs)**——用 UniProt 数据库里数亿条蛋白质序列来训练。最著名的成果是 **AlphaFold**，它几乎完美预测了蛋白质的三维结构。

蛋白质语言的"成功密码"是什么？地球上几十亿年的进化，已经把大量有用的蛋白质序列"写"在了数据库里。模型从中学习到了"什么样的氨基酸组合是合理的、什么样的不行"。

DNA 的情况类似，但也有独特挑战——**基因组中有大量不编码任何功能的"非功能性区域"**，真正有用的部分只占很小比例。

## 核心概念

### 1. 语言模型的两种训练方式

| 方式 | 英文类比 | DNA 上的做法 |
|------|---------|-------------|
| **MLM (Masked Language Model)** | BERT 式的"填空"：把一句话中的某些词遮起来，让模型猜 | 随机遮住 DNA 序列中 15% 的碱基，让模型根据上下文预测 |
| **CLM (Causal Language Model)** | GPT 式的"从左到右续写"：给前面的词，预测下一个词 | 从左到右依次预测下一个碱基是什么 |

#### 代码示例 1：MLM 的"填空"思维

```python
# 一段真实的 DNA 序列（人类第 1 号染色体上的启动子区域）
sequence = "ATGCGTACCTAGNNNNTTACGG"

# NNNN 是我们"遮起来"的位置，模型需要预测这里是什么
# 模型会看到周围的 ATGCGTACCTAG 和 TTACGG，然后猜 NNNN 最可能是什么

# 假设模型预测结果是：ATCG
# 那么模型学到的条件概率就是：
# P(ATCG | ATGCGTACCTAG, TTACGG)

# 这类似于 BERT 在 NLP 中的训练方式：
# 原文: "The cat sat on the [MASK]"
# BERT 预测: "mat"
```

#### 代码示例 2：CLM 的"续写"思维

```python
# 一段 DNA 序列，模型从左到右预测下一个碱基
sequence = "ATGCGTACCTAG"

# 第一步：看到 "A"，预测 "T"
# P(T | A)

# 第二步：看到 "AT"，预测 "G"
# P(G | AT)

# 第三步：看到 "ATG"，预测 "C"
# P(C | ATG)

# ... 一直继续，直到预测完整个序列

# 这就是 GPT 的方式。做这种预测，模型必须理解
# DNA 序列中的"语法"和"模式"。
```

### 2. 嵌入（Embedding）：把 DNA 变成数学向量

语言模型学到的最大宝藏是 **嵌入向量 (embedding)**。

**类比**：想象每段 DNA 序列被模型"翻译"成一串数字。相似的序列（比如都是启动子区域）会被翻译成相近的数字。

```python
# 假设一段 100bp 的 DNA 序列经过模型处理后，
# 模型给这段序列的每个碱基位置生成一个向量

# 位置 1: [0.23, -0.45, 0.12, ..., 0.87]  → 640维向量
# 位置 2: [0.19, -0.41, 0.15, ..., 0.83]  → 640维向量
# 位置 3: [0.31, -0.52, 0.08, ..., 0.91]  → 640维向量
# ...

# 如果你把这些向量画成图（降维到 2D），
# 你会发现：编码区、非编码区、启动子区域
# 会自动聚成不同的"团"——即使模型从未被告知
# 这些区域的名称是什么！这就是"无监督学习"的力量。
```

### 3. 对数似然比 (LLR)：判断一个突变是否有害

这是 gLM 最强大的应用之一。

**核心思想**：如果你把参考序列中的一个碱基 A 突变成 T，模型对突变后序列的概率评分下降了，说明模型认为这个突变是"不好的"——因为它没见过类似的组合。

```
LLR = log [ P(序列含A) / P(序列含T) ]

如果 LLR > 0：A 比 T 更"自然"，T 突变可能有害
如果 LLR < 0：T 比 A 更"自然"，A 可能是有害的

这个分数不需要任何疾病标签就能算出来！
```

### 4. Sequence Design：让模型"写"出全新的 DNA 序列

gLM 不仅能读，还能写。

```python
# 给模型一个"提示词"（control tag）：
prompt = "[HIGH_EXPRESSION]"

# 然后模型会生成一段全新的启动子序列：
generated_sequence = "TATAATGCGATCCGTTAGGCAATCG..."

# 这段序列的特征：
# - 具有启动子的结构模式
# - 模型"认为"它能驱动基因高表达
# - 但这段序列在自然界中从未出现过
```

## gLM 的三大应用

### 应用一：功能约束预测 (Functional Constraint Prediction)

**问题**：一段 DNA 序列中，哪些位置是"不可改动"的？

**为什么重要**：如果某个位置无论在哪个人类个体中都一模一样（高度保守），说明这个位置对生命至关重要——改变它可能致死。

**gLM 怎么做**：

1. 把一段 DNA 中某个位置"遮起来"，用 MLM 预测
2. 计算参考碱基和替代碱基的概率比（LLR）
3. LLR 越高 → 那个位置越重要

**实际效果**：
- GPN 模型在拟南芥基因组上实现了当前最佳性能
- 即使只训练了 **一个基因组**，LLR 分数也与自然界中该物种的等位基因频率高度相关

### 应用二：序列设计 (Sequence Design)

**问题**：能不能让 AI 设计一个全新的基因调控元件？

**gLM 怎么做**：
1. 给定"提示词"（如 `[HIGH_EXPRESSION]`）
2. 模型从左到右依次生成下一个碱基
3. 生成一段功能性的全新 DNA 序列

**具体案例**：
- **regLM**：训练出能设计启动子和增强子的模型
- **EVO**：训练出能设计全新的 CRISPR-Cas 系统的模型
- **MegaDNA**：能生成完整的噬菌体基因组（约 96kb）

### 应用三：迁移学习 (Transfer Learning)

**类比**：就像你学会了英文语法，再去学德文会比从零开始快得多。

**gLM 的做法**：
1. 先在大量基因组数据上预训练 gLM（像 BERT 一样）
2. 把预训练好的模型"微调"到具体任务上（如预测基因表达）

**实际案例**：
- **SegmentNT**：用 NT gLM 微调，在基因和顺式调控元件注释上达到 SOTA
- **AgroNT**：在 48 种植物上预训练，然后微调预测作物基因表达
- **IsoFormer**：跨模态迁移——用 DNA 模型和蛋白质模型一起做任务

## 开发 gLM 的关键考虑

### 训练数据

| 挑战 | 说明 |
|------|------|
| **数据质量** | 人类参考基因组中只有约 3.3% 的碱基是功能性的 |
| **重复序列** | 人类基因组中 50% 是重复序列，大多数 gLM 没有处理这个问题 |
| **数据多样性** | 一个物种的一个基因组不够，需要多个物种 |
| **上下文长度** | 真核生物基因组太大，无法一次性作为输入 |

### 模型架构

| 架构 | 特点 | 代表模型 | 最大上下文 |
|------|------|---------|-----------|
| **Transformer** | 自注意力机制，捕捉全局依赖 | NT, DNABERT | ~12kb |
| **CNN** | 擅长捕捉局部模式（如转录因子结合位点） | GPN | 较短 |
| **SSM (状态空间模型)** | 线性缩放，能处理超长序列 | HyenaDNA, EVO | 1M+ nucleotides |
| **CNN + Transformer 混合** | 兼顾局部和全局 | LOGO | 中等 |

**为什么 SSM 是未来**：

Transformer 的自注意力计算量是 O(n²)，序列长度翻倍，计算量翻四倍。而 SSM 是 O(n)，线性增长。对于动辄数百万碱基的基因组，SSM 是唯一实用的方案。

### 学习目标

| 目标 | MLM | CLM |
|------|-----|-----|
| 适合做嵌入和迁移学习 | ✅ 更好 | ⚠️ 一般 |
| 适合做序列生成 | ⚠️ 需渐进去掩码 | ✅ 自然 |
| SNP 变异分析 | 需伪 LLR | ✅ 一次查询即可 |

## 挑战与未来方向

### 1. 上下文长度的"不可能三角"

基因组需要长上下文来捕捉远距调控关系（增强子可能离目标基因 1Mb 远），但：
- Transformer 存不下（显存爆炸）
- SSM 能放下，但丢失了"全局注意力"

### 2. 人类基因组的特殊困难

人类基因组的 LLR 分数在功能约束预测上 **不如** 传统保守性分数。原因可能是：
- 人类功能性碱基比例太低（3.3%）
- 个体间变异太少
- 需要更多物种的对比数据

### 3. 扩展假设 (Scaling Hypothesis) 是否适用

在 NLP 中，数据量 + 计算量越大，模型越强。但在基因组学中，这个问题还在研究——蛋白质语言模型的一个研究发现：扩展只对结构预测有效，对其他任务（如功能预测）并没有提升。gLM 也需要同样的审视。

### 4. 可解释性

模型说"这个位置重要"，但我们仍然不知道 **为什么**。科学家需要知道具体的分子机制（如"这个突变影响了哪个转录因子"），而不仅仅是一个分数。

---

## 总结

gLM 的核心洞察一句话概括：

> **DNA 是一段用 {A, C, G, T} 写成的"语言"，语言模型学会了它的"语法"和"语义"后，就能预测突变后果、设计新序列、辅助功能注释。**

这就像 NLP 从规则系统（手写语法规则）走向深度学习（让数据说话）一样——**基因组学正在经历同样的范式转变**。

未来，一个能够"读懂"整个基因组的 AI 模型，可能彻底改变我们理解和编辑生命的方式。
