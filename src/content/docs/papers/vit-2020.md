## 一句话介绍

ViT（Vision Transformer）是 Google 在 2020 年提出的一种方法：**把图片切成小块，像处理文字一样用 Transformer 来识别图片**。它证明了 CNN（卷积神经网络）在视觉任务上并不是不可替代的。

## 从日常类比开始

### 类比 1：拼图 vs 扫视

想象你在认一张照片里的动物：

- **CNN（老方法）** 像一个"扫视者"：从左上角开始，一寸一寸地扫过整张图，边扫边记特征——"这里有条毛"、"那里有个鼻子"——最后拼出一个结论。这种"局部扫描再聚合"的方式就是卷积。
- **ViT（新方法）** 像一个"拼图玩家"：先把照片裁成若干个小方块（比如 16×16 像素一块），然后同时看所有方块，让每个方块和其他方块"商量"——"我这块看起来像耳朵，你那块像尾巴，咱们一起猜这是什么动物。"这种"互相商量"的机制叫 **self-attention（自注意力）**。

关键区别：CNN 是"串行扫描"，ViT 是"并行商量"。

### 类比 2：单词 vs 图片

NLP（自然语言处理）里，我们把句子拆成单词："我 / 喜欢 / 猫"。Transformer 就是为处理这种"单词序列"而生的。

ViT 的核心想法是：**图片也可以拆成"单词"**。每个 16×16 像素的小方块就是一个"单词"（论文里叫 **patch**）。一张 224×224 的图片被切成 196 个 patch，Transformer 把它们当成 196 个"单词"来处理。

## 核心概念

### 1. Patch Embedding（把图片变成"单词"）

一张 RGB 图片的尺寸是 224×224×3（高×宽×颜色通道）。ViT 用步长为 16 的卷积把它切成不重叠的 16×16 小块：

```
224 ÷ 16 = 14
14 × 14 = 196 个 patch
```

每个 patch 有 16×16×3 = 768 个像素值。把这个 768 维的向量通过一个线性层投影到 768 维（ViT-Base 的隐藏层维度）。这样每个 patch 就变成了一个 768 维的向量——这就是它的"词嵌入"。

### 2. Position Encoding（告诉模型"谁在哪里"）

Transformer 本身不知道输入的先后顺序（它不像 RNN 那样按顺序读）。所以在 NLP 中，我们给每个单词加上位置信息。ViT 同理——给每个 patch 向量加上一个 **可学习的位置编码**，让它知道自己在图片中的位置。

论文发现：用简单的 1D 位置编码就够了，不需要复杂的 2D 编码。模型会自动学会"相邻 patch 的位置编码应该相似"。

### 3. [CLS] Token（一个"班长"汇总全图）

在 196 个 patch 向量前面，ViT 额外加入了一个特殊的、可学习的向量，叫做 **[CLS] token**（这个设计来自 BERT）。经过所有 Transformer 层的计算后，[CLS] token 会汇聚所有 patch 的信息，它的最终输出就代表了"整张图片"。

### 4. Transformer Encoder（"互相商量"的核心）

ViT 用的是标准的 Transformer Encoder，和 BERT 用的完全一样：

- **Multi-Head Self-Attention（多头自注意力）**：每个 patch 都跟所有其他 patch 计算"相关性"，权重高的说明它们关系密切。比如"耳朵"的 patch 和"眼睛"的 patch 在识别猫的时候会互相关注。
- **MLP（多层感知机）**：对每个 patch 的信息做进一步处理。
- **LayerNorm + Residual Connection**：帮助训练稳定。

整个流程重复 12 次（ViT-Base 是 12 层），每次都在做一轮"全图协商"。

### 5. Classification Head（最后的判断）

[CLS] token 的输出接一个简单的两层 MLP 分类头，输出每个类别的概率。比如 ImageNet 有 1000 类，就输出 1000 个概率值。

## 数据是怎么流动的？（完整示例）

假设输入一张 224×224 的彩色图片：

```
输入: x ∈ R^(224 × 224 × 3)
       ↓
切成 196 个 patch，每个 16×16×3 = 768 维
       ↓
线性投影: 768 → 768（ViT-Base 的隐藏维度 D）
       ↓
前面加一个 [CLS] token → 序列长度变成 197
       ↓
加上位置编码（197 个位置）
       ↓
喂给 12 层 Transformer Encoder（每层包含 MSA + MLP）
       ↓
取 [CLS] token 的最终输出 z_L[0] ∈ R^768
       ↓
接分类头: 768 → 1000（ImageNet 的 1000 个类别）
       ↓
Softmax → 输出 1000 个类别的概率
```

## 代码示例

### 示例 1：用最简 PyTorch 实现 ViT 的 Patch 切割和嵌入

这是 ViT 最关键的一步——把图片变成 patch 序列。注意这里用了 `unfold` 技巧，等价于一个 kernel_size=16、stride=16 的卷积：

```python
import torch
import torch.nn as nn

class SimpleViTEncoder(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_channels=3,
                 hidden_dim=768, num_layers=12, num_heads=12, num_classes=1000):
        super().__init__()
        self.num_patches = (img_size // patch_size) ** 2  # 196
        self.patch_dim = patch_size ** 2 * in_channels     # 768

        # [1] Patch embedding: 把每个 patch 投影到 hidden_dim 维
        self.patch_proj = nn.Linear(self.patch_dim, hidden_dim)

        # [2] 可学习的 [CLS] token
        self.cls_token = nn.Parameter(torch.randn(1, 1, hidden_dim))

        # [3] 可学习的位置编码（196 个 patch + 1 个 CLS = 197）
        self.pos_embed = nn.Parameter(torch.randn(1, self.num_patches + 1, hidden_dim))

        # [4] 标准的 Transformer Encoder
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=hidden_dim,
            nhead=num_heads,
            dim_feedforward=hidden_dim * 4,  # MLP 中间层维度
            activation='gelu',
            batch_first=True
        )
        self.encoder = nn.TransformerEncoder(encoder_layer, num_layers=num_layers)

        # [5] 分类头: [CLS] 输出 → 类别数
        self.classifier = nn.Sequential(
            nn.LayerNorm(hidden_dim),
            nn.Linear(hidden_dim, num_classes)
        )

    def forward(self, x):
        # x: (B, C, H, W) = (batch, 3, 224, 224)

        # Patch 切割: (B, num_patches, patch_dim)
        # unfold 在 H 和 W 维度上切 16×16 的窗口，步长也是 16
        patches = x.unfold(2, 16, 16).unfold(3, 16, 16)  # (B, 14, 14, 16, 16, 3)
        patches = patches.reshape(x.size(0), -1, self.patch_dim)  # (B, 196, 768)

        # 线性投影到隐藏维度
        x = self.patch_proj(patches)  # (B, 196, 768)

        # 拼接 [CLS] token
        cls_tokens = self.cls_token.expand(x.size(0), -1, -1)  # (B, 1, 768)
        x = torch.cat([cls_tokens, x], dim=1)  # (B, 197, 768)

        # 加上位置编码
        x = x + self.pos_embed  # (B, 197, 768)

        # Transformer Encoder
        x = self.encoder(x)  # (B, 197, 768)

        # 取 [CLS] token 做分类
        cls_output = x[:, 0]  # (B, 768)
        logits = self.classifier(cls_output)  # (B, 1000)

        return logits


# 测试一下
model = SimpleViTEncoder()
dummy_input = torch.randn(2, 3, 224, 224)  # 2 张图片
output = model(dummy_input)
print(output.shape)  # torch.Size([2, 1000])  → 每张图 1000 个类别分数
```

这段代码展示了 ViT 的完整前向流程。关键点：
- `unfold` 把图片切成 patch 是 ViT 的入口
- 整个网络**没有卷积**（patch_proj 是线性层，不是 conv2d）
- 只用约 30 行就实现了完整的 ViT-Base

### 示例 2：手动计算 Attention 的过程（理解 Self-Attention）

下面的代码演示了 Self-Attention 到底在做什么。不需要理解所有细节，重点是感受"每个 patch 如何跟所有其他 patch 交互"：

```python
def simple_self_attention(Q, K, V):
    """
    简化的 Self-Attention 计算（去掉缩放和多头的细节）

    参数:
        Q, K, V: (batch, seq_len, d_model)
        在 ViT 中，Q=K=V（因为是 self-attention）
        seq_len = 197（196 个 patch + 1 个 CLS）
        d_model = 768
    """
    batch_size, seq_len, d_model = Q.shape

    # Step 1: 计算注意力分数 — 每个 query 跟每个 key 的点积
    # (B, seq_len, d_model) × (B, d_model, seq_len) → (B, seq_len, seq_len)
    attention_scores = torch.matmul(Q, K.transpose(-2, -1))

    # Step 2: 缩放（除以 sqrt(d_model) 防止梯度消失）
    scaled_scores = attention_scores / (d_model ** 0.5)

    # Step 3: Softmax 归一化 — 每一行加起来等于 1
    # 每一行代表"这个 patch 对其他所有 patch 的注意力权重"
    attention_weights = torch.softmax(scaled_scores, dim=-1)

    # Step 4: 用权重加权求和 V
    # (B, seq_len, seq_len) × (B, seq_len, d_model) → (B, seq_len, d_model)
    output = torch.matmul(attention_weights, V)

    return output, attention_weights


# 模拟 ViT 中的一层 Self-Attention
batch_size = 2
seq_len = 197      # 196 patches + 1 CLS
d_model = 768

# 模拟输入（实际中来自上一层的输出）
Q = torch.randn(batch_size, seq_len, d_model)
K = torch.randn(batch_size, seq_len, d_model)
V = torch.randn(batch_size, seq_len, d_model)

output, attn_weights = simple_self_attention(Q, K, V)

print("输入形状:", Q.shape)
print("输出形状:", output.shape)
print("注意力矩阵形状:", attn_weights.shape)
print("注意力矩阵一行之和:", attn_weights[0, 0].sum().item())  # 应该是 1.0

# 看第一个 patch（位置 0，即 CLS token）的注意力分布
# 它会关注哪些 patch？
top_k_indices = attn_weights[0, 0].topk(5).indices
print(f"CLS token 最关注的 5 个位置: {top_k_indices}")
```

运行这段代码你会看到：
- 注意力矩阵的形状是 **(197, 197)**——每个 patch 都跟所有其他 patch 建立了连接
- 每一行的和是 **1.0**——注意力权重被 softmax 归一化了
- CLS token 通常会均匀地关注所有 patch（因为它负责汇总全局信息）

### 示例 3：用 HuggingFace 加载预训练的 ViT（一行代码就能用）

实际项目中不需要自己写 ViT，HuggingFace 提供了开箱即用的实现：

```python
from transformers import ViTForImageClassification, ViTImageProcessor
from PIL import Image

# 加载预处理器和预训练模型
processor = ViTImageProcessor.from_pretrained('google/vit-base-patch16-224')
model = ViTForImageClassification.from_pretrained('google/vit-base-patch16-224')

# 读取图片并预处理
image = Image.open("cat.jpg").convert("RGB")

# 预处理: resize → normalize → 切成 patch → 转 tensor
inputs = processor(images=image, return_tensors="pt")

# 推理
with torch.no_grad():
    outputs = model(**inputs)

# 获取预测结果
predicted_class_idx = outputs.logits.argmax(-1).item()
print(f"预测类别: {model.config.id2label[predicted_class_idx]}")
```

HuggingFace 的 `google/vit-base-patch16-224` 就是在 ImageNet-21k 上预训练过的 ViT-Base，在 ImageNet-1k 上 fine-tune 后 top-1 准确率约 81.8%。

## 为什么 ViT 需要大量数据才能训好？

这是 ViT 最著名的特点，也是论文的核心发现之一。

**CNN 有"归纳偏置"（inductive bias）**：卷积操作天生假设"局部像素相关"和"平移不变性"。这就像给了模型一个"世界观预设"——它一开始就知道图片的基本结构。

**ViT 几乎没有这种预设**：它把 patch 当成无序的"单词"，全靠自注意力去学习 patch 之间的关系。这意味着它必须**从数据中自己学出**"相邻像素更相关"这个规律。

结果就是：
- **小数据集（< 10M 张图）**：CNN 赢了。ViT 没见过足够多的样本来学会正确的规律。
- **大数据集（> 30M 张图）**：ViT 反超了。当数据足够多时，"自己学"比"预设"更灵活、更强。

论文用一个表格清楚地展示了这一点（简化版）：

| 预训练数据 | ViT-L top-1 | ResNet-L top-1 | 谁赢 |
|-----------|-------------|----------------|------|
| ImageNet 1k (1.3M) | 77.5% | 82.1% | CNN |
| ImageNet 21k (14M) | 81.8% | 82.5% | 接近 |
| JFT-300M (300M) | **88.5%** | 87.5% | ViT |

这个发现改变了很多人的认知：**架构选择不是绝对的，取决于数据量**。

## 三个关键数字

1. **16×16**：patch 的大小。224÷16=14，正好整除。不是魔法数字，是实验结果——太小 token 太多算不动，太大单个 patch 信息太稀疏。
2. **196**：224×224 图片切成 16×16 patch 后的数量。14×14=196 个 token。加上 CLS 就是 197。
3. **300M**：JFT 数据集的图片数量。ViT 在这个规模上才真正超越 CNN。

## 踩过的坑

1. **直接在 ImageNet-1k 上训练效果很差**：原始 ViT 在 1.3M 张图上 top-1 只有 77%，比 ResNet 差 5 个百分点。后来 DeiT 用知识蒸馏 + 数据增强解决了这个问题。

2. **位置编码换分辨率要插值**：预训练时用 224×224（196 个位置编码），fine-tune 时换成 384×384（576 个位置），需要对位置编码做 2D 插值。这不是优雅的设计，是 hack。

3. **O(N²) 复杂度**：197 个 token 的注意力矩阵是 197×197≈39K 个元素。如果图片变成 1024×1024，token 数变成 4096，注意力矩阵就是 4096²≈16M——显存直接爆。这也是 Swin Transformer 和 Flash Attention 出现的原因。

4. **优化器要用 Adam，不能用 SGD**：论文发现 ViT 用 SGD 训练不稳定，Adam + 大 weight decay（0.1）才是正确配置。这跟传统 CV 训练习惯相反。

## 学到了什么

1. **一个架构可以通吃不同模态**：ViT 之前，NLP 用 Transformer、CV 用 CNN、音频用 RNN。ViT 之后，所有模态都可以用同一套 Transformer 架构。这是深度学习的第一次"架构大一统"。

2. **数据量决定了架构的选择**：CNN 赢了不是因为本质更好，而是因为"恰好"数据量在 CNN 的舒适区内。数据多了，CNN 的预设反而成了束缚。

3. **"Patch 化"是通用的序列化方法**：图片切 patch、音频切帧、视频切片段——把任意结构化数据变成序列，再喂给 Transformer。这个方法论影响深远。

4. **ViT 是多模态大模型的基石**：GPT-4V、Gemini、LLaVA 等所有多模态模型的视觉编码器都是 ViT。ViT 输出的 token 序列天然兼容 LLM 的 attention，这是它能成为事实标准的根本原因。
