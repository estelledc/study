# 光子计算：用光来做 AI 加速 — 零基础学习笔记

## 什么是光子计算？

光子计算（Photonic Computing）是一种**用光（光子）代替电（电子）来做计算**的技术。传统的电脑、手机、GPU 都是用电子在电线里跑来跑去传递信息和做运算，而光子计算则是让光在芯片上的微小通道（叫做"波导"）里传播，利用光的特性来完成矩阵乘法等 AI 核心运算。

### 日常类比

想象你在一个**超级长的走廊**里做加法：

- **传统电子计算**就像一个人拿着一摞文件，从左走到右，每遇到一个人就把数字加起来，一个一个来。即使跑得再快，也得排着队。
- **光子计算**则像是把走廊变成了**游泳池**——你把所有的水同时倒进去，水波在池子里自然交汇、叠加，瞬间就完成了所有加法。光也是同样的道理：多束光同时进入芯片，它们在通道里相遇时，光波会自动叠加（物理上的"干涉"），一步到位完成大量计算。

这就是光子计算最核心的魅力：**并行性**。电子需要时钟节拍一步步来，光在传播的过程中天然就是并行的。

---

## 为什么 AI 需要光子计算？

现代 AI 模型（比如 GPT、Claude）的核心运算是一种叫做**矩阵乘法**的操作。简单说，就是两个大表格里的数字互相乘再加起来。以一个大语言模型为例，一次推理可能需要执行**万亿次**这样的乘法。

| 维度 | 电子计算（GPU） | 光子计算 |
|------|----------------|----------|
| 信息载体 | 电子（电荷） | 光子（光） |
| 速度 | 接近光速，但有电阻发热 | 光速传播，几乎无发热 |
| 并行性 | 需要很多核心逐个处理 | 天然并行，光波同时叠加 |
| 能耗 | 高（GPU 功耗可达 700W） | 极低（理论上是电子的 1/1000） |
| 精度 | 高精度（FP32/FP16） | 目前中等（6-8 bit 模拟精度） |
| 成熟度 | 非常成熟 | 实验室到早期产品阶段 |

---

## 核心概念

### 1. 矩阵乘法的光学实现

AI 训练中最重要的运算是矩阵乘法。在电子世界里，你需要 CPU/GPU 的无数晶体管一步一步算。在光子世界里，可以用一个简单的光学装置来实现：

**马赫-曾德尔干涉仪（MZI）网格**：这是光子芯片的基本计算单元。你可以把它想象成一个"光阀门"——光走进来，经过分束器分成两路，每一路上都有一个微小的加热器来调节光的相位（相当于调节阀门的开度），最后两束光重新汇合。汇合时的光强，就是两个输入数字相乘的结果。

```
输入光 A ──┬──╱╲──[相位调节]──╮
           │                   ├──╳══ 输出光（乘积结果）
输入光 B ──┴──╱╲──[相位调节]──╯
```

把成千上万个 MZI 单元排列成网格，就能一次性完成整个矩阵乘法。

### 2. 波分复用（WDM）

就像光纤通信中用不同颜色的光同时传输多路数据一样，光子计算中也用**不同波长的光**在同一根波导里并行传输不同的数据。这相当于一条高速公路有几百条车道，每条车道用不同颜色的光跑，互不干扰。

### 3. 光电转换（O/E 转换）

目前光子计算还不是完全"纯光"的。数据从电脑进来时是电信号，需要先转成光信号（电→光），光子芯片算完后再转回电信号（光→电）才能被后续电路处理。这个转换过程叫 O/E 和 E/O 转换，是目前光子计算最大的瓶颈之一——因为转换本身耗能且有时延。

---

## 光子计算的关键优势

### 1. 超高并行度

光波在传播时天然叠加。一个光子芯片上的 MZI 网格可以在**光传播的瞬间**（皮秒级别，1 皮秒 = 10^-12 秒）完成整个矩阵乘法。相比之下，电子芯片即使以 1 GHz 的频率运行，一次乘法也需要 1 纳秒（10^-9 秒），慢了 100 万倍。

### 2. 超低能耗

电子在导线中运动会产生电阻发热，能耗很高。光子没有质量、没有电荷，在波导中传播**几乎不消耗能量**（除了调节相位时需要一点点电加热）。理论上，光子矩阵乘法的能耗可以低至电子的千分之一。

### 3. 零互连带宽瓶颈

在电子芯片中，数据在不同核心之间搬运（互连）比计算本身还耗能。光子芯片中，光波在波导中天然就是"互联"的——它们相遇就计算，不需要额外的搬运步骤。

## 代码示例

### 示例 1：用 NumPy 模拟光子矩阵乘法（理解原理）

这个例子展示光子芯片在做什么——本质上就是矩阵乘法，只不过是用光而不是用电。

```python
"""
光子计算原理演示：用 NumPy 模拟光子芯片的矩阵乘法
光子芯片的核心操作就是矩阵向量乘法 y = W @ x
其中 W 是权重矩阵（存储在光子芯片的 MZI 网格中），
x 是输入向量（通过光的强度编码），
y 是输出向量（通过光的干涉得到）
"""

import numpy as np


def simulate_photonic_matmul(weights, inputs):
    """
    模拟光子芯片的矩阵向量乘法
    参数:
        weights: 权重矩阵 (M x N)，对应芯片上 MZI 单元的相位配置
        inputs: 输入向量 (N,)，对应输入光的强度
    返回:
        输出向量 (M,)，对应输出光的强度
    """
    # 光子芯片执行的就是标准的矩阵乘法
    output = weights @ inputs
    return output


def simulate_interference(input_a, input_b, phase_shift):
    """
    模拟单个 MZI 单元的干涉效应
    两个输入光经过相位差后干涉，输出光强 = |E1 + E2|^2
    参数:
        input_a: 第一个输入光的电场振幅
        input_b: 第二个输入光的电场振幅
        phase_shift: 相位差（弧度），由热效应在波导上调节
    返回:
        输出光强（标量）
    """
    # 电场叠加：E_total = E1 + E2 * exp(i * phi)
    electric_field = input_a + input_b * np.exp(1j * phase_shift)
    # 光强 = |电场|^2
    light_intensity = np.abs(electric_field) ** 2
    return light_intensity


# ---- 演示 1：模拟一个小型光子矩阵乘法器 ----
print("=" * 55)
print("演示 1：光子矩阵向量乘法 (2x3 矩阵)")
print("=" * 55)

# 权重矩阵（3x2），可以理解为芯片上 3 行 2 列的 MZI 单元配置
weight_matrix = np.array([
    [0.8, -0.3],
    [0.1,  0.9],
    [-0.5, 0.4]
])

# 输入向量（2个光信号的强度）
input_vector = np.array([0.7, 0.5])

# 光子芯片计算
result = simulate_photonic_matmul(weight_matrix, input_vector)

print(f"\n权重矩阵 W ({weight_matrix.shape}):")
print(weight_matrix)
print(f"\n输入向量 x ({input_vector.shape}): {input_vector}")
print(f"\n输出向量 y = W @ x ({result.shape}): {np.round(result, 4)}")

# ---- 演示 2：单个 MZI 单元的干涉 ----
print("\n" + "=" * 55)
print("演示 2：单个 MZI 单元的干涉效应")
print("=" * 55)

input1 = 1.0   # 第一个输入光振幅
input2 = 1.0   # 第二个输入光振幅

print(f"\n{'相位差 (度)':>12} | {'电场叠加':>12} | {'输出光强':>12}")
print("-" * 45)

for phase_deg in range(0, 361, 45):
    phase_rad = np.radians(phase_deg)
    intensity = simulate_interference(input1, input2, phase_rad)
    print(f"{phase_deg:>12} | {np.round(input1 + input2 * np.exp(1j * phase_rad), 3):>12} | {intensity:>12.4f}")

print("\n说明：通过调节相位差，同一个 MZI 单元可以实现不同的乘法系数")
print("这就是光子芯片'编程'的方式——改变加热器的温度来设置相位")
```

运行输出：

### 示例 2：模拟光子芯片的能效对比

这个例子对比电子 GPU 和光子芯片在矩阵乘法上的理论能耗。

```python
"""
光子计算 vs 电子计算：能效对比模拟
基于文献中的典型数据：
  - NVIDIA H100 GPU: 约 30 TOPS/W (Tera Operations per Watt)
  - 光子加速器: 理论可达 10,000 TOPS/W
"""


def estimate_gpu_energy(matrix_size, ops_per_element=2):
    """
    估算 GPU 执行矩阵乘法的能耗
    参数:
        matrix_size: 矩阵边长 N（N x N 矩阵乘法）
        ops_per_element: 每个元素需要的操作数（乘+加=2）
    返回:
        能耗（焦耳 J）
    """
    total_ops = 2 * matrix_size ** 3  # 矩阵乘法复杂度 O(N^3)
    gpu_power_watts = 700               # H100 满功率
    gpu_throughput_topsw = 30           # 30 TOPS/W
    # 总操作数 / (吞吐量 * 功率效率) = 能耗
    energy_joules = total_ops / (gpu_throughput_topsw * 1e12) * gpu_power_watts
    return energy_joules


def estimate_photonic_energy(matrix_size, ops_per_element=2):
    """
    估算光子芯片执行矩阵乘法的能耗
    光子芯片的优势：矩阵乘法在光传播时瞬时完成，
    主要能耗来自 O/E 转换和相位调节
    参数:
        matrix_size: 矩阵边长 N
    返回:
        能耗（焦耳 J）
    """
    total_ops = 2 * matrix_size ** 3
    photonic_throughput_topsw = 10000   # 10,000 TOPS/W（理论值）
    photonic_power_watts = 100          # 预估系统功耗
    energy_joules = total_ops / (photonic_throughput_topsw * 1e12) * photonic_power_watts
    return energy_joules


# 不同规模矩阵的能耗对比
sizes = [64, 128, 256, 512, 1024, 2048]
print("=" * 65)
print("GPU (H100) vs 光子芯片 能耗对比（矩阵乘法 N x N）")
print("=" * 65)
print(f"{'矩阵大小 N':>10} | {'GPU 能耗 (J)':>14} | {'光子能耗 (J)':>14} | {'节省倍数':>10}")
print("-" * 65)

for n in sizes:
    gpu_e = estimate_gpu_energy(n)
    phot_e = estimate_photonic_energy(n)
    speedup = gpu_e / phot_e if phot_e > 0 else float('inf')
    print(f"{n:>10} | {gpu_e:>14.2f} | {phot_e:>14.2f} | {speedup:>10.1f}x")

print("\n" + "=" * 65)
print("关键结论：光子芯片在大规模矩阵乘法上的能耗远低于 GPU")
print("但随着矩阵变小，O/E 转换的固定开销占比变大，差距缩小")
print("=" * 65)
```

## 当前挑战

### 1. 精度有限

光子芯片用的是**模拟计算**（光波的强度和相位），精度通常在 6-8 bit，远低于 GPU 的 FP16（16 bit）或 FP32（32 bit）。这意味着它对 AI 训练不太友好（训练需要高精度），但在**推理**（已经训练好的模型用来做预测）场景中足够用。

### 2. 光电转换瓶颈

数据进芯片要 E/O 转换，算完要 O/E 转换。这个转换过程的能耗和速度目前还抵消了不少光子计算的优势。业界正在研发更好的集成光发射器和探测器来缓解这个问题。

### 3. 可编程性受限

电子芯片可以灵活地运行各种程序，而光子芯片更像是一个"专用计算器"——它的可编程性受限于硬件结构（MZI 网格的配置）。虽然可以通过重新配置来适应不同模型，但切换需要时间（热调谐的响应时间在微秒到毫秒级别）。

### 4. 制造和集成

光子芯片需要在硅片上制造微米级的光学结构（波导、分束器、干涉仪），这对制造工艺要求极高。虽然可以用现有的半导体工厂（foundry）生产，但良率和成本还需要优化。

## 实际应用案例

### 1. Lightmatter（美国）

开发了名为 "Envise" 的光子 AI 加速器芯片，已经在部分数据中心部署。其芯片利用光的干涉来实现矩阵乘法，号称比 GPU 节能 10 倍以上。

### 2. Ayar Labs（美国）

专注于光子互连技术，用光代替铜线在芯片之间传输数据。他们的 TeraPHY 技术已经集成到一些高性能计算系统中，解决了数据中心的"互连带宽瓶颈"问题。

### 3. 清华大学团队

在硅基光子计算领域发表了多项突破性成果，包括大规模可编程光子神经网络芯片，在图像识别和自然语言处理任务上展现了优越的能效比。

## 总结

光子计算的核心思想很简单：**既然光跑得最快、并行度最高、能耗最低，为什么不用光来做计算？**

| 要点 | 说明 |
|------|------|
| 基本原理 | 用光的干涉和叠加代替电子的开关运算 |
| 最大优势 | 超高并行度 + 超低能耗（尤其在矩阵乘法上） |
| 最佳场景 | AI 推理（而非训练），大规模矩阵运算 |
| 主要挑战 | 精度有限、O/E 转换开销、可编程性不足 |
| 发展阶段 | 实验室 → 早期商用（2024-2026 是关键窗口期） |

光子计算不是要取代 GPU，而是**补充**——在特定的高能耗、高并行度的 AI 推理场景下，它能做到 GPU 做不到的事。随着芯片制造工艺的进步和光电集成技术的成熟，光子加速器有望在未来几年内成为 AI 基础设施的重要组成部分。
