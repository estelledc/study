## 是什么

Chain-of-Thought（思维链）是一种**让大语言模型在给出最终答案之前，先一步步写出中间推理过程**的提示方法。

先做一个日常类比。

想象你正在做一道数学应用题。你有两种答题方式：

方式 A：直接把答案写上去
```
题目：食堂有 23 个苹果，下午用了 20 个做午餐，又买了 6 个。现在有几个？
答案：27
```

方式 B：先把步骤写出来，再写答案
```
题目：食堂有 23 个苹果，下午用了 20 个做午餐，又买了 6 个。现在有几个？
步骤：23 - 20 = 3，3 + 6 = 9
答案：9
```

人类从小被教导用方式 B，因为写步骤能帮你发现错误。大语言模型呢？它默认用方式 A——直接从输入跳到答案。

Chain-of-Thought 的核心发现就是：**如果你在做题前，给模型看几个"先写步骤、再给答案"的例子（这叫 few-shot 示例），模型就会跟着学，也开始先写步骤再答。这一招，不改模型、不微调，就能让推理任务的正确率翻好几倍。**

这篇论文 2022 年 1 月由 Google Brain 的 Jason Wei、Xuezhi Wang 等 9 位作者发表在 arXiv 上，被公认为大模型"推理能力"研究的开山之作。之后所有的"先思考再回答"现象——包括 Claude 的 thinking、OpenAI o1 系列——追根溯源都来自这里。

## 核心概念

### 1. 标准提示 vs. 思维链提示

大语言模型默认"看到输入就猜输出"。标准提示（standard prompting）就是给出几个输入-输出的例子，让模型猜下一个输出。

思维链提示（chain-of-thought prompting）在标准提示的每个示例里多塞了一样东西：**中间推理过程**。

结构从 `<输入, 输出>` 变成了 `<输入, 思维链, 输出>`。思维链就是一系列自然的语言推理步骤，引导到最终答案。

### 2. 涌现能力（Emergent Ability）

这是论文最重要的发现之一。思维链的效果**不是随模型变大而线性提升的**——它是一个台阶。

小模型（几百亿参数以下）用了思维链，效果反而比不用还差。因为小模型生成的"思维链"看起来 fluent（通顺），但里面全是胡说八道的逻辑。只有跨过大约 100B（千亿）参数这个门槛，思维链才真正起作用。这个现象叫"涌现"——能力不是慢慢增长的，而是某个阈值到了之后突然冒出来的。

### 3. 为什么思维链有效（论文提出的三个原因）

第一，**分解**。思维链把多步问题拆成中间步骤，让模型能把额外的计算资源花在更难的问题上。

第二，**可解释性**。你看到了模型的推理路径，能 debug 它在哪一步出错了。

第三，**通用性**。数学、常识、符号操作都能用同一种方式——用自然语言把思考过程写出来。

### 4. 对照实验——拆掉看还灵不灵

论文做了三个"破坏性"对照实验，证明"自然语言 + 逐步推理"这个组合缺一不可：

- 只让模型写算式，不写自然语言推理：对复杂题目没用（语义太复杂，不能直接翻译为算式）
- 让模型写一堆点 `....` 假装在思考，但不给实际内容：没用（证明不是"多写几个字"就行的）
- 让模型先写答案，再写思维链：没用（证明模型真的依赖思维链来"推导"，不是写完答案再找理由）

这三个实验排除了"只是多算了几步"或"只是激活了知识"的替代解释。

### 5. 三种变体

思维链在论文之后发展出三个主要变体，按"用起来多省事"排序：

**Few-shot CoT**（原版）：在 prompt 里放 8 个"问题 + 推理过程 + 答案"的例子，再让模型补全新问题。效果最稳，但示例需要手写。

**Zero-shot CoT**（半年后 Kojima 发现）：不用任何示例，在 prompt 末尾加一句"Let's think step by step."大模型就会自动展开推理。一句话魔法。

**Self-Consistency**（同年 Wang 提出）：让模型对同一道题做 40 遍，取出现频率最高的答案投票。比单次 CoT 再涨 5-10 个百分点，代价是算力 ×N。

## 代码示例

### 示例 1：数学应用题——Few-shot CoT

下面用 OpenAI API 演示。关键区别在 `prompt` 里：标准提示 vs. 思维链提示。

**标准提示（不加思维链）：**

```python
import openai

# GSM8K 风格的数学应用题
question = "詹妮芙原来有 56 朵花。她把 8 朵给了妈妈，又把 3 朵给了爸爸。现在她还有多少朵？"

prompt_standard = f"Q: {question}\nA:"

response = openai.Completion.create(
    engine="text-davinci-002",   # GPT-3 175B
    prompt=prompt_standard,
    temperature=0,
    max_tokens=50
)

print(response.choices[0].text.strip())
# 模型可能直接输出："45"
# 但它可能做错——没有中间步骤可检查
```

输出可能完全正确，也可能算错。对于单步问题（56-8=48）模型通常能答对。但对于多步问题（如 GSM8K 上的题），准确率只有约 17.9%。

**思维链提示（加上推理步骤的示例）：**

```python
# 关键改动：在示例的答案里加入推理过程
exemplar_a = (
    "Q: 厨房原来有 9 台电脑。每天加 5 台，加了 4 天。"
    "先算 5 * 4 = 20，再加原来的 9。20 + 9 = 29。答案是 29。"
)
exemplar_b = (
    "Q: 特劳特贝格有 22164 居民。阿沙芬堡有 88639 居民。"
    "两者相加：22164 + 88639 = 110803。答案是 110803。"
)
exemplar_c = (
    "Q: 温迪有 25 个气球。给了莉兹 6 个，给妮基 6 个。"
    "先算给了多少人：6 + 6 = 12。再算剩下：25 - 12 = 13。答案是 13。"
)

prompt_cot = f"""{exemplar_a}

{exemplar_b}

{exemplar_c}

Q: {question}
A:"""

response_cot = openai.Completion.create(
    engine="text-davinci-002",
    prompt=prompt_cot,
    temperature=0,
    max_tokens=150
)

print(response_cot.choices[0].text.strip())
# 模型会这样回答：
# "詹妮芙原来有 56 朵。给了妈妈 8 朵，还剩 56 - 8 = 48。
#  又给了爸爸 3 朵，还剩 48 - 3 = 45。答案是 45。"
```

效果对比（PaLM 540B 在 GSM8K 上）：

| 提示方式 | 正确率 |
|---------|--------|
| 标准提示 | 17.9% |
| 思维链提示 | 56.9% |

**不改模型、不微调，三个数字就让正确率翻了 3 倍。**

### 示例 2：常识推理——策略问题回答

思维链不只适用于数学。论文在常识推理任务（StrategyQA，要求回答"亚里士多德用过笔记本电脑吗？"这种需要多步推理的问题）上也有显著效果。

```python
# StrategyQA 风格的常识推理题
question = "亚里士多德用过笔记本电脑吗？"

prompt_cot = """Q: 棒球比赛需要投球吗？
A: 投球是棒球的核心动作。每场比赛投手都会投球。
所以答案是"是"。

Q: 章鱼有三颗心吗？
A: 章鱼有一颗主心（给全身供血），两颗鳃心（给鳃供血）。
所以答案是"是"。

Q: 亚里士多德用过笔记本电脑吗？
A:"""

response = openai.Completion.create(
    engine="text-davinci-002",
    prompt=prompt_cot,
    temperature=0,
    max_tokens=150
)

print(response.choices[0].text.strip())
# 模型会这样推理：
# "亚里士多德生活在公元前 384-322 年。
#  笔记本电脑是 20 世纪末发明的。
#  所以亚里士多德不可能用过笔记本电脑。
#  所以答案是"否"。"
```

这个例子展示了思维链的第二个威力：**把常识分解为可验证的步骤**。每一步（亚里士多德的生活年代、笔记本电脑的发明年代）都可以单独检查是否正确。

### 示例 3：Zero-shot CoT——一句话魔法

半年后，Kojima 等人发现了一个更简单的变体：你连 few-shot 示例都不需要了。只要在大模型的 prompt 末尾加一句"Let's think step by step."就行。

```python
question = "我有 5 个苹果。吃了 2 个，又买了一袋 8 个。现在有几个？"

prompt_zero_shot = f"Q: {question}\nA: Let's think step by step."

response = openai.Completion.create(
    engine="text-davinci-002",
    prompt=prompt_zero_shot,
    temperature=0,
    max_tokens=100
)

print(response.choices[0].text.strip())
# 模型自动展开：
# "我原来有 5 个苹果。吃了 2 个，还剩 5 - 2 = 3 个。
#  又买了 8 个，所以现在有 3 + 8 = 11 个。答案是 11。"
```

零示例、零调优，一句话就让 GSM8K 正确率从 17% 跳到 43%。这说明大模型其实"会"推理——它们只是默认习惯直接给出答案。prompt 起的是**触发**作用，不是**教学**作用。

## 实验结果速览

论文评估了三个大任务族上的效果：

**算术推理（5 个数据集）**
GSM8K（小学奥数题）是 hardest。PaLM 540B 用思维链达到 56.9% 正确率，超过了之前所有用训练数据微调过的模型。

**常识推理（5 个数据集）**
在 StrategyQA 上，PaLM 540B 从 69.4% 提到 75.6%，超过之前的监督学习最优。在 Sports Understanding 上，模型 95.4% 的正确率超过了人类体育爱好者的 84%。

**符号推理（2 个任务）**
"拼接名字最后一个字母"和"硬币翻转"两个任务上，思维链让模型能泛化到比示例更长的序列——这是标准提示做不到的。

## 踩过的坑

**坑 1：小模型用思维链反而变差。** 小于 100B 参数的模型生成的"思维链"看起来通顺但逻辑错。在本地跑 7B 模型时别直接套思维链示例。

**坑 2：思维链不等于真推理。** Turpin 2023 证明，模型可能是先决定答案，再编理由（事后合理化）。在 prompt 里植入偏见测试时，思维链模型仍按偏见给出答案，但推理过程完全不提偏见。**别把 CoT 输出当作模型真实的内部推理过程。**

**坑 3：简单题别加思维链。** 单步任务（"2 + 2 = ?"）加思维链效果反而下降，还浪费 token。思维链是"多步推理任务的开关"，不是万能药。

## 适用与不适用场景

**适用：**
- 多步数学题、逻辑推理、规划任务
- 需要"看到模型怎么想"的解释性场景
- 大模型（≥ 100B 参数，或经过 RLHF / RL 训练的现代模型）

**不适用：**
- 单步事实问答（"巴黎是哪个国家的首都？"）—— 浪费 token
- 小模型（< 7B 原始预训练模型）—— 反而变差
- 需要严格可解释的医疗 / 法律决策—— CoT 输出可能只是事后合理化

## 从论文到今天的演化

这篇论文上传 arXiv 之后，一年内发生了很多事：

**2022-03** Wang 等人提出 Self-Consistency——多次采样投票，比单次 CoT 再涨 5-10%。

**2022-05** Kojima 等人发现 "Let's think step by step."——零样本思维链。

**2022-10** Yao 等人提出 ReAct——把 CoT 和工具调用结合起来，Thought → Action → Observation → Thought，成为 Agent 范式的源头。

**2023-05** Tree of Thoughts 把"线性链"扩展为"树形搜索"，在 24 点游戏等任务上超过 CoT。

**2024-09** OpenAI 的 o1 用强化学习把 CoT 训练进了模型权重——从 prompt trick 变成模型内置能力。

**2025-01** DeepSeek R1 用 RL-CoT 训练让 7B 小模型也能有推理能力——部分推翻了"需要 100B 参数才涌现"的说法。

## 学到了什么

1. **Prompt 不只是"问问题"**——它还能改变模型回答的"形态"，从"直接答"变成"先推再答"，仅靠示例引导。
2. **大模型里有大量隐藏能力，需要正确的提示来触发。** 这是 prompt-engineering 整门学问的起点。
3. **方法的可信度来自对照实验。** 原论文做了三个破坏性对照实验，全部失败，才证明"自然语言 + 逐步推理 → 答案"这个组合不可拆。
4. **从 Prompt 到 RL 的训练范式：** 技术演化常常是先发现现象（2022 的 prompt trick），再训练成内置能力（2024 的 o1）。
