# Rust 语言设计：所有权与借用

## 一、从一个日常类比开始

想象你有一个**充电宝**。这个充电宝只能被一个人同时使用：

- 如果你把充电宝**递给**朋友，那它就不再在你手上了。朋友用完之后，充电宝归朋友管。
- 如果你只是让朋友**看一眼**充电宝（拍照、读上面的标签），充电宝还在你手上，朋友只是"借用"了一下。
- 但如果你让朋友**拆开充电宝改 wiring**，那在朋友改的时候，你不能同时拆开它——否则两人拧到一起去就炸了。

Rust 的**所有权（Ownership）**和**借用（Borrowing）**系统，就是给电脑内存立了这样一套规矩。

## 二、为什么需要这套规矩？

几乎所有程序都要管理内存。不同语言的做法不一样：

- **Python / Java**：有一个"垃圾回收器"（Garbage Collector, GC）在后台自动清理不再使用的内存。方便，但偶尔会卡顿。
- **C / C++**：程序员手动分配（`malloc`）和释放（`free`）内存。灵活，但容易忘记释放（内存泄漏），或者释放得太早（使用已释放的内存）——这些都是严重的 bug。
- **Rust**：第三种选择。编译器在**编译时**就帮你检查内存安全，不需要 GC，也不需要你手动释放。

Rust 靠什么做到这一点？就是所有权系统。

## 三、内存的两个仓库：栈与堆

在深入之前，先理解电脑内存的两个区域：

### 栈（Stack）—— 摞盘子的架子

- 数据按顺序**从上往下**放（Last In, First Out）
- 取数据时从**最上面**拿
- 速度快，但只能存**大小已知且固定**的数据
- 类比：餐厅里摞盘子，新盘子放最上面，取也取最上面的

### 堆（Heap）—— 找桌子的餐厅

- 按需分配，**不固定顺序**
- 分配时需要"查找空位、标记已用"，稍慢
- 可以存**大小未知**的数据（比如用户输入的文字）
- 类比：餐厅里你说"6 个人"，服务员帮你找一张能坐 6 人的桌子

**栈上存指针，堆上存实际数据。** 指针很小、大小固定，放栈上；真正的文字内容大小不定，放堆上。

## 四、所有权的三条规则

1. 每个值都有且仅有**一个所有者（owner）**
2. 同一时刻，只能有**一个**所有者
3. 当所有者**离开作用域（scope）**时，值自动被丢弃（drop）

这三条规则是 Rust 内存安全的基石。

## 五、核心概念一：移动（Move）

先看一个例子。Rust 中，基本类型（如整数）赋值时是**复制**：

```rust
let x = 5;
let y = x;
// x 仍然可以使用，因为整数大小已知，直接复制了值
println!("x = {}, y = {}", x, y);
```

但**复杂类型**（如 String）不一样：

```rust
let s1 = String::from("hello");
let s2 = s1;
// s1 被"移动"到了 s2，s1 不能再用了！
// println!("{}", s1); // 编译错误：use of moved value: `s1`
println!("{}", s2); // 没问题
```

为什么？`String` 在内存中是三部分：

```
栈上：  s1 → [ 指针 ─┐
              长度 5  │  ← 这三部分在栈上，大小固定
              容量 5 ─┘
            堆上：  "h e l l o"  ← 实际文字内容在堆上
```

`let s2 = s1` 时，Rust 只复制了栈上的三个数字（指针、长度、容量），**没有复制堆上的文字**。如果两个变量都指向同一块堆内存，当它们分别离开作用域时就会尝试**释放两次同一块内存**——这就是"双重释放（double free）"漏洞，会导致内存损坏和安全问题。

Rust 的解决方案：**让 s1 失效**。移动之后只有 s2 有效，释放时只释放一次。这在 Rust 里叫 **move**。

> 如果你想真正复制一份堆上的数据，可以用 `s2 = s1.clone()`，但 clone 是显式操作，提醒你"这里比较贵"。

## 六、核心概念二：借用（Borrowing）

如果我想让一个函数**使用**某个值，但不想交出所有权怎么办？答案：**借用**。

借用用 `&` 表示，类比现实中的"借东西"——用完还回来，你不拥有它。

```rust
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // 把 s1 借给函数
    println!("{} 的长度是 {}", s1, len); // s1 仍然可以使用！
}

fn calculate_length(s: &String) -> usize {
    s.len() // 只读，不修改
}
```

这里发生了什么事：

1. `&s1`：创建 s1 的**引用（reference）**，不转移所有权
2. `s: &String`：函数参数类型是"对 String 的引用"
3. 函数结束时 s 被丢弃，但**s1 指向的堆数据不会被释放**——因为 s1 还是所有者

这叫 **不可变借用**。

### 可变借用

如果函数需要**修改**借来的值：

```rust
fn main() {
    let mut s = String::from("hello");
    change(&mut s); // 可变借用
    println!("{}", s); // "hello world"
}

fn change(s: &mut String) {
    s.push_str(", world");
}
```

## 七、借用的两条铁律

### 铁律 1：要么一个可变引用，要么任意多个不可变引用

```rust
let mut data = String::from("hello");
let r1 = &mut data; // OK：第一个可变引用
let r2 = &mut data; // 编译错误！不能同时有两个可变引用
```

**为什么？** 如果两个人同时在同一块内存上写数据，就会发生**数据竞争（data race）**：

- 两个或多个指针同时访问同一数据
- 至少一个指针在写
- 没有同步机制

数据竞争会导致未定义行为。Rust 在**编译时**就消灭了它。

### 铁律 2：引用必须始终有效

```rust
fn main() {
    let s = String::from("hello");
    let r = &s;     // 引用 s
    drop(s);        // 错误！s 被释放了，r 成了"悬垂引用"
    println!("{}", r);
}
```

编译器保证：引用指向的数据不会比引用本身更早失效。这解决了 C/C++ 中最臭名昭著的**悬垂指针**问题。

## 八、核心概念三：切片（Slices）—— 更安全的借用

切片让你引用**连续的数据片段**，而不是整个数据。最常用的是字符串切片 `&str`：

```rust
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[..i];  // 返回空格之前的部分
        }
    }
    s // 没有找到空格，返回整个字符串
}

fn main() {
    let text = String::from("hello world");
    let word = first_word(&text);
    println!("第一个单词: {}", word); // "hello"

    // 下面这行编译错误！因为 word 借用自 text，
    // 而 clear 需要可变借用，两者冲突：
    // text.clear();
}
```

切片是一个**双字段结构**：指向起始位置的指针 + 长度。它不拥有数据，只是"借用一段"。

## 九、Copy trait：栈上数据的特权

某些简单类型（整数、布尔、字符等）实现了 `Copy` trait。它们的赋值是**逐位复制**，不涉及堆内存，所以所有权规则更宽松：

```rust
let x = 42;      // i32 实现了 Copy
let y = x;       // 复制，x 仍然有效
let z = x;       // 再复制，x 还是有效
println!("x={}, y={}, z={}", x, y, z); // 100% OK
```

实现了 `Drop` trait（需要特殊清理的资源）的类型**不能**实现 `Copy`，因为不能随意复制。

## 十、总结：一句话记忆

| 概念 | 符号 | 类比 | 规则 |
|------|------|------|------|
| 所有权 | 无 | 你拥有充电宝 | 一个值一个主人 |
| 移动 | `=` | 把充电宝递给别人 | 移动后原变量失效 |
| 不可变借用 | `&` | 让别人看一眼 | 可以同时有多个 |
| 可变借用 | `&mut` | 让别人拆开改装 | 只能有一个，且不能和不可变借用共存 |

Rust 的设计哲学是：**在编译时做尽可能多的检查，运行时不做任何额外开销**。所有权系统就是这一哲学的核心体现——它让程序在跑起来之前就发现内存问题，而不是等到运行时崩溃。
