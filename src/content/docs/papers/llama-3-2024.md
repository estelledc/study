# Llama 3 Technical Report 学习笔记

## 一、什么是 Llama 3？打个比方

想象一家大型出版社（Meta），它要培养一批"超级编辑"（语言模型）。

这些编辑能做的事很多：
- 写文章（自然语言理解与生成）
- 写代码（编程能力）
- 做数学题（推理能力）
- 用工具（调用外部 API、搜索引擎等）
- 说多种语言（多语言能力）

Llama 3 就是这个出版社在 2024 年培养出的最新一批超级编辑，一共有三个"字号"：8B（80 亿参数）、70B（700 亿参数）、405B（4050 亿参数）。数字越大，编辑读过的书越多，能力越强，但需要的资源也越多。

> **类比映射**：参数 = 编辑脑子里记住的"知识点"数量。参数越多，能处理的任务越复杂。

## 二、核心概念拆解

### 2.1 两个训练阶段

Llama 3 的培养分为两步：

**第一阶段：预训练（Pre-training）**
编辑被扔进图书馆，疯狂阅读 15 万亿个 token（可以粗略理解为"字"），学习语言的基本结构和世界知识。这个阶段的任务很简单——"预测下一个词是什么"。就像让小学生做填空题，填得多了，自然就会写了。

**第二阶段：后训练（Post-training）**
编辑虽然读了很多书，但还不知道怎么听话、怎么帮忙。这个阶段通过三种技术来"调教"：

1. **监督微调（SFT）**：老师给出问题和答案，编辑模仿学习
2. **拒绝采样（RS）**：从多个回答中挑出更好的，让编辑学习"什么是不好的"
3. **直接偏好优化（DPO）**：直接告诉编辑"这个回答比那个好"，编辑自己学会排序

> **类比映射**：预训练 = 通识教育；后训练 = 职业培训 + 职业道德教育

### 2.2 关键架构改进

Llama 3 对底层"大脑结构"做了几处小优化：

- **分组查询注意力（GQA）**：原来每个注意力头都有自己的"记忆"（key/value），GQA 让 8 个头共享一组记忆，推理速度更快，需要的内存更少
- **更大的词表**：从 Llama 2 的 32K 涨到 128K 个 token，相当于编辑的词汇量翻了 4 倍，每次能"读懂"更多文字
- **更长的上下文窗口**：从 4K 扩展到 128K token，相当于编辑能一次性读完一本书而不忘记开头

### 2.3 数据配比

预训练数据的"食谱"非常讲究：

| 数据类别 | 占比 | 说明 |
|---------|------|------|
| 通用知识 | 50% | 百科全书、新闻、百科等 |
| 数学与推理 | 25% | 数学证明、逻辑推理内容 |
| 代码 | 17% | GitHub、技术文档等 |
| 多语言 | 8% | 非英语的高质量文本 |

> **类比映射**：就像做饭不能只吃一种菜，模型也需要"均衡饮食"才能全面发展。

### 2.4 缩放定律（Scaling Laws）

研究人员发现了一个规律：**模型大小 × 训练数据量 = 一个最优平衡点**。

他们像做化学实验一样，先用小模型做测试，预测大模型的表现，最终确定 405B 参数是他们的计算预算下的"最优选择"。训练花了 3.8 × 10^25 次浮点运算，比 Llama 2 大了近 50 倍。

## 三、代码示例

### 3.1 用 Hugging Face 加载 Llama 3 模型

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

# 加载 Llama 3 8B 模型的词元器和模型
model_name = "meta-llama/Meta-Llama-3-8B"

tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",      # 自动选择数据类型（如 FP16/BF16）
    device_map="auto"        # 自动选择 GPU/CPU
)

# 准备输入文本
text = "请问什么是人工智能？请用简单的话回答。"
inputs = tokenizer(text, return_tensors="pt").to(model.device)

# 生成回答
outputs = model.generate(
    **inputs,
    max_new_tokens=256,      # 最多生成 256 个新 token
    temperature=0.7,         # 创造性程度（0=确定，1=随机）
    top_p=0.9                # 只考虑概率最高的 token 集合
)

# 解码输出
response = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(response)
```

> **说明**：`return_tensors="pt"` 告诉模型"给我 PyTorch 格式的张量"；`max_new_tokens` 控制生成长度；`temperature` 越低回答越确定，越高越有创意。

### 3.2 对话场景（Instruct 版本）

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

model_name = "meta-llama/Meta-Llama-3-8B-Instruct"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",
    device_map="auto"
)

# 构造对话格式
messages = [
    {"role": "user", "content": "请解释Transformer架构，就像我是初学者一样"},
]

# 用 tokenizer 把对话转成模型能理解的格式
input_text = tokenizer.apply_chat_template(
    messages,
    tokenize=False,          # 不编码成数字，先拿到字符串
    add_generation_prompt=True  # 添加"请回答"的提示符
)

inputs = tokenizer(input_text, return_tensors="pt").to(model.device)
outputs = model.generate(**inputs, max_new_tokens=512, temperature=0.7)

# 只显示新生成的部分
response = tokenizer.decode(outputs[0][inputs.input_ids.shape[1]:], skip_special_tokens=True)
print(response)
```

> **说明**：Instruct 版本用了特殊的对话格式（`apply_chat_template`），每个角色（user/assistant）都有标记，模型知道什么时候该"说话"什么时候该"听"。`input_ids.shape[1]` 是输入的长度，切片 `[inputs.input_ids.shape[1]:]` 只取新生成的部分。

### 3.3 长文本处理（128K 上下文窗口）

```python
from transformers import AutoTokenizer, AutoModelForCausalLM

model_name = "meta-llama/Meta-Llama-3-1-70B"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",
    device_map="auto"
)

# 超长文本输入（模拟）
long_context = "这是一段很长的文本。" * 10000  # 约 6 万个 token

inputs = tokenizer(long_context, return_tensors="pt", truncation=False)

# 长上下文问答
prompt = f"""
请总结以下文章的主要观点：

{long_context}

摘要：
"""

inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
outputs = model.generate(**inputs, max_new_tokens=512, temperature=0.5)
response = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(response)
```

> **说明**：Llama 3 支持 128K token 的上下文窗口，意味着可以一次性处理约 10 万字的内容。设置 `truncation=False` 防止自动截断。

## 四、性能亮点

Llama 3 405B 在多个基准测试中与 GPT-4 等领先模型匹敌：

- **MMLU**（综合知识）：87.3 分 vs GPT-4 的 85.1 分
- **HumanEval**（代码能力）：89.0 分
- **GSM8K**（数学推理）：96.8 分
- **MATH**（高等数学）：73.8 分

有趣的是，8B 小模型在同等参数规模下是"同类最佳"——它打败了 Mistral 7B 等竞争对手。

## 五、安全机制

Llama 3 引入了 **Llama Guard 3**，像一个"安全审查员"，在输入和输出两端同时工作：

- 输入端：过滤有害请求（比如教人做危险的事）
- 输出端：过滤不当回答（比如泄露隐私、产生偏见内容）

这在后训练阶段完成，让模型在保持有用的同时，尽量减少伤害。

## 六、我的理解总结

| 概念 | 我的理解 |
|------|---------|
| 参数 | 模型记住的知识量，越多越强但也越贵 |
| 预训练 | 通识教育，海量阅读，学语言基础 |
| 后训练 | 职业培训，学会听话、学会帮忙 |
| DPO | 直接告诉模型"好"和"更好"，让它自己学会判断 |
| 128K 上下文 | 能一次性读完一本厚书不忘记开头 |
| 缩放定律 | 大小和数据量要找到最优平衡，不是越大越好 |
