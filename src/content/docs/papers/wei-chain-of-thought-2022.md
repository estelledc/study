# Chain-of-Thought Prompting（思维链提示）学习笔记

## 一句话总结

让大语言模型"把解题过程写出来"，它的推理能力就会突然变强。

这就是 Chain-of-Thought（CoT）Prompting 的核心思想。

## 日常类比：心算 vs 列竖式

想象你有这样一道题：

> 食堂原来有 23 个苹果，今天运来了 6 箱，每箱 12 个。又送走了 15 个。现在有多少个？

如果你让一个人**心算**（不写过程），很多人会算错。
但如果你让他**列竖式**（一步一步写出来），他大概率算对。

大语言模型也是一样的。

它的训练方式决定了它本质上是一个"下一个词预测器"——根据前面的文字，猜下一个最可能出现的词。这种机制让它擅长"接话"，但不擅长"多步推理"。

Chain-of-Thought 的做法就是：**别让它直接给答案，先让它把推理步骤写出来。**

## 核心概念

### 1. 零样本（Zero-Shot）vs 少样本（Few-Shot）

在 CoT 论文出现之前，已经有"少样本提示"（Few-Shot Prompting）的概念：

- **零样本**：不给任何例子，直接问问题
- **少样本**：在问题前面给几个"问题-答案"的例子，让模型模仿

少样本已经比零样本好，但对复杂推理任务还不够。

### 2. 思维链（Chain of Thought）

这就是这篇论文的核心创新。

少样本提示给的是：

```
问题：...
答案：42
```

思维链提示给的是：

```
问题：...
推理过程：首先...然后...最后...
答案：42
```

关键区别在于**中间多了一步推理过程**。

### 3. 为什么有效？

论文发现，当模型足够大时，推理能力其实已经"藏在"模型里了，只是它不知道要展现出来。

思维链提示就像是告诉模型："你不需要一口气给答案，可以一步一步来。"

这触发了模型内部的推理能力。

### 4. 关键发现

论文做了大量实验，有几个重要发现：

- **规模阈值**：只有足够大的模型才会在 CoT 下表现更好（大约 100 亿参数以上）
- **不需要微调**：CoT 是一种 prompt 技巧，不需要重新训练模型
- **通用性**：在算术、常识推理、符号推理等多个任务上都有效
- **GSM8K 成绩**：用 5400 亿参数的模型，只给 8 个 CoT 示例，就达到了当时最佳

## 代码示例

### 示例一：数学推理（GSM8K 风格）

**不用思维链（直接给答案）：**

```python
prompt = """
Q: 娜塔莎每天爬楼梯的数量比伊夫琳多40%。伊夫琳每天爬100级。
   卡尔每天爬的数量是娜塔莎的三分之一。
   卡尔每天爬多少级？

A: """

# 模型可能直接回答（而且很可能错）：
# 输出：33
```

**用思维链（先写推理步骤）：**

```python
prompt = """
Q: 食堂有23个苹果。今天运来6箱，每箱12个。又送走了15个。
   现在有多少个？
A: 首先，6箱苹果一共有 6 × 12 = 72 个。
   然后，原来的23个加上新运来的72个，共有 23 + 72 = 95 个。
   最后，送走了15个，剩下 95 - 15 = 80 个。
   答案是80。

Q: 娜塔莎每天爬楼梯的数量比伊夫琳多40%。伊夫琳每天爬100级。
   卡尔每天爬的数量是娜塔莎的三分之一。
   卡尔每天爬多少级？
A: """

# 模型会模仿推理过程：
# 输出：首先，娜塔莎每天爬 100 + 40 = 140 级。
#       然后，卡尔每天爬 140 / 3 = 46.67 级。
#       四舍五入后答案是47级。
```

注意少样本中的第一个例子就是一个完整的 CoT 示例，模型会模仿这种"一步一步写出来"的格式。

### 示例二：常识推理（StrategyQA 风格）

**不用思维链：**

```python
prompt = """
Q: 熊猫能吃素活下去吗？
A: """

# 模型可能输出：是的
```

**用思维链：**

```python
prompt = """
Q: 企鹅能在北极生活吗？
A: 不能。企鹅主要生活在南半球，特别是南极地区。
   北极没有企鹅的自然栖息地。
   所以答案是不能。

Q: 熊猫能吃素活下去吗？
A: """

# 模型会模仿推理格式：
# 输出：虽然熊猫属于食肉目动物，但它们的饮食中99%是竹子。
#       竹子是植物。所以熊猫基本吃素。
#       所以答案是是的。
```

### 示例三：实际 Python 调用（OpenAI API）

```python
import openai

# 思维链提示模板
cot_prompt = """
Q: 李明有5个苹果，他买了3箱，每箱8个。
   他给了朋友一半，还剩多少个？
A: 首先，3箱苹果共有 3 × 8 = 24 个。
   然后，他一共有 5 + 24 = 29 个苹果。
   给了一半，剩下 29 / 2 = 14.5，即14个。
   答案是14。

Q: 工厂第一周生产120件产品，第二周是第一周的1.5倍，
   第三周又生产了80件。三周平均每周生产多少件？
A: """

response = openai.Completion.create(
    model="text-davinci-003",
    prompt=cot_prompt,
    max_tokens=200,
    temperature=0
)

answer = response.choices[0].text.strip()
print(answer)
# 预期输出类似：
# 第二周生产了 120 × 1.5 = 180 件。
# 三周总共生产了 120 + 180 + 80 = 380 件。
# 平均每周生产 380 / 3 = 126.67 件。
# 答案是127件。
```

## 论文实验框架

论文评估了三种主要能力：

| 任务类型 | 代表数据集 | 解决什么 |
|---------|-----------|---------|
| 算术推理 | GSM8K | 数学应用题 |
| 常识推理 | StrategyQA | 需要多步常识判断 |
| 符号推理 | Last Letter Concat | 按规则处理字母 |

每种任务都对比了：

- **Zero-Shot**：直接提问
- **Few-Shot**：给几个"问题+答案"示例
- **Chain-of-Thought Few-Shot**：给几个"问题+推理过程+答案"示例

结果一致表明：CoT 在三种任务上都显著优于传统少样本方法。

## 为什么这很重要？

用一句话说：

> CoT 证明了，大模型的推理能力不一定需要额外的训练数据或微调，
> 一个好的 prompt 设计就能"激活"它。

这改变了人们对大模型能力的理解——不是模型"不会"，而是模型"没被告诉要写出来"。

## 后续影响

这篇论文开启了大量后续研究：

- **Self-Consistency**：让模型多写几条推理路径，选最多的答案
- **Tree of Thoughts**：不只一条路径，而是像树一样探索多种可能
- **Instruction Tuning**：用 CoT 数据微调模型，让它学会自己推理

但所有这些方法，都建立在 CoT 论文的核心发现之上：

**让模型"一步一步想"，比"直接给答案"好得多。**

## 关键术语速查

| 术语 | 含义 |
|-----|------|
| Chain-of-Thought | 思维链——让模型输出推理步骤 |
| Zero-Shot | 零样本——不给任何例子 |
| Few-Shot | 少样本——给几个例子 |
| GSM8K | 小学数学题数据集，CoT 的主战场 |
| Prompt | 提示——给模型的输入 |
| 微调（Fine-tuning） | 用新数据继续训练模型 |
