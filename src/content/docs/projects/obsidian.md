# Obsidian 零基础学习笔记

## 一、Obsidian 是什么：从"笔记本"说起

想象一下你有一本普通的笔记本——你写在纸上，用笔记录想法。这本笔记本只属于你，放在你的书桌上，谁也看不到里面的内容。

Obsidian 做的事情和这本笔记本很像，只不过它不是纸和笔，而是**电脑和手机里的一堆纯文本文件（.md 文件）**。

核心一句话：**Obsidian 是一个本地优先的笔记应用，你的笔记以 Markdown 格式保存在你自己的设备上，不依赖任何云服务商。**

## 二、核心概念

### 1. Vault（库）

Vault 是 Obsidian 里最重要的概念。

你可以把 Vault 想象成一个**装满笔记的文件夹**。你的所有笔记文件（.md 文件）都放在这个文件夹里。

- 一个 Vault = 一个文件夹
- 文件夹里的每个 .md 文件 = 一篇笔记
- 你可以创建任意多个 Vault，比如"工作笔记库"、"学习笔记库"

在 Obsidian 中打开一个文件夹，就是打开一个 Vault。

```
my-vault/                    ← 这就是一个 Vault（就是一个普通文件夹）
├── Welcome.md               ← 一篇笔记
├── 日常笔记/
│   ├── 2026-06-13.md        ← 每日笔记
│   └── 会议记录.md
├── 项目/
│   ├── 项目A计划.md
│   └── 项目B计划.md
└── .obsidian/               ← Obsidian 的配置文件夹（不需要手动改）
```

### 2. Markdown 文件格式

Obsidian 的笔记就是普通的 Markdown 文本文件。这意味着：

- 用任何文本编辑器都能打开
- 不依赖 Obsidian 也能读写
- 数据永远属于你，不会被锁定

### 3. 双向链接（Backlinks）

这是 Obsidian 最核心的亮点。

**类比**：在维基百科里，每个词条都有很多超链接指向其他词条。当你点击链接跳转到另一篇文章后，文章底部会告诉你"有哪些文章链接到了当前这篇文章"——这就是双向链接。

在 Obsidian 里，你用 `[[ ]]` 来链接笔记。比如你在笔记 A 中写了 `[[学习笔记库/项目A计划]]`，就表示指向"项目A计划"那篇笔记。然后在"项目A计划"这篇笔记的右侧面板里，你会看到笔记 A 的链接——这就是**反向链接**。

### 4. 标签（Tags）

标签就像给笔记贴便利贴。你用一个 `#` 开头加上关键词来标记笔记：

```markdown
#编程 #python #入门笔记
#读书笔记
#工作
```

### 5. Graph View（图谱视图）

Obsidian 可以把你的所有笔记和它们之间的链接画成一张**关系图**。每个节点是一篇笔记，每条线表示两篇笔记之间有链接。

这个功能让你能看到自己知识体系的全貌，发现隐藏的联系。

## 三、Markdown 语法示例

Obsidian 使用标准的 Markdown 语法来编辑笔记。下面是最重要的几种语法。

### 示例 1：写一篇结构清晰的笔记

```markdown
# 这是什么编程语言

## 什么是 Python

Python 是一种**解释型**、**高级**、**通用**的编程语言。

## Python 的特点

- 语法简洁，容易上手
- 有丰富的第三方库
- 跨平台：可以在 Windows、Mac、Linux 上运行

## Python 能做什么

1. 网站开发
2. 数据分析
3. 人工智能

## 延伸阅读

- [[Python 安装教程]]
- [[第一个 Python 程序]]
```

## 四、常用功能

### 1. 每日笔记（Daily Notes）

Obsidian 可以自动生成每日笔记。你只需要点击按钮，它就会创建一个以日期命名的文件（如 `2026-06-13.md`），你可以在这篇笔记里记录今天做了什么、想到什么。

### 2. 命令面板（Command Palette）

按 `Ctrl/Cmd + P` 可以打开命令面板，输入关键词就能快速找到任何功能，比如"切换暗色模式"、"插入今日日期"等。

### 3. 社区插件

Obsidian 本身功能已经很强大，但社区还贡献了成千上万的免费插件，进一步扩展能力：

- **Calendar**：日历视图，快速跳转到任何一天的笔记
- **Kanban**：看板视图，管理项目和任务
- **Dataview**：数据查询，用 SQL 类似的语法检索笔记
- **Tasks**：跨笔记的任务管理

## 五、Obsidian 的三大设计哲学

Obsidian 官方提出了三个核心原则：

1. **你的思想属于你**：笔记存储在本地设备上，离线也能访问，连 Obsidian 公司自己也看不到你的内容。

2. **你的思维方式是独特的**：通过几千个社区插件和主题，Obsidian 可以调整成任何你喜欢的样子。

3. **你的知识应该持久**：使用开放的文件格式（Markdown 纯文本），你永远不会被锁定在某个平台里。换到别的软件，你的笔记依然能用。

## 六、适合谁用

- **学生**：整理课堂笔记，建立知识之间的联系
- **研究者**：管理大量参考文献和想法
- **写作者**：搭建个人知识库和写作大纲
- **开发者**：记录技术笔记、代码片段和项目规划
