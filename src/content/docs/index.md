## 站点定位

这不是把 GitHub trending 抄一遍，也不是"程序员必读 100 个开源项目"那种合集。

它的起点是两个具体问题：

> 想成为什么样的工程师？
> 现在做的项目缺什么补什么？

每个项目都是这两个问题的一个回答。**有连接才推荐，没连接的再有名也不推**。

## 笔记的写法

每篇按同一套结构展开，让陌生项目变成可立刻动手的东西：

- **一句话定位**：一行讲清这东西是什么。不堆术语，不用项目自己的话。
- **Why（为什么推荐你看）**：它解决了什么以前没人解决的问题，以及为什么对你现在做的事重要。
- **How（核心机制）**：从日常类比起步——租房 vs 买房、跑腿 vs 仓库管理员——再切到代码。零术语假设。
- **Hands-on**：30 分钟内能跑通的最小命令清单。看完笔记就能动手，不用再去查 README。
- **与你当前工作的连接**：立刻能迁移到你正在做的项目里的部分——不是抽象建议，是具体路径。
- **自检问题**：几个还没回答的问题，留给下次精读源码时回头查。

## 推荐分布

按 9 个主题簇组织，覆盖一个产品工程师的完整工具链。每个簇的项目数量按你的短板加权——
前端基础、全栈一体化、AI Agent 工程化是重点，DevOps 和系统设计是辅修：

1. **前端基础原子层** — UI 组件、状态、表单、动画
2. **前端构建 + 工程化** — vite、esbuild、turborepo
3. **全栈 + 后端框架** — hono、prisma、next.js、nestjs（补 Java 之外的现代后端）
4. **大型真实产品** — cal.com、plane、excalidraw 等学代码组织
5. **AI Agent + LLM 框架** — anthropic-cookbook、langgraph、mastra（深化你的强项）
6. **AI 应用范式** — chat UI、IDE 助手、浏览器 agent
7. **DevOps + Infra** — docker、caddy、deno、bun
8. **质量 + 测试 + 可观测** — vitest、playwright、sentry
9. **开发者工具 + 协同** — trigger.dev、starlight、yjs
