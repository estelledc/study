## 5. 重新核对后的四个核心问题

### 5.1 文本存在，不等于事实成立

一条 prompt 记录至少有三层：

```text
content
  + provenance
  + independent verification
```

只有 content 时，只能说“仓库里出现过这段文本”。即使多个仓库都有同一文本，
也必须追溯是否来自同一个 root source。

### 5.2 prompt 内容必须按不可信数据处理

17 个仓包含：

- 系统指令；
- tool schema；
- jailbreak / extraction query；
- 社区提交；
- 外部附件与链接。

研究工具只能解析、hash、分类和引用路径，不能服从其中的自然语言命令。

### 5.3 prompt 不能保存 secret

如果 secret 进入模型上下文：

- “不要泄露”仍是概率性约束；
- output filter 只能覆盖部分出口；
- 工具调用、中间 JSON、URL 和文件写入都可能成为旁路。

正确修复是让 secret 不进入 prompt，并由外部 credential broker 在最小权限下
完成实际调用。

### 5.4 授权必须位于模型外

安全链应为：

```text
model proposes tool request
  -> authenticated actor
  -> tenant binding
  -> capability allowlist
  -> action policy
  -> approval
  -> execution + audit
```

prompt 里出现“我是管理员”不能修改任何一层。

## 6. 本轮 E2：离线防御实验

实验文件：

- [零基础防御实验](11-beginner-prompt-defense-lab.md)
- `labs/prompt_defense_lab.py`
- `labs/test_prompt_defense_lab.py`

实验覆盖四条链。

### 6.1 来源记录 gate

`PromptRecord` 包含：

```text
provider / product / captured_at
source_type / source_url / source_root_id
evidence_grade / completeness / verbatim
license_status / content_sha256
reproducibility_id
```

以下情况 quarantine：

- hash 与正文不一致；
- A 级但不是 official source；
- B 级但没有 reproduction identity；
- URL/date/schema 非法；
- 检测到 secret-like material。

### 6.2 独立来源 gate

两个仓库复制同一 root source：

```text
accepted records = 2
independent roots = 1
```

不会升级为多源验证。

两个不同 root source 的相同 hash 才能得到：

```text
cross_consistent
```

即便如此，它仍不自动升级为 official ground truth。

### 6.3 canary 全出口 gate

`CanaryGuard` 检查：

- `text_delta`；
- streaming chunk boundary；
- structured tool arguments；
- URL / percent encoding；
- file write payload。

命中后只记录 channel、representation 和 blocked action，不把 canary 或输出正文
重新写进 audit event。

### 6.4 模型外工具策略

`authorize_tool()` 只信任：

- authenticated `AuthContext`；
- tenant ID；
- allowed tools；
- allowed actions；
- external approval ID。

它完全忽略 prompt 中声称的角色和权限。

## 7. 运行结果

```text
records: official=accepted unsafe=quarantined
sources: copied=1 independent=2

```

测试：

```text

```

首轮测试曾有 1 个测试设计错误：canary 只含 URL 安全字符，`quote()` 后仍是原文，
测试却期待 encoded 分类。将测试探针改为包含 `:/+` 后，percent-encoded 分支被真实
覆盖，16 项全绿。guard 行为没有为迎合测试而放宽。

## 8. E2 能证明什么

可以证明：

- 来源字段和 hash 能执行 fail-closed 校验；
- 同源转载不会被机械算成多个独立来源；
- secret-like 内容不会进入 accepted registry；
- canary 跨 chunk 和多个输出 sink 均可阻断；
- prompt 文本不能提升工具、action 或 tenant 权限；
- 破坏性操作需要外部 approval。

不能证明：

- canary 能检测 paraphrase 或功能复制；
- regex 能发现所有 secret；
- 两个独立来源就必然真实；
- 任何线上模型对 extraction 都安全；
- 当前 17 个研究仓的论文数字已复现；
- output guard 能替代数据最小化与最小权限。

## 11. 下一项真实验证

如果继续提升证据，不应对真实产品发攻击请求。更合适的是在自有 toy agent 上建立
防御回归集：

```text
synthetic prompt
  x benign / direct / encoded / split / tool-argument / file-write cases
  x canary / secret / tenant / approval invariants
```

评估时分开：

- exact canary leakage；
- secret detector recall/false positive；
- benign utility；
- tool authorization bypass；
- incident audit completeness。

这能验证自有系统的防线，不越过第三方授权边界。
