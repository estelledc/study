## 3. 四个漂移仓

### 3.1 Trellis：inline 是政策，不是 Codex 限制

旧材料写成：

```text
Codex 子 Agent 不能可靠继承 task context
  -> Trellis 默认 inline
```

upstream #440 明确修正为：

```text
Codex 可由调用方选择 fresh / bounded / full history
fresh-history 子 Agent 仍收到显式任务和 session config
Trellis 默认 inline 是避免依赖隐式 transcript 的政策选择
```

这一区分很重要：

- **能力事实**：Codex 支持不同 `fork_turns`。
- **Trellis 政策**：默认让主 Agent inline 实现。
- **设计理由**：降低隐式上下文依赖和递归 dispatch 风险。

不能把一个产品的默认政策写成另一个产品的能力缺失。

### 3.2 Spec Kit：从 SDD 工具扩展为 Harness primitives

三个提交只改文档，但定位更清楚：

- Agentic SDD 仍是核心流程。
- CLI 还管理 integrations、extensions、presets、workflows 和 bundles。
- Workflows 支持条件、循环、fan-out/fan-in、human checkpoint 和暂停恢复。
- 新增 agentic bugfix 的 assess -> fix -> validate 流程。

源码主链未变，但“Spec Kit 只负责从 spec 到 tasks”已经过窄。更准确的定位是：它以 SDD 为核心，同时向可组合 Harness primitive 扩展。

### 3.3 GSD Core：重试升级必须同时改变 model 和 effort

旧 bug：

```text
attempt 增加
  -> reasoning effort 升级
  -> model 仍停在原 tier
```

当前修复让显式 `--attempt` 同时经过 `resolveModelForTier`：

```text
attempt 0 -> light/standard
attempt 1 -> 下一 tier
达到 max_escalations -> 固定在 heavy
```

同时，`resolve_model_ids: "omit"` 现在按当前 runtime 和项目显式配置判断，避免一个共享默认值污染支持原生 model alias 的 Claude runtime。

这强化两个 Harness 原则：

1. 重试策略必须作用于完整执行配置，不能只改一个字段。
2. 机器级默认值不能无条件覆盖项目或 runtime 级事实。

### 3.4 Compound Engineering：task list 是用户视图，不是工作流源真相

新增 `ce-handoff` 保存跨 session continuity；`ce-brainstorm` 又把六步 workflow spine 投影到宿主 task 工具：

1. 检查已有内容。
2. 询问范围。
3. 比较方案并推荐。
4. 写入前确认范围。
5. 写 requirements plan。
6. 提供下一步。

关键约束：

- task 名称描述用户可验收的结果，不写内部 phase。
- 条件分支只在 gate 真正触发时创建 task。
- skip 要显式记录，不能假装 complete。
- task list 只是视图，不改变 phase 的真实 exit condition。

这与本研究的核心判断一致：计划 UI、状态文件和流程规则是三种不同事实。

## 5. 重新验收后的稳定结论

1. **状态字段和工件存在必须联合判断。**  
   `status=in_progress` 不证明 PRD、实现上下文和检查上下文齐备。

2. **task list 是投影，不是 source of truth。**  
   用户看见的进度条不能替代 phase exit condition 和验证证据。

3. **上下文继承是策略选择。**  
   fresh context、bounded history、full history 各有成本，不能用“平台不支持”掩盖 Harness 取舍。

4. **重试必须是完整执行策略。**  
   model、effort、预算、attempt 和退出条件要一起收敛。

5. **完成必须绑定验证。**  
   生成很多 spec/task 文件不等于外部任务已接受、合并或上线。
